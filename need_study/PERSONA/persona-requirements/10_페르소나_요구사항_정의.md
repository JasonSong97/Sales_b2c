## 2. 제품 정의

### 2.1 한 줄 정의

**영업자가 회사 시스템에 남기기 어렵거나 흩어져 있는 고객·거래처·매장·매물의 현장 정보를 모바일에서 10초 안에 찾고, 다음 행동을 놓치지 않게 하는 개인 영업 카드 앱**

### 2.2 제품이 해결할 문제

13개 피드백에서 공통적으로 확인된 문제는 다음과 같다.

1. 회사 CRM/ERP/본사 시스템은 있지만, 현장 메모와 관계 정보는 잘 담기지 않는다.
2. 고객 정보가 엑셀, 카카오톡, 명함, 수첩, 사진첩, 머릿속에 흩어져 있다.
3. 방문 직전 필요한 정보를 빠르게 찾기 어렵다.
4. 상담 직후 후속 조치가 누락된다.
5. 재구매, 만기, 약정, 제품 소진, 미방문 같은 반복 타이밍을 놓친다.
6. 자동화가 과하면 영업자와 고객의 관계를 해친다.

### 2.3 제품이 하지 않을 일

| 제외 영역 | 제외 이유 | 관련 피드백 |
|----------|-----------|-------------|
| 본사 CRM/ERP 대체 | 대부분 회사 시스템이 이미 있고 대체 불가 | 자동차, 산업재, SaaS, 반도체, 제약 |
| 견적/계약 자동 생성 | 가격/법률/회사 정책 책임 문제 | 자동차, 부동산, 산업재 |
| 주문/개통/결제 처리 | 본사 시스템 영역 | 도매, 휴대폰, 산업재 |
| 자동 고객 메시지 발송 | 단골 관계 훼손, 스팸 우려 | 자동차, 보험, 방문 화장품 |
| 민감정보 무제한 저장 | 개인정보/보안/컴플라이언스 위험 | 보험, 휴대폰, 제약, 방산, 반도체 |
| 방산용 일반 클라우드 SaaS | 보안상 부적합 | 방산 |

## 3. 타깃 정의

### 3.1 1차 타깃

1차 타깃은 MVP 검증에 가장 적합한 사용자군이다.

| 타깃 | 선정 이유 |
|------|-----------|
| 산업재 영업 | 거래처/담당자/품목/교체 주기 관리 니즈가 강함 |
| 도매 영업 | 주문 패턴, 미수금, 루트, 오프라인 조회 니즈가 강함 |
| 자동차 영업 | 출고 후 재구매, 시승, 소개망 관리가 매출과 직접 연결 |
| 보험 설계사 | 가족 관계, 만기, 생애 이벤트, 소개망 관리가 핵심 |
| 방문 화장품 상담사 | 단골, 피부 타입, 제품 소진, 샘플 반응 관리가 핵심 |
| 미용실/살롱 영업 | 인물 관계도, 교육/시연, 제품 소진 관리가 중요 |

### 3.2 2차 타깃

| 타깃 | 2차인 이유 |
|------|------------|
| 부동산 중개 | 매물/고객 조건 매칭이 필요해 업종 특화 모델이 큼 |
| 휴대폰 매장 | 약정/개통/민감정보 정책과 충돌하지 않는 설계 필요 |
| 화장품 리테일 | 매장/사진/품절/리포트 중심이라 고객 CRM과 모델이 다름 |

### 3.3 제한 타깃

| 타깃 | 제한 이유 |
|------|-----------|
| SaaS B2B AE | Salesforce 등 회사 CRM 대체 불가. 개인 보조 도구로만 가능 |
| 반도체 영업 | 보안/대형 계정/장기 프로젝트 구조 필요 |
| 제약 MR | 컴플라이언스 가드레일이 필수 |

### 3.4 제외 타깃

| 타깃 | 판단 |
|------|------|
| 방산 영업 | 일반 개인 클라우드 SaaS에서 제외. 폐쇄망/감사 로그/권한 등급 기반 별도 제품 필요 |

## 4. MVP 범위

### 4.1 MVP 목표

MVP의 목표는 업종별 고급 기능을 모두 구현하는 것이 아니다. 1차 타깃 영업자가 공통으로 겪는 문제를 해결해야 한다.

**MVP 목표**

> 모바일에서 고객/거래처 카드를 10초 안에 찾고, 현장 메모와 다음 액션을 30초 안에 저장할 수 있게 한다.

### 4.2 MVP 포함 기능

| ID | 기능 | 설명 | 우선순위 |
|----|------|------|----------|
| MVP-01 | 카드 관리 | 고객, 거래처, 회사, 매장, 기관 등 영업 대상을 카드로 관리 | P0 |
| MVP-02 | 빠른 검색 | 이름, 전화번호, 회사명, 매장명, 태그, 메모 키워드 검색 | P0 |
| MVP-03 | 현장 메모 | 회사 시스템과 분리된 개인 메모 작성 | P0 |
| MVP-04 | 태그 | 재구매, 미방문, 견적후속, 샘플, 만기 등 상태 표시 | P0 |
| MVP-05 | 다음 액션 | 후속 연락/방문/처리 예정일 관리 | P0 |
| MVP-06 | 저장 필터 | 자주 보는 조건을 저장 | P0 |
| MVP-07 | 모바일 우선 UI | 방문 직전/이동 중 한눈에 확인 | P0 |
| MVP-08 | 민감정보 경고 | 저장하면 안 되는 정보 입력 시 경고 | P0 |
| MVP-09 | 사용자별 데이터 분리 | 개인 계정 단위 데이터 분리 | P0 |
| MVP-10 | 최근 카드 오프라인 조회 | 최근 열람 카드와 메모는 네트워크가 약해도 조회 | P1 |

### 4.3 MVP 제외 기능

| 기능 | 제외 사유 |
|------|-----------|
| PDF Export | 다수 persona에서 현장 사용 빈도 낮음 |
| 명함 OCR | 일부 업종에는 유용하지만 공통 MVP 핵심은 아님 |
| 자동 메시지 발송 | 사용자 거부감이 큼 |
| 견적/계약 자동 생성 | 책임 소재와 회사 정책 문제 |
| 주문/개통/결제 처리 | 회사 시스템 영역 |
| AI 추천 | 초기 데이터 부족과 오추천 책임 문제 |

## 5. 기능 요구사항

### FR-01. 카드 관리

**설명**  
영업 대상은 카드로 관리한다. 카드 타입은 고객, 거래처, 회사, 매장, 매물, 기관, 계정 등으로 확장 가능해야 한다.

**사용자 스토리**

> 영업자로서, 나는 방문 전 고객이나 거래처 카드를 열어 최근 메모와 다음 액션을 확인하고 싶다. 그래야 이전 상담 맥락을 잊지 않고 이어갈 수 있다.

**수용 기준**

- 사용자는 카드를 생성/수정/삭제할 수 있다.
- 카드에는 이름, 연락처, 지역, 타입, 중요도, 요약 메모가 있다.
- 카드에는 메모, 태그, 다음 액션이 연결된다.
- 카드 상세 화면 상단에는 핵심 태그, 마지막 메모, 다음 액션이 표시된다.

**근거 피드백**

- 전 페르소나 공통
- 산업재, 자동차, 보험, 도매, 부동산에서 특히 강함

**우선순위**: P0

### FR-02. 빠른 검색

**설명**  
모바일 첫 화면에서 영업 대상을 빠르게 찾는다.

**사용자 스토리**

> 영업자로서, 나는 고객 이름이 정확히 기억나지 않아도 전화번호, 회사명, 태그, 메모 키워드로 찾아야 한다.

**수용 기준**

- 이름/회사명/매장명/전화번호로 검색 가능하다.
- 태그와 메모 본문으로 검색 가능하다.
- 최근 검색과 최근 열람 카드를 보여준다.
- 모바일에서 검색 결과가 2초 이내 표시되는 것을 목표로 한다.

**근거 피드백**

- 자동차: 고객 재방문 시 즉시 조회
- 산업재: 공장/담당자 조회
- 도매: 거래처 빠른 조회
- 부동산: 고객 조건/매물 검색

**우선순위**: P0

### FR-03. 현장 메모

**설명**  
공식 시스템에 남기기 어려운 관계, 성향, 선호, 이슈를 기록한다.

**사용자 스토리**

> 영업자로서, 나는 고객의 실제 결정권자, 선호, 거절 이유, 다음에 꺼낼 이야기를 개인 메모로 남기고 싶다.

**수용 기준**

- 카드별 메모를 작성할 수 있다.
- 메모에 태그를 붙일 수 있다.
- 메모 본문 검색이 가능하다.
- 민감 메모로 표시할 수 있다.
- 메모 작성/수정 시간이 기록된다.

**명칭 원칙**

`Off-the-record`라는 표현은 제품 UI에서 사용하지 않는다. `현장 메모`, `개인 메모`, `상담 메모`를 사용한다.

**우선순위**: P0

### FR-04. 태그

**설명**  
영업 상태와 특성을 빠르게 분류한다.

**기본 태그 예시**

- 재구매
- 미방문
- 견적후속
- 샘플
- 만기
- 시승
- 품절
- 클레임
- 소개
- 민감

**수용 기준**

- 사용자는 카드와 메모에 태그를 붙일 수 있다.
- 태그로 검색/필터링할 수 있다.
- 자주 쓰는 태그를 상단에 표시한다.
- 업종 템플릿별 기본 태그를 제공할 수 있다.

**우선순위**: P0

### FR-05. 다음 액션

**설명**  
카드별로 다음에 해야 할 행동을 관리한다.

**사용자 스토리**

> 영업자로서, 나는 상담이 끝난 직후 다음 연락일과 해야 할 일을 남기고 싶다. 그래야 후속 조치를 놓치지 않는다.

**필드**

- 액션 제목
- 예정일
- 상태: 예정/완료/보류
- 관련 카드
- 관련 메모

**수용 기준**

- 카드에서 다음 액션을 추가할 수 있다.
- 오늘/이번 주 액션 목록을 볼 수 있다.
- 완료 처리할 수 있다.
- 예정일이 지난 액션은 별도로 표시한다.

**근거 피드백**

- SaaS: next_action과 due date 필요
- 산업재: 견적/샘플/테스트 후속
- 보험: 보장 점검
- 제약: 요청 자료 전달
- 화장품 리테일: 매장 이슈 처리

**우선순위**: P0

### FR-06. 저장 필터

**설명**  
자주 보는 조건을 저장해서 반복 조회한다.

**필터 예시**

- 오늘 연락할 고객
- 이번 주 방문할 거래처
- 재구매 임박
- 약정 만료 3개월 이내
- 견적 후 미회신
- 샘플 제공 후 미구매
- 품절 발생 매장
- 미방문 거래처

**수용 기준**

- 사용자는 검색/필터 조건을 저장할 수 있다.
- 저장 필터를 첫 화면에서 바로 실행할 수 있다.
- 필터에는 태그, 날짜, 카드 타입, 지역 조건을 포함할 수 있다.

**우선순위**: P0

### FR-07. 리마인더

**설명**  
리마인더는 고객에게 자동 발송하지 않고, 영업자가 직접 연락하도록 돕는다.

**수용 기준**

- 예정일 기반 리마인더를 생성한다.
- 오늘/이번 주 리마인더를 볼 수 있다.
- 리마인더는 카드와 연결된다.
- 연락 문구 초안은 제공할 수 있으나 자동 발송하지 않는다.

**근거 피드백**

- 자동차: 출고 후 재구매
- 보험: 만기/갱신
- 휴대폰: 약정/할부 만료
- 방문 화장품: 제품 소진
- SaaS: 알림 Phase 상향 요청

**우선순위**: P1

### FR-08. 업종별 카드 템플릿

**설명**  
모든 업종을 하나의 고객/딜 구조로 처리하지 않고, 업종별 기본 필드와 태그를 제공한다.

**템플릿 후보**

| 업종 | 템플릿 |
|------|--------|
| 자동차 | 고객, 차량, 시승, 출고, 소개 |
| 방문 화장품 | 고객, 피부, 제품, 샘플, 재구매 |
| 화장품 리테일 | 매장, 담당자, 방문, 품절, 경쟁사 |
| 살롱 | 살롱, 원장/실장/디자이너, 교육, 시연 |
| 산업재 | 공장, 부서, 담당자, 품목, 설비 |
| 보험 | 고객, 가족, 계약, 생애 이벤트 |
| 휴대폰 | 고객, 기기, 약정, 가족 결합 |
| 부동산 | 고객 조건, 매물, 임장, 계약 단계 |
| 제약 | 병의원, 의료진, 제품 반응 |
| 도매 | 거래처, 주문 패턴, 미수, 루트 |

**우선순위**: P1

### FR-09. 사진/첨부

**설명**  
현장 사진, 매물 사진, 진열 사진, 설비 사진, 명함 이미지를 카드에 연결한다.

**수용 기준**

- 카드에 사진을 첨부할 수 있다.
- 사진에 태그를 붙일 수 있다.
- 사진은 방문 기록 또는 메모와 연결된다.

**근거 피드백**

- 화장품 리테일: 진열/품절/경쟁사 매대 사진
- 부동산: 매물 사진
- 산업재: 설비 사진
- 살롱: 시연/교육 사진

**우선순위**: P2

### FR-10. Import/Export

**설명**  
기존 고객/거래처 목록을 가져오고, 필요한 경우 제한적으로 내보낸다.

**Import 수용 기준**

- CSV/Excel Import를 지원한다.
- 컬럼 매핑을 제공한다.
- 중복 항목을 확인한다.

**Export 수용 기준**

- 민감 필드를 제외할 수 있다.
- 전화번호 마스킹 옵션을 제공한다.
- Export 전 개인정보 경고를 표시한다.

**우선순위**

- Import: P1
- Export: P2

### FR-11. 카톡 채팅방 검색

**설명**
카카오톡 1:1 채팅방에서 약속·주문·고객 메시지를 검색·정리하고 카드와 매칭한다.

**사용자 스토리**

> 영업자로서, 나는 1년 전 카톡 대화에서 "이 고객이 어떤 보장을 원했는지", "이 사장님이 어떤 주문을 했는지"를 다시 찾고 싶다. 카톡 검색만으로는 찾기 어렵다.

**수용 기준**

- 카톡 1:1 채팅방을 카드에 연결할 수 있다.
- 채팅방 메시지에서 키워드 검색이 가능하다.
- 카드 상세 화면에 "마지막 카톡 5개"를 표시할 수 있다.
- 자동 매칭 실패 시 수동 카드 연결을 지원한다.

**근거 피드백**

- 보험: 카톡 300+ 채팅방에서 1년 전 대화 검색 거의 불가
- 방문 화장품: 단골 200명 카톡에서 약속·메모 찾기 어려움
- 자동차: 단골 500명 카톡 + 인스타 DM 흩어짐
- 산업재·도매: 거래처별 카톡 메시지 누락

**제한**

- 카톡 자동 발송 금지 (FR-07 리마인더 원칙 적용)
- 카톡 데이터는 사용자가 직접 가져온다 (자동 크롤링 금지)
- 민감 메시지는 민감 메모로 분류 (NFR-05)

**우선순위**: P1

## 6. 비기능 요구사항

### NFR-01. 모바일 우선

모바일 사용성을 최우선으로 한다. PC는 정리, 대량 입력, Export용이다.

**수용 기준**

- 모바일 첫 화면에서 검색창이 가장 먼저 보인다.
- 카드 상세에서 핵심 정보가 첫 화면에 들어온다.
- 주요 작업은 한 손으로 가능해야 한다.

### NFR-02. 빠른 입력

현장에서는 긴 폼을 입력하지 않는다.

**수용 기준**

- 빠른 메모 작성 가능
- 태그 원터치 추가
- 다음 액션 빠른 생성
- 임시 저장 가능

### NFR-03. 쉬운 모드

방문 화장품, 도매, 현장 영업자에게는 쉬운 UI가 필요하다.

**수용 기준**

- 큰 글씨 모드
- 큰 버튼
- 최소 메뉴
- 핵심 카드 중심 화면

### NFR-04. 보안

개인 영업 데이터와 민감 메모를 보호한다.

**수용 기준**

- 모든 데이터는 사용자별로 분리한다.
- 민감 메모 잠금 기능을 제공한다.
- 계정 삭제 시 데이터 삭제 정책을 명시한다.
- Export 전 개인정보 경고를 제공한다.

### NFR-05. 민감정보 제한

아래 정보는 저장 금지 또는 경고 대상이다.

- 주민번호
- 신분증 이미지
- 계좌번호
- 환자 정보
- 방산 기밀
- 처방 대가성 표현
- 리베이트 관련 기록
- 법률 판단이 필요한 계약 문구

## 12. 기존 기획서 수정 지침

`01_서비스_기획서.md`는 다음 방향으로 수정한다.

1. `거래처/딜` 중심 표현을 `카드/메모/다음 액션` 중심으로 바꾼다.
2. MVP에서 OCR과 PDF Export의 우선순위를 낮춘다.
3. 다음 액션, 저장 필터, 리마인더의 우선순위를 올린다.
4. 자동화보다 영업자가 직접 판단하고 연락하는 보조 도구로 정의한다.
5. 업종별 차이는 MVP에 모두 넣지 않고 카드 템플릿으로 분리한다.
6. 방산은 제외하고, 제약/반도체/SaaS는 제한 타깃으로 둔다.
7. 보안/민감정보 저장 정책을 기능 요구사항으로 명시한다.
8. **v2 추가**: 시니어 모드(쉬운 모드 NFR-03)는 P0로 격상한다.
9. **v2 추가**: 카톡 채팅방 검색을 FR-11로 신설한다.
10. **v2 추가**: 부동산 4트랙·SaaS MEDDIC·제약 다층 트랙은 `relationships.roles[]` 다중 매핑으로 해결한다.
11. **v2 추가**: 본사 정책 변경 요약·보고서 자동 초안은 1차 MVP에 넣지 않고, 휴대폰·H&B 2차 타깃 확정 후 별도 모듈로 검토한다.

## 13. 성공 기준

### 정량 기준

| 항목 | 목표 |
|------|------|
| 카드 검색 시간 | 모바일에서 10초 이내 |
| 메모 저장 시간 | 상담 직후 30초 이내 |
| 다음 액션 등록률 | 활성 사용자 60% 이상 |
| 7일 재방문율 | 베타 사용자 50% 이상 |
| 저장 필터 사용률 | 활성 사용자 40% 이상 |

### 정성 기준

사용자가 아래처럼 말하면 요구사항이 맞게 구현된 것이다.

- "방문 전에 이 카드만 보면 기억이 난다."
- "회사 시스템에는 못 적는 걸 여기에 적는다."
- "누구에게 다시 연락해야 하는지 안 놓친다."
- "엑셀보다 빠르고 카톡보다 찾기 쉽다."
- "자동으로 보내지는 건 없어서 부담이 없다."
