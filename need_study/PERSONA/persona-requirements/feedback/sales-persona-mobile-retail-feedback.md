## 1. 전체 평가

휴대폰 매장 영업 관점에서 현재 기획은 **단골 고객 재방문과 약정 만료 관리를 위한 개인 CRM**으로 가능성이 있습니다. 매장 시스템은 개통과 정산 중심이고, 고객이 언제 약정이 끝나는지, 가족 결합이 어떻게 묶여 있는지, 어떤 요금제를 싫어했는지는 개인 메모로 관리되는 경우가 많습니다.

다만 휴대폰 영업은 개인정보, 신분증, 통신사 정책, 보조금 규정이 민감합니다. 이 서비스는 정책 계산이나 개통 처리 영역에 들어가면 안 됩니다.

### 1.1 적합도 요약

| 기획 요소 | 적합도 | 이유 |
|----------|--------|------|
| 개인 고객 관리 | 높음 | 단골, 가족 결합, 약정 만료 관리 필요 |
| 필터/검색 | 매우 높음 | 약정 만료, 기기 변경 가능 고객 조회 |
| 모바일 동기화 | 높음 | 매장/외부 행사 중 바로 확인 |
| Off-the-record 메모 | 높음 | 요금제 선호, 가족 상황, 민원 성향 기록 |
| Excel Import/Export | 보통 | 기존 고객 목록 이전에 필요 |
| 명함 OCR | 낮음 | 일반 고객은 명함보다 휴대폰 번호 중심 |
| PDF Export | 낮음 | 공식 서류는 통신사 시스템 처리 |

## 2. 기능별 실용성 평가

### 2.1 영업 데이터 관리(CRUD)

**실용성**: 높음

필요 필드:

| 구분 | 필드 |
|------|------|
| 고객 | 이름, 연락처, 통신사, 선호 연락 방식 |
| 기기 | 현재 기종, 개통일, 약정 만료일, 할부 잔여 |
| 요금제 | 현재 요금제, 데이터 사용 성향, 결합 여부 |
| 가족 | 가족 회선, 인터넷/TV 결합, 변경 가능 시점 |
| 상담 | 원하는 기종, 가격 민감도, 거절 사유 |
| 민감 제외 | 신분증 사본, 주민번호 등은 저장 금지 |

**고객 응대 중 5분 안에 열까?**  
네. "예전에 왜 바꾸지 않으셨지?", "가족 결합이 있었나?"를 바로 확인해야 합니다.

### 2.2 Import/Export

**실용성**: 보통

초기 고객 목록 이전에는 필요합니다. 다만 통신 고객 정보는 민감하므로 Export는 제한적이어야 합니다.

권장:

- 전화번호 마스킹 옵션
- 민감정보 저장 금지 안내
- 백업 암호화

### 2.3 필터 시스템

**실용성**: 매우 높음

필요 필터:

- 약정 만료 3개월 이내
- 할부 종료 예정
- 가족 결합 가능 고객
- 인터넷/TV 결합 만료
- 아이폰/갤럭시 신제품 관심 고객
- 고가 요금제 거부 고객
- 최근 상담 후 미개통 고객
- 단골 고객

현재 기획의 "이번 달 구매 가능성 높은 사람"은 휴대폰 매장에서는 **기기 변경 타이밍이 온 고객**입니다.

### 2.4 명함 OCR

**실용성**: 낮음

법인폰 고객에는 일부 유용하지만, 일반 매장 영업에서는 명함보다 휴대폰 번호, 가족 회선, 약정 만료일이 더 중요합니다.

### 2.5 Off-the-record 메모

**실용성**: 높음

예시:

- 부모님 회선까지 같이 관리함
- 데이터 많이 쓰지만 고가 요금제는 싫어함
- 이전에 위약금 때문에 불만이 있었음
- 아이폰만 선호
- 자녀 입학 시 태블릿 가능성
- 문자보다 카톡 선호

단, 신분증 이미지, 주민번호, 계좌번호 같은 민감정보는 저장하지 못하게 해야 합니다.

## 3. 기획서가 놓친 휴대폰 매장 페인포인트

### 3.1 약정/할부 만료 관리

이 업종의 재구매 타이밍은 약정 만료입니다. 이 기능이 없으면 사용 이유가 약합니다.

필수:

- 개통일
- 약정 종료일
- 할부 종료일
- 변경 가능일
- 만료 전 연락 예정일

### 3.2 가족 결합 관리

휴대폰 매출은 개인 한 명보다 가족 단위로 커집니다. 가족 회선, 인터넷/TV 결합, 자녀 개통 가능성을 관리해야 합니다.

### 3.3 통신사 정책 변화 메모

정책은 자주 바뀌지만 이 서비스가 자동으로 정책 계산을 하면 위험합니다. 대신 고객별로 "정책 좋아지면 연락" 같은 태그를 남기는 정도가 적절합니다.

### 3.4 직원 교육과 인수인계

매장에서는 직원이 바뀌어도 단골 응대 이력이 이어져야 합니다. 개인용이더라도 매장 계정/공유 범위 옵션을 고려할 수 있습니다.

## 4. 우선순위 제안

### Phase 1

- 고객 카드
- 기기/통신사/약정 만료일
- 상담 메모
- 빠른 검색
- 약정 만료 필터

### Phase 2

- 가족 결합 관리
- 기기 변경 예정 알림
- 신제품 관심 태그
- 민감정보 저장 방지 UX

### Phase 3

- Excel Import
- 매장 내 공유 권한
- 상담 이력 요약

### Phase 4 이후 또는 제외

- 개통 처리
- 보조금/정책 자동 계산
- 신분증 저장
- 자동 영업 문자 발송

## 6. 핵심 요약(TL;DR)

휴대폰 매장 영업에게 이 서비스는 **약정 만료와 가족 결합을 놓치지 않는 고객 관리 도구**가 되어야 합니다.

반드시 보강할 기능:

1. 약정/할부 만료일 관리
2. 가족 회선/결합 관리
3. 신제품 관심 태그
4. 민감정보 저장 금지 UX
5. 기기 변경 예정 고객 필터

개통이나 보조금 계산은 하지 말고, **다시 연락해야 할 고객을 놓치지 않는 도구**로 좁히는 것이 맞습니다.
