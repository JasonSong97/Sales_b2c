> **피드백 작성자**: 임철수 (18년차 방산/공공 영업, 장기 입찰·기관 관계 담당)

## 1. 전체 평가

방산 영업 관점에서 현재 기획은 **일반 개인용 SaaS로는 거의 사용하기 어렵습니다.** 이유는 명확합니다. 방산 영업은 보안, 입찰 규정, 기관 접촉 기록, 내부 승인 절차가 강하게 걸려 있기 때문입니다.

다만 기획의 일부 방향, 특히 `개인 메모`, `관계 히스토리`, `다음 액션 관리`는 방산 영업에도 필요합니다. 문제는 그것을 클라우드 개인 SaaS로 제공하면 안 된다는 점입니다.

### 1.1 적합도 요약

| 기획 요소 | 적합도 | 이유 |
|----------|--------|------|
| 개인용 영업 관리 | 낮음 | 회사 보안 정책상 외부 SaaS 사용 제한 |
| Off-the-record 메모 | 위험 | 민감 정보로 분류될 수 있음 |
| 모바일/PC 동기화 | 낮음 | 외부 클라우드 동기화가 보안 리스크 |
| 필터/검색 | 보통 | 내부 시스템이라면 유용 |
| Excel Export | 위험 | 자료 반출 이슈 |
| 명함 OCR | 낮음 | 기관 인물 정보 저장 자체가 민감 |

### 1.2 결론

이 persona는 현재 B2C 개인 영업 SaaS의 직접 타깃으로 삼으면 안 됩니다. 방산 영역은 **온프레미스/폐쇄망/기업 계약형 별도 제품**으로 분리해야 합니다.

## 2. 기능별 실용성 평가

### 2.1 영업 데이터 관리(CRUD)

**실용성**: 내부망이라면 높음, 개인 SaaS라면 낮음

방산 영업에서 필요한 데이터는 단순 고객/딜이 아닙니다.

필요 엔티티:

| 엔티티 | 예시 |
|--------|------|
| 기관 | 군, 방사청, 연구소, 공기업, 협력사 |
| 인물 | 담당자, 의사결정 영향력, 보직 변경 이력 |
| 사업 | 예산 사업, RFI/RFP, 입찰, 제안, 평가, 계약 |
| 파트너 | 컨소시엄, 협력업체, 하도급 관계 |
| 리스크 | 보안, 규정, 경쟁사, 납기, 기술 성숙도 |

**미팅 직후 5분 안에 열까?**  
개인 휴대폰 SaaS라면 열지 않습니다. 회사 승인된 보안 단말이나 내부망 시스템이라면 기록합니다.

### 2.2 Import/Export

**실용성**: 매우 제한적

Excel Import 자체는 유용할 수 있으나 자료 등급, 반출 통제, 접근 권한이 선행되어야 합니다. Export 기능은 특히 위험합니다.

권장:

- 기본적으로 Export 제한
- 관리자 승인 후 Export
- 워터마크/감사 로그
- 열람/수정/다운로드 이력 기록

### 2.3 필터 시스템

**실용성**: 내부 시스템이라면 높음

필요 필터:

- 입찰 단계별 사업
- 예산 반영 가능성
- RFP 예상 시점
- 보안 검토 필요 건
- 경쟁사 우세 건
- 파트너 조정 필요 건
- 다음 승인 미팅 예정 건

현재 기획의 "구매 가능성 높은 사람"은 방산에서는 맞지 않습니다. 방산은 사람보다 **사업 단계와 예산 사이클**이 중요합니다.

### 2.4 명함 OCR

**실용성**: 낮음 또는 위험

기관 담당자 명함을 외부 SaaS OCR로 처리하는 것은 보안상 문제가 될 수 있습니다. OCR 서버가 외부에 있으면 더 어렵습니다.

필요하다면:

- 로컬 OCR
- 내부망 OCR
- 저장 전 등급 분류
- 개인정보/기관정보 처리 정책

### 2.5 Off-the-record 메모

**실용성**: 위험하지만 내부적으로는 중요

방산에서는 비공식 메모가 가장 민감합니다. "누가 어떤 입장이다", "어느 부서가 반대한다" 같은 정보는 내부 영업 전략에는 중요하지만 외부 개인 SaaS에 저장하면 안 됩니다.

따라서 기능명도 `Off-the-record`보다는 `비공개 내부 메모` 또는 `제한 열람 메모`가 맞습니다.

필수 조건:

- 권한 등급
- 열람자 제한
- 감사 로그
- 암호화
- 데이터 보존/삭제 정책

## 3. 기획서가 놓친 방산 영업 페인포인트

### 3.1 긴 사업 주기

방산 영업은 몇 주, 몇 달이 아니라 몇 년 단위입니다. 현재 기획의 `expected_close_date` 정도로는 부족합니다.

사업 단계:

1. 소요 제기
2. 예산 반영
3. RFI
4. RFP
5. 제안
6. 평가
7. 협상
8. 계약
9. 납품/검수

### 3.2 기관 인사 이동

담당자가 바뀌면 관계가 초기화됩니다. 보직 변경 이력, 전임자/후임자 관계를 추적해야 합니다.

### 3.3 파트너/컨소시엄 관리

방산은 단독 영업보다 협력 구조가 많습니다. 협력사, 기술 파트너, 제조사, SI 업체의 역할과 이해관계를 관리해야 합니다.

### 3.4 보안 등급과 감사 로그

현재 기획에는 보안이 로그인/사용자 분리 수준으로만 표현되어 있습니다. 방산에는 부족합니다.

필요:

- 데이터 등급
- 프로젝트별 접근 권한
- 다운로드 제한
- 감사 로그
- 폐쇄망 배포

## 4. 우선순위 제안

### 개인용 SaaS에서는 제외 권장

- 기관 담당자 정보 저장
- 비공식 메모
- 명함 OCR
- 자료 Export
- 외부 클라우드 동기화

### 별도 엔터프라이즈 제품으로 만들 경우 Phase 1

- 기관/사업 카드
- 사업 단계 관리
- 권한 관리
- 감사 로그
- 내부 메모

### Phase 2

- 예산/입찰 일정 관리
- 파트너 관계 관리
- 인사 이동 이력
- 승인 워크플로우

### Phase 3

- 내부 문서 연결
- 보안 등급별 검색
- 폐쇄망 OCR

## 5. 사용 의향

| 조건 | 사용 의향 | 이유 |
|------|----------|------|
| 현재 개인 SaaS 형태 | 사용 불가 | 보안/규정 리스크 |
| 회사 승인 내부망 제품 | 검토 가능 | 관계와 사업 단계 관리는 필요 |
| Export 자유 제공 | 거부 | 자료 반출 위험 |
| Off-the-record 메모 외부 저장 | 거부 | 감사/보안 문제 |
| 폐쇄망/권한/감사 로그 제공 | 사용 가능 | 방산 업무 조건에 맞음 |

## 6. 핵심 요약(TL;DR)

방산 영업은 현재 기획의 직접 타깃이 아닙니다. 이 persona를 억지로 B2C 개인 SaaS에 넣으면 제품 방향이 흐려지고 보안 리스크가 커집니다.

배울 점:

1. 민감 업종은 개인 SaaS에서 제외하거나 별도 제품화
2. Off-the-record 메모에는 등급/권한/감사 로그 필요
3. 긴 영업 주기는 `딜`보다 `사업 단계`로 관리
4. Export와 OCR은 보안 정책이 먼저

현재 서비스는 방산보다 자동차, 보험, 방문판매, 산업재 같은 개인 영업자에게 집중하는 편이 맞습니다.
