## 1. 전체 평가

보험 설계사 관점에서 현재 기획은 **개인 고객 관계 관리 도구**로 가능성이 높습니다. 보험사는 자체 시스템을 제공하지만, 설계사가 실제로 관리해야 하는 가족 관계, 생애 이벤트, 보장 공백, 소개망, 민감한 상담 메모는 개인 노트에 흩어지는 경우가 많습니다.

다만 보험 영업은 금융 정보와 개인정보가 민감합니다. 이 서비스가 보험료 계산, 상품 추천, 계약 자동화까지 하려고 하면 규제와 책임 문제가 커집니다. 반대로 **관계 관리와 리마인더**에 집중하면 실사용 가능성이 있습니다.

### 1.1 적합도 요약

| 기획 요소 | 적합도 | 이유 |
|----------|--------|------|
| 개인용 고객 관리 | 매우 높음 | 설계사 개인 고객 자산 관리와 맞음 |
| Off-the-record 메모 | 매우 높음 | 가족 상황, 건강 이슈, 상담 반응 기록 필요 |
| 필터/검색 | 매우 높음 | 갱신, 만기, 생일, 보장 점검 대상 조회 |
| 모바일 동기화 | 높음 | 고객 방문 전후 확인 필요 |
| Excel Import/Export | 보통 | 기존 고객 목록 이전에 필요 |
| 명함 OCR | 낮음 | 고객은 명함보다 연락처/가족관계가 중요 |
| PDF Export | 낮음 | 공식 보험 문서는 회사 시스템에서 처리 |

## 2. 기능별 실용성 평가

### 2.1 영업 데이터 관리(CRUD)

**실용성**: 매우 높음

보험 설계사는 고객 한 명보다 **가구 단위**로 봐야 합니다.

필요 필드:

| 구분 | 필드 |
|------|------|
| 고객 | 이름, 연락처, 직업, 가족관계, 선호 연락 방식 |
| 가족 | 배우자, 자녀, 부모, 각 가족의 생년월일 |
| 계약 | 보험 종류, 가입일, 갱신일, 만기일, 담당 보험사 |
| 상담 | 관심 보장, 거절 사유, 다음 점검일 |
| 이벤트 | 결혼, 출산, 자녀 입학, 은퇴, 대출, 이직 |
| 소개 | 소개자, 소개받은 사람, 관계 |

**상담 직전 5분 안에 열까?**  
네. "자녀가 몇 살이었지?", "작년에 암보험 얘기하다가 왜 보류됐지?"를 바로 봐야 합니다.

### 2.2 Import/Export

**실용성**: 보통

초기 고객 목록 이전에는 Import가 필요합니다. 하지만 계약 정보는 회사 시스템에서 가져오기 어려울 수 있습니다.

Export는 백업 용도 외에는 조심해야 합니다. 보험 고객 정보는 민감하므로 내려받기 기능에는 보안 안내와 제한이 필요합니다.

### 2.3 필터 시스템

**실용성**: 매우 높음

필요 필터:

- 이번 달 갱신/만기 고객
- 보장 점검 1년 경과 고객
- 자녀 출생/입학 시점 고객
- 최근 상담 후 미계약 고객
- 소개 가능성 높은 고객
- 보험료 부담으로 보류한 고객
- 생일/기념일 고객
- 장기 미연락 고객

현재 기획의 "이번 달 구매 가능성 높은 사람"은 보험에서는 **보장 점검 타이밍이 온 고객**으로 바꿔야 합니다.

### 2.4 명함 OCR

**실용성**: 낮음

법인 단체보험이나 사업자 고객에게는 쓸 수 있지만, 개인 보험 영업에서는 명함보다 가족관계와 생애 이벤트가 중요합니다.

### 2.5 Off-the-record 메모

**실용성**: 매우 높음. 단, 보안 필요.

예시:

- 남편이 보험에 부정적임
- 부모님 병력 때문에 걱정이 큼
- 보험료 20만원 이상은 부담스러워함
- 출산 후 어린이보험 관심 가능성
- 친구 소개는 가능하지만 직접 부탁하면 부담스러워함
- 카톡보다 전화 상담 선호

민감한 건강 정보가 들어갈 수 있으므로, 암호화와 잠금 기능이 필요합니다.

## 3. 기획서가 놓친 보험 영업 페인포인트

### 3.1 가족 단위 관리

보험은 개인이 아니라 가족 포트폴리오로 봐야 합니다. 고객 A의 배우자, 자녀, 부모까지 연결되어야 다음 제안이 가능합니다.

### 3.2 생애 이벤트 리마인더

보험 영업의 좋은 타이밍은 생애 이벤트입니다.

- 결혼
- 출산
- 자녀 입학
- 주택 구입
- 대출
- 이직
- 은퇴
- 부모 간병

현재 기획의 알림이 Phase 4로 밀려 있는데, 보험 persona에서는 Phase 2로 올려야 합니다.

### 3.3 계약 만기/갱신/보장 점검

보험은 계약 후 관리가 중요합니다. 만기, 갱신, 보장 점검일을 놓치면 고객 신뢰가 떨어집니다.

### 3.4 소개망 관리

보험 영업은 소개가 핵심입니다. 누가 누구를 소개했는지, 감사 인사를 했는지, 소개 요청을 언제 할지 관리해야 합니다.

## 4. 우선순위 제안

### Phase 1

- 고객 카드
- 가족 관계
- 상담 메모
- 빠른 검색
- 다음 상담일
- 사용자별 데이터 분리

### Phase 2

- 계약 만기/갱신 리마인더
- 생애 이벤트 태그
- 소개망 관리
- 보장 점검 예정 필터

### Phase 3

- Excel Import
- 고객별 상담 히스토리
- 민감 메모 잠금

### Phase 4 이후 또는 제외

- 보험료 자동 계산
- 상품 자동 추천
- 가입설계서 자동 생성
- 자동 영업 메시지

## 6. 핵심 요약(TL;DR)

보험 설계사에게 이 서비스는 **가족 단위 관계와 보장 점검 타이밍을 기억하는 도구**가 되어야 합니다.

반드시 보강할 기능:

1. 가족 관계 관리
2. 계약 만기/갱신 리마인더
3. 생애 이벤트 태그
4. 소개망 관리
5. 민감 메모 잠금/보안

보험 상품 추천이나 가입설계서 자동화보다 중요한 것은 **언제 누구에게 다시 연락해야 하는지 잊지 않는 것**입니다.
