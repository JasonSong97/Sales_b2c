## 1. 전체 평가

제약 영업 관점에서 현재 기획은 **개인 메모 도구로는 일부 가능하지만, 제품 방향을 매우 조심해야 합니다.** 제약 MR은 의사, 약사, 병원 관계자와의 방문 이력과 반응을 관리해야 하지만, 규정과 컴플라이언스가 강합니다.

현재 기획의 `Off-the-record 메모`는 제약 영업에서는 특히 위험할 수 있습니다. 표현을 바꾸고, 저장 가능한 정보와 저장하면 안 되는 정보를 명확히 해야 합니다.

### 1.1 적합도 요약

| 기획 요소 | 적합도 | 이유 |
|----------|--------|------|
| 방문 이력 관리 | 높음 | 병의원 방문 후 다음 액션 관리 필요 |
| 모바일 조회 | 높음 | 병원 방문 직전 빠른 확인 필요 |
| 필터/검색 | 높음 | 미방문 병원, 제품 관심, 다음 방문 대상 조회 |
| Off-the-record 메모 | 제한적 | 규정 위반 소지가 있어 주의 필요 |
| 명함 OCR | 보통 | 의사/약사 명함 등록은 가능하지만 민감 |
| Excel Import/Export | 보통 | 기존 방문처 이전에는 필요 |
| PDF Export | 낮음 | 공식 보고는 회사 시스템 중심 |

### 1.2 핵심 전제

제약 영업은 회사 CRM, VEEVA류 시스템, 내부 보고 체계가 강합니다. 이 서비스가 그 시스템을 대체할 수는 없습니다. 개인 SaaS라면 **방문 준비용 개인 노트** 정도로 포지셔닝해야 합니다.

## 2. 기능별 실용성 평가

### 2.1 영업 데이터 관리(CRUD)

**실용성**: 보통에서 높음

제약 영업의 기본 단위는 병원/의원, 진료과, 의료진, 제품입니다.

필요 필드:

| 구분 | 필드 |
|------|------|
| 기관 | 병원명, 지역, 진료과, 규모, 방문 가능 시간 |
| 인물 | 의사, 약사, 간호사, 원무 담당, 영향력 |
| 제품 | 관심 제품, 처방 반응, 경쟁 제품 |
| 방문 | 방문일, 전달 내용, 반응, 다음 방문 예정 |
| 제한 | 제공 물품, 학술자료 전달 여부, 컴플라이언스 메모 |

**병원 방문 직전 5분 안에 열까?**  
네. "이 선생님은 어떤 자료에 반응했지?", "지난번에 어떤 질문을 하셨지?"를 확인해야 합니다. 다만 회사가 허용한 정보만 저장해야 합니다.

### 2.2 Import/Export

**실용성**: 제한적

기존 병의원 목록 이전에는 Import가 유용합니다. 하지만 Export는 회사 정책과 충돌할 수 있습니다.

권장:

- Export 기본 비활성 또는 제한
- 민감 정보 경고
- 데이터 삭제/계정 탈퇴 정책 명확화

### 2.3 필터 시스템

**실용성**: 높음

필요 필터:

- 최근 30일 미방문 병원
- 특정 제품 관심 의료진
- 경쟁 제품 사용 병원
- 학술자료 요청 후 미전달
- 다음 방문 예정
- 방문 가능 시간대
- 반응 긍정/중립/부정

현재 기획의 "구매 가능성 높은 사람"은 제약에서는 맞지 않습니다. **제품 관심도와 방문 우선순위**로 바꿔야 합니다.

### 2.4 명함 OCR

**실용성**: 보통

명함 OCR은 신규 병원 관계자 등록에 유용합니다. 다만 OCR 처리 서버와 개인정보 처리 동의가 문제될 수 있습니다. 외부 OCR을 쓰면 회사 정책상 제한될 수 있습니다.

### 2.5 Off-the-record 메모

**실용성**: 제한적

제약에서는 "비공식 메모"라는 표현이 좋지 않습니다. 개인 성향이나 불필요한 민감 정보는 저장하지 않는 쪽이 맞습니다.

허용 가능한 메모 예시:

- 선호 자료 형식
- 관심 제품군
- 지난 방문 질문
- 다음 방문 요청 자료
- 방문 가능 시간

위험한 메모 예시:

- 처방 유도성 표현
- 금품/편의 제공 관련 내용
- 민감한 개인 사생활
- 내부 규정 위반 소지가 있는 약속

## 3. 기획서가 놓친 제약 영업 페인포인트

### 3.1 컴플라이언스 가드레일

제약 영업용으로 쓰려면 입력 단계에서 주의 문구와 제한이 필요합니다.

예:

- 금품/리베이트 관련 메모 금지
- 환자 개인정보 저장 금지
- 처방 대가성 표현 금지
- 회사 정책 확인 안내

### 3.2 방문 가능 시간과 동선

의료진은 방문 가능한 시간이 제한적입니다. 병원별 방문 가능 요일/시간과 대기 패턴을 저장해야 합니다.

### 3.3 제품별 반응 기록

단순 고객 메모보다 제품별 반응이 중요합니다.

필드:

```
product_feedback
  - institution_id
  - doctor_id
  - product_name
  - reaction
  - question
  - next_material
```

### 3.4 회사 CRM과의 관계

제약 MR은 이미 회사 CRM을 씁니다. 이 제품은 대체가 아니라 방문 직전 기억 보조로 한정해야 합니다.

## 4. 우선순위 제안

### Phase 1

- 병의원 카드
- 의료진 카드
- 방문 이력
- 제품 관심 메모
- 방문 가능 시간

### Phase 2

- 다음 방문 예정일
- 요청 자료 체크리스트
- 제품별 반응 기록
- 컴플라이언스 금지어/주의 UX

### Phase 3

- 명함 OCR
- 방문 동선 메모
- Excel Import

### Phase 4 이후 또는 제외

- 처방 예측
- 자동 판촉 메시지
- 리베이트성 기록
- 회사 CRM 대체
- 환자 정보 저장

## 6. 핵심 요약(TL;DR)

제약 영업에게 이 서비스는 **회사 CRM을 대체하는 도구가 아니라 병원 방문 전 기억을 정리하는 개인 노트**여야 합니다.

반드시 보강할 기능:

1. 병의원/의료진/제품 단위 데이터 구조
2. 방문 가능 시간 관리
3. 제품별 반응과 요청 자료 기록
4. 컴플라이언스 가드레일
5. 환자 정보와 민감 정보 저장 금지

제약 persona는 적합성이 낮지는 않지만, 규정 리스크 때문에 기능 범위를 좁히는 것이 필수입니다.
