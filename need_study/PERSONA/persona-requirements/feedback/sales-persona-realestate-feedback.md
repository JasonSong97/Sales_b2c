## 1. 전체 평가

부동산 중개 영업 관점에서 현재 기획은 **고객 메모와 후속 연락 관리 도구**로 가능성이 있습니다. 하지만 현재 구조는 일반 영업 CRM에 가깝고, 부동산 중개에서 가장 중요한 **매물**, **고객 조건**, **매칭**, **방문/임장 이력**, **계약 단계**가 부족합니다.

부동산은 고객만 관리해서는 안 됩니다. 고객 조건과 매물을 연결하는 것이 핵심입니다.

### 1.1 적합도 요약

| 기획 요소 | 적합도 | 이유 |
|----------|--------|------|
| 고객 관리 | 높음 | 매수/임차 고객 조건 관리 필요 |
| 필터/검색 | 매우 높음 | 예산, 지역, 평형, 입주 시점으로 찾아야 함 |
| 모바일 동기화 | 매우 높음 | 현장 방문 중 확인 필요 |
| Off-the-record 메모 | 높음 | 실제 결정권자, 가격 민감도, 급한 사정 기록 |
| 명함 OCR | 낮음 | 일반 고객보다 연락처와 조건이 중요 |
| Excel Import/Export | 보통 | 매물/고객 목록 이전에 필요 |
| PDF Export | 보통 | 고객용 매물 제안서로 활용 가능 |

## 2. 기능별 실용성 평가

### 2.1 영업 데이터 관리(CRUD)

**실용성**: 높음. 단, 매물 엔티티 필요.

필수 데이터:

| 구분 | 필드 |
|------|------|
| 고객 | 이름, 연락처, 매수/전세/월세, 예산, 지역, 입주 시점 |
| 조건 | 평형, 방 수, 주차, 학군, 역세권, 반려동물, 대출 가능성 |
| 매물 | 주소, 가격, 면적, 층, 방향, 관리비, 입주 가능일 |
| 매칭 | 고객별 추천 매물, 반응, 탈락 사유 |
| 방문 | 임장일, 동행자, 반응, 다음 제안 |
| 계약 | 가계약, 계약, 잔금, 입주, 서류 체크 |

**현장 방문 직후 5분 안에 열까?**  
네. 고객이 "아까 본 집 말고 역에서 더 가까운 건 없나요?"라고 하면 바로 조건과 매물을 다시 봐야 합니다.

### 2.2 Import/Export

**실용성**: 보통

매물 목록이나 기존 고객 목록을 가져오는 데 Import가 필요합니다. Export는 고객에게 제안할 매물 리스트, 사무실 내부 공유용으로 쓸 수 있습니다.

PDF Export는 부동산에서는 다른 업종보다 의미가 있습니다. 다만 공식 계약 문서 자동 생성은 제외해야 합니다.

### 2.3 필터 시스템

**실용성**: 매우 높음

필요 필터:

- 예산 범위
- 지역/역세권
- 입주 가능일
- 전세/월세/매매
- 반려동물 가능
- 주차 가능
- 학군
- 급매
- 고객 반응 좋았던 매물
- 계약 임박 고객

현재 기획의 "이번 달 구매 가능성 높은 사람"은 부동산에서는 **조건이 맞는 매물이 나온 고객**입니다.

### 2.4 명함 OCR

**실용성**: 낮음

부동산 중개에서는 명함보다 고객 조건과 매물 사진이 중요합니다. 명함 OCR은 법인 임대차나 투자자 네트워크 관리에는 일부 유용하지만 핵심은 아닙니다.

### 2.5 Off-the-record 메모

**실용성**: 높음

예시:

- 실제 결정권자는 배우자
- 전세대출 승인 여부가 불확실함
- 아이 학교 때문에 특정 단지 선호
- 집주인과 가격 협상 여지 있음
- 고양이 때문에 반려동물 가능 필수
- 급하지만 가격에는 민감함
- 이전 중개사에게 불만이 있었음

단, 법적 분쟁 소지가 있는 표현은 조심해야 합니다.

## 3. 기획서가 놓친 부동산 중개 페인포인트

### 3.1 고객 조건과 매물 매칭

부동산 중개의 핵심은 고객 조건과 매물의 빠른 매칭입니다.

필요 기능:

- 고객 조건 저장
- 매물 조건 저장
- 조건에 맞는 매물 필터
- 고객별 추천 매물 목록
- 탈락 사유 기록

### 3.2 매물 사진과 현장 메모

매물은 사진이 중요합니다. 사진 없이 텍스트만 있으면 현장에서 쓰기 어렵습니다.

필요:

- 매물 사진 첨부
- 장점/단점 메모
- 채광, 소음, 냄새, 관리 상태
- 집주인 협상 여지

### 3.3 계약 단계 체크리스트

계약은 실수가 나면 큰 문제가 됩니다. 자동 계약서 생성은 위험하지만 체크리스트는 필요합니다.

체크리스트:

- 등기 확인
- 권리관계 확인
- 대출 가능성
- 가계약금
- 계약일
- 잔금일
- 입주일

### 3.4 임장/방문 이력

고객이 어떤 매물을 봤고 왜 탈락했는지 잊으면 신뢰를 잃습니다. 임장 이력은 고객과 매물 양쪽에 연결되어야 합니다.

## 4. 우선순위 제안

### Phase 1

- 고객 카드
- 고객 조건
- 매물 카드
- 매물 검색/필터
- 방문/임장 메모

### Phase 2

- 고객-매물 매칭
- 매물 사진 첨부
- 탈락 사유 기록
- 계약 단계 체크리스트

### Phase 3

- PDF 매물 제안서
- Excel Import
- 지도 앱 연결

### Phase 4 이후 또는 제외

- 계약서 자동 생성
- 법률 판단 자동화
- 시세 예측
- 자동 광고 등록

## 5. 사용 의향

| 조건 | 사용 의향 | 이유 |
|------|----------|------|
| 현재 기획 그대로 | 조건부 사용 | 고객 메모로는 가능하지만 매물 관리가 없음 |
| 고객 조건/매물 매칭 추가 | 적극 사용 | 중개 업무 핵심과 맞음 |
| 계약서 자동화 중심 | 위험 | 법적 책임 문제 |
| 사진/지도 연동 제공 | 높음 | 현장 사용성이 큼 |

## 6. 핵심 요약(TL;DR)

부동산 중개 영업에게 이 서비스는 **고객 조건과 매물을 연결하는 현장 CRM**이어야 합니다.

반드시 보강할 기능:

1. 고객 조건 관리
2. 매물 카드와 사진 첨부
3. 고객-매물 매칭
4. 임장 이력과 탈락 사유
5. 계약 단계 체크리스트

현재 기획은 고객 관리만으로는 부족합니다. `properties` 또는 `listings` 엔티티를 추가해야 부동산 persona가 실제로 사용할 수 있습니다.
