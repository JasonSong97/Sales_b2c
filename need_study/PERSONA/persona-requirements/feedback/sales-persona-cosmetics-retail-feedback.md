## 1. 전체 평가

현재 기획은 화장품 리테일 영업에게 **매장별 현장 이슈 관리 도구**로 방향을 잡으면 쓸 수 있습니다. 하지만 현재 문서의 고객/거래처/딜 구조는 매장 담당 영업의 실제 업무와 조금 다릅니다.

이 persona에게 중요한 것은 사람보다 **매장**, **담당자**, **진열 상태**, **재고/품절**, **프로모션 집행**, **경쟁사 매대**, **방문 리포트**입니다.

### 1.1 잘 맞는 부분

| 기획 요소 | 적합도 | 이유 |
|----------|--------|------|
| 모바일/PC 동기화 | 매우 높음 | 매장 방문 중 찍고, 사무실에서 정리해야 함 |
| 필터/검색 | 매우 높음 | 지역, 채널, 매장 등급, 품절, 프로모션 기준 조회가 필요 |
| Off-the-record 메모 | 높음 | 점장 성향, 담당자 교체, 매대 협상 내용은 중요 |
| Excel Import/Export | 높음 | 본사 보고와 매장 리스트 관리에 필요 |
| 명함 OCR | 보통 | 점장/바이어 명함 등록에는 유용 |
| PDF Export | 보통 | 주간 방문 리포트로 쓰일 수 있음 |

### 1.2 핵심 문제

현재 기획은 `사람 중심 CRM`에 가깝습니다. 리테일 영업은 **매장 단위 CRM**이어야 합니다. 고객 카드보다 매장 카드가 먼저 나와야 합니다.

## 2. 기능별 실용성 평가

### 2.1 영업 데이터 관리(CRUD)

**실용성**: 높음. 단, 데이터 모델 수정 필요.

필수 엔티티:

| 엔티티 | 필드 예시 |
|--------|-----------|
| stores | 매장명, 채널, 지역, 등급, 주소, 담당자, 방문 주기 |
| contacts | 점장, 부점장, 뷰티 어드바이저, 본사 바이어 |
| visits | 방문일, 진열 상태, 품절, 요청사항, 사진 |
| promotions | 행사 기간, 행사 품목, 집행 상태, 매대 위치 |
| competitors | 경쟁 브랜드, 가격, 프로모션, 진열 사진 |

**매장 방문 직후 5분 안에 열까?**  
네. 사진을 찍고, 품절 상품과 점장 요청사항을 바로 기록해야 합니다.

### 2.2 Import/Export

**실용성**: 매우 높음

매장 리스트는 보통 엑셀로 내려옵니다. 초기 Import는 필수에 가깝습니다. Export는 주간 리포트, 월간 방문 실적, 품절 현황 공유에 필요합니다.

단, PDF Export보다 Excel Export가 훨씬 중요합니다. 현업 보고는 대부분 표 형태로 돌아갑니다.

### 2.3 필터 시스템

**실용성**: 매우 높음

필요한 필터:

- 이번 주 방문 예정 매장
- 품절 발생 매장
- 프로모션 미집행 매장
- A등급 매장
- 점장 교체 매장
- 경쟁사 행사 강한 매장
- 진열 사진 미등록 매장
- 클레임 발생 매장

현재 기획의 "이번 달 구매 가능성 높은 사람"은 이 업종에서는 맞지 않습니다. 대신 **이번 주 조치가 필요한 매장**이 핵심입니다.

### 2.4 명함 OCR

**실용성**: 보통

점장, 본사 바이어, 채널 담당자 명함 등록에는 유용합니다. 하지만 매일 쓰는 기능은 아닙니다.

더 중요한 것은:

- 담당자 변경 이력
- 매장별 관계 온도
- 점장 선호 커뮤니케이션 방식
- 요청사항 처리 상태

### 2.5 Off-the-record 메모

**실용성**: 높음

리테일 영업에서 공식 보고서에 쓰기 어려운 정보가 많습니다.

예시:

- 점장이 특정 브랜드에 호의적임
- 본사 지침보다 현장 재량이 큰 매장
- 행사 매대 협상 가능성
- 경쟁사 판촉 직원이 강함
- 신입 직원이라 교육 필요
- 매장 동선상 우리 브랜드 위치가 불리함

이 메모는 매장별로 누적되어야 합니다.

## 3. 기획서가 놓친 리테일 영업 페인포인트

### 3.1 매장 사진 관리

화장품 리테일 영업은 사진이 증거입니다. 진열 전/후, 경쟁사 매대, 품절, POP 설치 상태를 사진으로 남깁니다.

필요 기능:

- 방문 기록에 사진 첨부
- 사진별 태그: 진열, 품절, 경쟁사, 행사, 클레임
- 매장별 사진 히스토리
- 주간 보고서에 사진 포함

### 3.2 품절과 재고 이슈

매장 매출을 놓치는 가장 현실적인 문제는 품절입니다.

추가 필드:

```
stock_issues
  - store_id
  - product_name
  - issue_type: 품절/과재고/입고지연
  - reported_date
  - resolved_date
  - memo
```

### 3.3 방문 리포트 자동 정리

영업자는 매장을 돌고 나서 보고서 정리에 시간을 많이 씁니다. 기획서의 PDF Export를 단순 목록 출력이 아니라 **방문 리포트 생성**으로 바꾸면 가치가 커집니다.

리포트 항목:

- 방문 매장
- 핵심 이슈
- 품절 상품
- 경쟁사 동향
- 사진
- 다음 조치

### 3.4 교육/매대 요청 관리

매장 직원 교육, 테스터 교체, POP 요청, 행사 물량 요청 같은 액션 아이템을 추적해야 합니다. 현재 `notes`만으로는 부족합니다.

## 4. 우선순위 제안

### Phase 1

- 매장 카드
- 담당자 카드
- 방문 기록
- 사진 첨부
- 매장 검색/필터
- Excel Import

### Phase 2

- 품절/재고 이슈 관리
- 경쟁사 동향 기록
- 방문 리포트 Export
- 다음 조치 due date

### Phase 3

- 명함 OCR
- 사진 태그 자동 분류
- 매장 등급별 대시보드

### Phase 4 이후 또는 제외

- 개인 고객 CRM
- 자동 주문
- 본사 ERP 대체
- 가격/프로모션 자동 결정

## 5. 사용 의향

| 조건 | 사용 의향 | 이유 |
|------|----------|------|
| 현재 기획 그대로 | 낮음 | 매장 중심이 아니라 사람 중심이라 현장과 다름 |
| 매장/사진/방문 리포트 중심 | 적극 사용 | 매일 쓰는 업무와 직접 연결 |
| Excel 보고서 연동 | 높음 | 본사 보고 부담이 줄어듦 |
| 주문/재고 시스템 대체 | 어려움 | 회사 시스템 영역 |

## 6. 핵심 요약(TL;DR)

화장품 리테일 영업에게 필요한 것은 **고객 CRM이 아니라 매장 운영 CRM**입니다.

반드시 보강할 기능:

1. 매장 카드 중심 데이터 모델
2. 방문 기록과 사진 첨부
3. 품절/재고 이슈 추적
4. 경쟁사 매대/프로모션 기록
5. 주간 방문 리포트 Export

현재 기획을 이 업종에 맞추려면 `customers`보다 `stores`가 첫 번째 엔티티가 되어야 합니다.
