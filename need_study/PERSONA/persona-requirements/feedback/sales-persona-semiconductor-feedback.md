## 1. 전체 평가 — 반도체 대형 어카운트 영업 관점

### 1.1 타겟 시장 문제 (가장 큰 이슈)

**"B2C 개인 구독"이 반도체 영업 환경과 맞지 않습니다.**

- 14년간 개인 돈으로 업무 앱 구독한 적 없음
- 회사 데이터(어카운트 정보, 딜 금액, 경쟁사 동향)를 개인 앱에 넣으면 **컴플라이언스 위반**
- 회사 결재 필수. **보안팀 승인 없이는 설치조차 불가**
- 회사 지급 노트북·폰만 사용. 개인 디바이스에 회사 데이터 저장 금지

**결론**: 반도체 영업이 이 앱을 쓰려면 **B2B 전환**(회사 도입)이 전제 조건.

### 1.2 데이터 모델 미스매치

**현재 기획서**: `거래처(Customer) = 사람` 중심

**반도체 영업 실제**:

```
어카운트(Account) = 회사 중심
  ├─ S사 반도체 → 의사결정자 30명 (구매팀·공정팀·장비팀·품질팀)
  ├─ H사 → 의사결정자 15명
  └─ T사 → 의사결정자 10명
      ├─ 딜 A: 신규 장비 도입 (2년차, $50M)
      ├─ 딜 B: 소재 인증 (8개월차, $5M/년)
      └─ 딜 C: 추가 발주 협상 (진행 중, $20M)
```

- 어카운트 1~5개를 관리하지만, 각 어카운트 내 관계자가 10~30명
- 한 어카운트에서 여러 딜이 동시에 진행 (POC, 인증, 양산, 추가 발주)
- **"사람" 중심이 아니라 "어카운트→조직도→딜" 계층 구조**가 필요

## 2. 기능별 실용성 평가 ("팹 옆 회의실에서 정장 입고 쓸까?")

### 2.1 영업 데이터 관리 (CRUD)

**실용성**: ★★☆☆☆

- 노트북→모바일 동기화: 개념은 좋지만, **회사 보안 정책상 개인 앱 동기화 불가**
- 회사 지급 폰에 설치 가능해야 하는데, MDM(Mobile Device Management) 호환 필수
- 오프라인 조회(IndexedDB): 팹 내부는 휴대폰 반입 금지인 경우 많음

**팹 옆 회의실에서 쓸까?** → **NO**. 회사 노트북에서 Salesforce/자체 CRM 사용.

### 2.2 엑셀 Import/Export

**실용성**: ★★★★☆

- 본사(해외)에 영문 주간 보고서 제출할 때 **엑셀 Export는 필수**
- BUT: 보고서 양식이 본사 지정 포맷이라 단순 Export로는 부족
- **진짜 필요한 것**: 본사 양식에 맞춘 보고서 자동 생성 (매주 2시간 절약 가능)

**팹 옆 회의실에서 쓸까?** → **NO**. 호텔 방에서 주간 보고서 쓸 때나 가능.

### 2.3 필터링 시스템

**실용성**: ★★★☆☆ (현재 기획 기준)

**문제**: "이번 달 구매 가능성 높은 사람" 탭은 반도체 영업과 맞지 않음.
- 반도체 영업 사이클은 **6개월~3년**. "이번 달" 개념 무의미
- 단가가 수십억~수백억이라 probability 0~100%도 너무 단순

**필요한 필터**:
| 필터 | 설명 |
|------|------|
| 영업 단계별 | Qualification → POC → 인증 → 양산 승인 → 추가 발주 |
| 어카운트별 | S사 / H사 / T사 |
| 기술 이슈 상태별 | Open / In Progress / Blocked / Resolved |
| 경쟁사 동향별 | A사 진입 시도 / B사 가격 공세 / C사 기술 우위 |
| Forecast 시점별 | 이번 분기 / 다음 분기 / 내년 |

**팹 옆 회의실에서 쓸까?** → 필터가 위와 같이 개선되면 **YES**.

### 2.4 명함 OCR

**실용성**: ★☆☆☆☆

- 연간 명함 5~10장 수준. 이미 Salesforce에 전부 등록되어 있음
- 반도체 업계는 보안 때문에 명함 교환 자체가 드문 경우도 있음
- **진짜 필요한 것**: 어카운트 내 의사결정자 조직도 관리 + LinkedIn 연동

**팹 옆 회의실에서 쓸까?** → **NO**. 명함은 미팅 후 호텔에서 Salesforce에 수동 입력.

### 2.5 Off-the-record 메모

**실용성**: ★★★★★

- **유일하게 100% 공감되는 기능**
- "S사 X공정 담당 김 수석은 우리 장비에 호의적. 구매팀 박 팀장은 경쟁사 A와 관계 있음"
- "H사 내부적으로 차세대 공정 투자 검토 중 (비공식)"
- 이런 정보는 Salesforce에 절대 못 넣음. 감사 시 문제 됨

**팹 옆 회의실에서 쓸까?** → **YES**. 미팅 직후 화장실에서 메모 입력.
**단, 보안 인증된 앱이어야만 가능.**

## 3. 보안 관련 우려사항 (P0 — 최우선)

기획서에 보안 관련 내용이 **거의 없습니다**. 반도체 영업 환경에서 이것은 치명적입니다.

### 3.1 필수 보안 요건

| 항목 | 필요 수준 | 현재 기획서 |
|------|----------|------------|
| ISO 27001 / ISMS 인증 | 필수 | ❌ 미언급 |
| 데이터 센터 위치 | 한국 내 서버 필수 (AWS Seoul) | ❌ 미언급 |
| SSO 지원 | Azure AD / Okta 연동 필수 | ❌ 미언급 |
| 감사 로그 (Audit Log) | 누가 언제 무슨 데이터 접근했는지 추적 | ❌ 미언급 |
| 데이터 암호화 | TLS 1.3 + AES-256 (저장 시) | ❌ 미언급 |
| MDM 호환 | 회사 지급 디바이스 관리 시스템 호환 | ❌ 미언급 |
| 데이터 삭제 정책 | 퇴사 시 데이터 완전 삭제 보장 | ❌ 미언급 |
| 2FA / MFA | 다중 인증 필수 | ❌ 미언급 |

**이 항목들이 충족되지 않으면 반도체 회사 보안팀 승인 자체가 불가능합니다.**

### 3.2 보안 등급에 따른 데이터 분류

반도체 영업 데이터는 다음과 같이 분류됩니다:

- **극비**: 고객사 차세대 공정 정보, 가격 협상 내역 → 외부 앱 저장 금지
- **비밀**: 어카운트 조직도, 경쟁사 동향 → 보안 인증된 앱만 가능
- **일반**: 연락처, 미팅 일정 → 비교적 자유

현재 기획서는 이런 **데이터 분류 체계가 없음**.

## 4. 반도체 영업에서 진짜 필요한 기능 (현재 기획서에 없음)

### 4.1 본사 영문 주간 보고서 자동 초안

- 매주 월요일 본사(미국/일본/유럽)에 영문 주간 보고서 제출
- 매주 2시간 이상 소요
- 어카운트별 딜 현황 + 기술 이슈 + 경쟁사 동향을 정해진 양식으로 작성
- **이것만 자동화해도 연간 100시간 절약**

### 4.2 기술 이슈 트래킹

```
기술 이슈:
  ├─ 어카운트: S사
  ├─ 장비/소재: 식각 장비 Model X
  ├─ 이슈: Etch rate 불균일 (center vs edge 5% 차이)
  ├─ 상태: FAE 검토 중
  ├─ 담당 FAE: 김엔지니어
  ├─ 고객사 담당: 이 수석 (공정팀)
  └─ 마감: 2026-06-15 (고객사 요청)
```

- FAE(Field Application Engineer)와 공유 필수
- 기술 이슈 해결 여부가 딜 성사를 좌우

### 4.3 경쟁사 동향 구조화 입력

```
경쟁사 동향:
  ├─ 어카운트: H사
  ├─ 경쟁사: A사 (일본)
  ├─ 동향: 차세대 공정용 신제품 데모 예정 (Q3)
  ├─ 위협 수준: High
  ├─ 대응 전략: Q2 내 POC 완료 필요
  └─ 소스: 고객사 김 팀장 (비공식)
```

### 4.4 다국어 UI

- 영어·일본어·중국어 지원 필수
- 본사 글로벌 감사 시 영문 UI 필요
- 해외 출장(중동·동남아·동유럽) 시 영어 UI 사용

## 5. 우선순위 제안 (반도체 영업 관점)

### 현재 기획서 Phase vs 반도체 영업 관점 조정

| 현재 Phase | 반도체 관점 우선순위 | 이유 |
|-----------|-------------------|------|
| Phase 0: 인증·레이아웃 | ✅ 유지 + **SSO/MFA 추가** | 보안 없으면 시작 불가 |
| Phase 1: CRUD·필터·Export | ⚠️ 필터 재설계 필요 | 영업 사이클 6개월~3년 반영 |
| Phase 2: Import·탭·PDF | ⚠️ 보고서 자동화 추가 | PDF보다 영문 주간 보고서가 우선 |
| Phase 3: 명함 OCR | ❌ 우선순위 대폭 하향 | 연간 명함 5~10장. 의미 없음 |
| — | 🆕 기술 이슈 트래킹 추가 | Phase 2로 상향 |
| — | 🆕 경쟁사 동향 관리 추가 | Phase 2로 상향 |
| — | 🆕 어카운트 조직도 관리 추가 | Phase 1로 상향 |

### 반도체 영업 전용 로드맵 (제안)

**Phase 0**: 보안 기반
- ISO 27001 준비 / SSO(Azure AD) / MFA / 감사 로그 / 한국 서버

**Phase 1**: 어카운트 중심 데이터 모델
- 어카운트(회사) → 조직도(사람) → 딜 계층 구조
- 영업 단계: Qualification → POC → 인증 → 양산 → 추가 발주
- Off-the-record 메모 (보안 등급 적용)

**Phase 2**: 영업 지원
- 기술 이슈 트래킹 (FAE 공유)
- 경쟁사 동향 관리
- 본사 영문 주간 보고서 자동 초안
- 엑셀 Export (본사 양식 호환)

**Phase 3**: 고도화
- 다국어 UI
- Salesforce 연동 (읽기 전용 동기화)
- 어카운트별 타임라인
- 알림 (인증 마감일, 고객사 요청 마감일)

## 6. 사용 확률 평가

### 현재 기획서 그대로 → 사용 확률: 0%

**이유**:
- 보안 인증 없음 → 회사 보안팀 승인 불가
- B2C 개인 구독 → 회사 비용 처리 불가

3. 어카운트 중심 데이터 모델
4. 영업 사이클 세분화 (6개월~3년)
5. 기술 이슈 트래킹
6. 경쟁사 동향 관리
7. 영문 주간 보고서 자동 초안
8. 다국어 UI

### 이 앱의 현재 포지션

| 항목 | 평가 |
|------|------|
| B2C 영업(보험·자동차) 타겟 | ✅ 적합 |

### 반도체 영업의 본질적 차이

- **영업 사이클**: 수개월~수년 (B2C는 수일~수주)
- **딜 규모**: 수십억~수백억 (B2C는 수만~수천만)
- **의사결정 구조**: 다층 (구매팀·공정팀·장비팀·품질팀·경영진)
- **보안 수준**: 극도로 높음 (외부 SaaS 사용 자체가 허들)
- **FAE 협업**: 기술 영업이므로 엔지니어와 팀 단위 움직임

**Off-the-record 메모** — 단, 보안 인증된 앱이어야만 사용 가능.
