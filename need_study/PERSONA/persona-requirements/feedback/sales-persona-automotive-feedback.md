# 자동차 영업 관점 피드백 - 영업 관리 프로그램 서비스 기획서

> **피드백 작성자**: 송영훈 (13년차 자동차 영업 차장, 장기 고객/소개망 관리)
> **작성일**: 2026-05-22
> **대상 문서**: `01_서비스_기획서.md`

## 1. 전체 평가

현재 기획은 자동차 영업에서 **개인 고객 장기관리 도구**로는 가능성이 있습니다. 특히 회사 시스템에 남기기 어려운 고객 성향, 가족 차량, 재구매 시점, 소개 관계를 개인이 따로 관리한다는 방향은 현장과 잘 맞습니다.

다만 현재 문서의 `영업 데이터 CRUD`, `거래처`, `딜`이라는 표현은 자동차 영업자에게 너무 B2B처럼 들립니다. 자동차 영업의 핵심은 거래처가 아니라 **고객 카드**, **차량 카드**, **출고 이력**, **재구매 주기**, **소개망**입니다.

### 1.1 가장 잘 맞는 부분

| 기획 요소 | 자동차 영업 적합도 | 이유 |
|----------|------------------|------|
| 개인용 영업 관리 | 매우 높음 | 본사 DMS/CRM은 견적·계약·출고 중심이고 개인 메모는 빈약함 |
| 모바일/PC 동기화 | 높음 | 전시장, 시승, 출고장, 이동 중에 바로 확인해야 함 |
| Off-the-record 메모 | 매우 높음 | 가족 구성, 선호 색상, 예산 감각, 말투 같은 정보는 회사 시스템에 넣기 어려움 |
| 필터/검색 | 높음 | 재구매 시점, 차종, 지역, 소개자 기준으로 찾는 일이 잦음 |
| 명함 OCR | 보통 | 법인 고객/사업자 고객에는 유용하지만 개인 고객은 명함보다 휴대폰·카톡이 중심 |
| Excel Import/Export | 보통 | 초기 고객 목록 이전에는 필요하지만 매일 쓰는 기능은 아님 |
| PDF Export | 낮음 | 자동차 영업자는 고객 관리 목록을 PDF로 뽑는 일이 많지 않음 |

### 1.2 가장 위험한 오해

자동차 영업자는 견적 자동화나 할인 추천을 원하지 않습니다. 가격, 프로모션, 금융 조건은 본사·캐피탈·재고·월별 정책에 따라 달라지고 민감합니다. 이 서비스가 그 영역까지 들어가면 오히려 신뢰를 잃습니다.

이 제품은 **본사 시스템 대체재**가 아니라 **내 고객을 오래 기억하게 해주는 개인 보조 도구**로 잡아야 합니다.

## 2. 기능별 실용성 평가

### 2.1 영업 데이터 관리(CRUD)

**실용성**: 매우 높음

자동차 영업에서 CRUD의 단위는 `고객`, `차량`, `상담`, `출고`, `소개`로 나뉘어야 합니다. 현재 기획처럼 고객과 딜만 있으면 출고 이후 관리가 약합니다.

필수로 들어가야 할 필드:

| 구분 | 필요한 필드 |
|------|-------------|
| 고객 | 이름, 연락처, 가족 구성, 거주 지역, 직업, 선호 연락 채널 |
| 차량 | 현재 차량, 관심 차종, 선호 색상, 옵션, 예산, 구매 방식 |
| 출고 | 출고일, 계약일, 차량번호, 금융 조건 요약, 인도 장소 |
| 재구매 | 4년/5년/6년 후 교체 가능성, 리스/할부 만기, 가족 차량 교체 가능성 |
| 소개 | 소개자, 소개받은 사람, 소개 보답 여부 |

**전시장 상담 직후 5분 안에 열까?**  
네. 특히 갑자기 재방문한 고객이 "저번에 봤던 차 있죠?"라고 하면 바로 고객 카드를 열어야 합니다.

### 2.2 Excel Import/Export

**실용성**: 보통

초기 이전에는 필요합니다. 기존 엑셀, 휴대폰 연락처, 카카오톡 고객 목록을 한 번에 가져오는 기능은 중요합니다. 하지만 매일 사용하는 핵심 기능은 아닙니다.

Export는 `내 고객 백업`, `이직 전후 개인 고객 정리`, `월간 상담 목록 정리` 정도로 의미가 있습니다. 단, 회사 고객 정보 반출 이슈가 있으므로 약관과 안내가 필요합니다.

### 2.3 필터 시스템

**실용성**: 매우 높음

현재 기획의 "이번 달 구매 가능성 높은 사람"은 자동차 영업에 맞게 바꿔야 합니다.

자동차 영업에 필요한 필터:

| 필터 | 사용 장면 |
|------|----------|
| 출고 후 4년 이상 | 재구매 연락 대상 |
| 리스/할부 만기 6개월 전 | 금융 재상담 대상 |
| 시승 후 미계약 | 후속 연락 대상 |
| 견적 후 7일 이상 미응답 | 재접촉 대상 |
| 가족 차량 보유 | 배우자/자녀 차량 소개 가능성 |
| SUV 선호/전기차 관심 | 신차 출시 때 연락 대상 |
| 소개 가능성 높음 | 감사 인사·이벤트 초대 대상 |

필터 저장 기능은 꼭 필요합니다. "이번 주 연락할 사람", "출고 3개월 케어", "재구매 후보" 같은 고정 리스트가 영업자의 실사용 화면이 됩니다.

### 2.4 명함 OCR과 거래처 정보

**실용성**: 보통

법인 차량, 사업자, 임원 고객, 지점 제휴 고객에는 유용합니다. 하지만 일반 개인 고객은 명함보다 카카오톡, 휴대폰, 전시장 방문 기록이 더 중요합니다.

기획을 자동차에 맞추려면 명함 OCR보다 아래가 더 중요합니다.

- 고객 연락처 빠른 등록
- 카카오톡 프로필명/메모 기반 등록
- 차량번호 또는 차종으로 검색
- 가족 관계와 소개 관계 연결

### 2.5 Off-the-record 메모

**실용성**: 매우 높음

자동차 영업에서 가장 중요한 기능입니다. 회사 시스템에는 "G80 관심" 정도만 들어가지만 실제 영업자는 아래를 기억해야 합니다.

- 배우자가 최종 결정권자
- 아이가 둘이라 SUV 선호
- 이전 딜러 응대에 불만이 있었음
- 전기차는 관심 있지만 충전 때문에 망설임
- 가격보다 출고일을 더 중요하게 봄
- 통화보다 카톡 선호
- 생일/기념일/출고 후 감사 연락 필요

이 메모는 태그와 검색이 되어야 합니다. 단순 자유 메모만 있으면 고객이 500명일 때 다시 찾기 어렵습니다.

### 2.6 인증과 데이터 분리

**실용성**: 필수

개인 고객 정보가 들어가므로 사용자별 데이터 분리는 필수입니다. 특히 자동차 영업자는 이직이 잦고, 회사와 개인 고객 데이터의 경계가 민감합니다.

개인정보 보호 안내, 데이터 내보내기 정책, 계정 삭제 시 데이터 삭제 정책이 명확해야 합니다.

## 3. 기획서가 놓친 자동차 영업 페인포인트

### 3.1 재구매 주기 관리

자동차 영업은 계약 한 번으로 끝나지 않습니다. 진짜 수익은 4~7년 뒤 재구매와 가족·지인 소개에서 나옵니다.

추가 제안:

- 출고일 기준 자동 리마인더
- 할부/리스 만기일 관리
- 보증 만료 시점 관리
- 신차 출시 시 관심 고객 목록 생성
- 기존 고객 가족 차량 교체 가능성 기록

### 3.2 소개망 관리

상위 영업자는 고객이 고객을 데려옵니다. 그런데 소개 관계는 머릿속, 카카오톡, 수첩에 흩어져 있습니다.

필요한 구조:

```
고객 A
  - 소개한 사람: 고객 B, 고객 C
  - 소개받은 사람: 고객 D
  - 소개 보답: 완료/미완료
  - 감사 연락일: 2026-05-22
```

이 기능은 자동차 영업자에게 일반 CRM보다 훨씬 강한 구매 이유가 됩니다.

### 3.3 출고 일정 3중 매칭

출고는 단순 일정이 아닙니다.

- 차량 입고 예정일
- 고객 인도 가능일
- 등록/보험/금융/탁송 준비 상태

현재 기획의 캘린더 보류 판단은 이해하지만, 자동차 영업에서는 최소한 출고 체크리스트는 필요합니다. 완전한 캘린더가 아니더라도 `출고 준비 상태`는 고객 카드 안에 있어야 합니다.

### 3.4 시승 이후 후속 관리

시승 고객은 구매 가능성이 높지만 놓치기 쉽습니다. 시승 후 하루, 3일, 7일 후 연락 리마인더가 있어야 합니다.

시승 기록 필드:

- 시승 차종
- 동승자
- 좋았던 점
- 불만 포인트
- 경쟁 차종
- 다음 연락 예정일

## 4. 우선순위 제안

### Phase 1에 반드시 넣을 것

- 고객 카드
- 차량 관심 정보
- 상담 메모
- 빠른 검색
- 재구매/후속 연락 필터
- 모바일 오프라인 조회

### Phase 2로 올려야 할 것

- 출고일 기반 리마인더
- 시승 후속 관리
- 소개 관계 관리
- 자주 쓰는 필터 저장

### Phase 3에 적합한 것

- 명함 OCR
- Excel Import
- 출고 체크리스트
- 고객 태그 고도화

### Phase 4 이후 또는 제외

- 견적 자동 생성
- 할인 자동 추천
- 본사 DMS 대체
- 자동 카톡 발송
- SNS 자동 포스팅

## 5. 사용 의향

| 조건 | 사용 의향 | 이유 |
|------|----------|------|
| 현재 기획 그대로 | 조건부 사용 | 고객 메모 도구로는 쓸 수 있으나 자동차 특화가 약함 |
| 재구매/소개망/출고 관리 추가 | 적극 사용 | 실제 계약으로 이어질 가능성이 있음 |
| 본사 시스템 연동 필수 | 사용 어려움 | 회사 정책과 보안 이슈가 큼 |
| 견적·할인 자동화 중심 | 거부 | 책임 소재와 정책 오류 위험 |
| 월 3~5만원 | 가능 | 재구매 1건만 잡아도 회수 가능 |

## 6. 핵심 요약(TL;DR)

자동차 영업 persona에게 이 서비스는 **개인 고객 자산을 장기 관리하는 도구**로 포지셔닝해야 합니다.

반드시 보강할 기능:

1. 출고일 기준 재구매 리마인더
2. 가족·지인 소개망 관리
3. 시승 후 미계약 고객 추적
4. 고객별 차량 선호와 구매 조건 기록
5. 출고 후 사후 관리 메모

지금 기획의 방향은 맞지만, `거래처/딜` 중심 언어를 `고객/차량/출고/소개` 중심으로 바꿔야 자동차 영업자가 바로 자기 도구라고 느낍니다.
