## 1. 왜 만드나 — 문제 정의

### 1.1 영업 현장의 7가지 공통 문제

13개 영업 페르소나(자동차·보험·도매·산업재·방판·살롱·H&B·제약·SaaS·반도체·방산·휴대폰·부동산)를 시뮬레이션한 결과, **7개 축**으로 수렴되는 공통 문제가 있다.

| 축 | 핵심 문제 | 강하게 나타나는 페르소나 |
|----|----------|------------------------|
| 1. 정보 분산 | 고객·거래처 정보가 엑셀·카톡·사진·수첩·회사 시스템에 흩어짐 | 도매·산업재·자동차·보험·방판 |
| 2. 기억 복원 | 방문 직전 맥락을 바로 떠올리지 못함 | 자동차·산업재·보험·SaaS·부동산·제약 |
| 3. 후속 누락 | 다음 연락·만기·재구매·잔금을 놓침 | 보험·자동차·휴대폰·방판·산업재·부동산 |
| 4. 관계 구조 | 가족·소개자·결정권자·매장 담당자 관계가 머릿속에만 | 보험·자동차·살롱·SaaS·반도체·방산·제약 |
| 5. 현장 입력 | 모바일 빠른 입력·사진·음성·오프라인 요구 | 도매·방판·H&B·산업재 |
| 6. 정책 변경 추적 | 회사·본사 정책이 주 단위로 바뀜 | 휴대폰·제약·반도체 |
| 7. 보고·KPI 부담 | 주 1~2시간 본사 보고 + 정량 KPI 압박 | H&B·휴대폰·SaaS·반도체 |

### 1.2 기존 솔루션의 한계

| 도구 | 한계 |
|------|------|
| 회사 ERP/CRM (VEEVA·Salesforce·본사 가입 시스템) | 영업자 시각이 아님. 비공식 메모 못 적음. 회사 강제라 대체도 불가 |
| 카톡 1:1 채팅방 | 300+ 채팅방에서 1년 전 대화 검색 거의 불가 |
| 엑셀 / 종이 수첩 / 갤러리 | 외근 중 못 봄. 검색 안 됨. 분실 위험 |
| 일반 CRM (Pipedrive·Hubspot 등) | 영업맨 개인의 비공식 관계 정보·현장 메모를 위한 도구 아님. 무거움 |
| 명함 OCR 단독 도구 (리멤버) | 명함은 잘하지만 후속 액션·재구매 알림이 없음 |

### 1.3 우리가 만드는 이유

> 13개 페르소나에서 공통으로 발견된 단 하나의 진실은 이것이다.
> **"영업 사원은 고객과 거래처를 더 많이 기록하고 싶은 것이 아니라, 흩어진 현장 정보를 필요한 순간에 바로 떠올리고, 다음 행동을 놓치지 않으며, 회사 시스템에 적기 어려운 관계 맥락을 안전하게 관리하고 싶다."**

즉 **현장 기억의 단절과 후속 행동 누락**이 핵심 문제. CRM 부재가 아니다.

| 그룹 | 페르소나 | 타깃 등급 |
|------|---------|---------|
| 일반 외근 (3) | 산업재(P-06) · 도매(P-13) · 보험(P-07) | 1차 베타 |
| 화장품 (3) | 방판(P-02) · 살롱(P-04) · H&B(P-03) | 2차 베타 |
| 고객 접점 多 (3) | 자동차(P-01) · 휴대폰(P-08) · 부동산(P-10) | 2~3차 베타 |
| 고이익 전문 (4) | 제약(P-09) · SaaS(P-11) · 반도체(P-12) · 방산(P-05) | 제한 / 별도 제품 |

### 2.2 1차 베타 타깃 (D-401)

**메인 인터뷰 5명**:
- 산업재 영업 3명 (자동차 부품·기계 부품·건자재 각 1)
- 도매 영업 2명 (식자재·음료/주류 각 1)

**보조 인터뷰 (시간 여유 시)**:
- 방판 1명 — 시니어 UI (NFR-03 P0 격상) 검증
- 보험 1명 — 라이프이벤트·카톡 검색 (FR-11) 검증

**선정 근거** (11번 v2 6장 매트릭스 P0 합계):
- P-06 산업재: 1위 — 정보 분산·비공식 메모·모바일 모두 ★★★★★
- P-13 도매: 4위 — 정보 분산·모바일 ★★★★★ + 한 손 조작 P0 임계점

### 2.3 페르소나 깊이 보기 — 박상호·최정훈

#### 박상호 차장 (P-06 산업재 영업, 40세, 13년차)

- **소속**: 중견 자동차 부품 회사, 영업1팀 차장
- **담당**: 거래처 80곳 (제조사 자재팀·정비공장·도매상)
- **하루**: 차량 외근 4일 + 사무실 1일, 하루 5~7곳 방문, 이동 150~250km
- **도구**: 사내 ERP(PC) · 엑셀 · 카톡 · 네이버 지도 · 종이 수첩 · 명함
- **페인**:
  1. 운전 중 다음 거래처 담당자 정보 못 봄 (갓길 세움)
  2. 거래처 80곳 담당자 가족·취향 다 못 외움
  3. 회사 ERP에 못 적는 비공식 메모 (가족·이혼 등)
  4. 명함 50장 콘솔에 쌓임
- **NO 리스트**: ERP 자동 연동 / 거래처에 앱 설치 요구 / 팀 공유 / 회의 녹음
- **자비 결제 한도**: 월 1만 원 이하

#### 최정훈 대리 (P-13 도매 영업, 30세, 6년차)

- **소속**: 중견 식자재 도매업체, 영업2팀 대리
- **담당**: 거래처 120곳 (식당·카페·편의점)
- **하루**: 새벽 6시 출고장 → 적재 → 오전 배송 + 주문 → 오후 신규 개척 → 저녁 식당 미팅, 하루 15~20곳
- **도구**: 회사 주문 시스템(PDA) · 카톡 단톡방 · 거래명세서 · 종이 수첩 · 카카오맵
- **페인**:
  1. 한 손/음성 메모 필수 (양손 짐 들고 매장 들어감)
  2. 거래처 주문 패턴·외상·사장님 성향 머릿속에만
  3. 카톡 단톡방 200개에서 사장님 주문 메시지 놓침
  4. 신규 개척 후보 매장 정리할 곳 없음
- **NO 리스트**: 회사 주문 시스템 대체 / 사장님에 앱 설치 / 자동 발주 / 두 손 UI
- **자비 결제 한도**: 월 5,000원 이하 (1만 원 넘으면 고민)

## 3. 무엇을 만드나 — 제품 정의

### 3.1 제품 정의 한 줄

> **회사 시스템에 못 적는 영업 현장 메모를, 거래처/고객 앞에서 10초 안에 찾고 다시 남기는 개인용 영업 카드 앱.**

### 3.2 핵심 가치 제안 4가지

| # | 가치 | 이게 없으면 무슨 일이 일어나나 |
|---|------|------------------------------|
| 1 | **3~10초 카드 조회** | 거래처 앞 갓길에 차 세우거나 카톡을 뒤짐 (박상호) |
| 2 | **회사 시스템 분리 비공식 메모** | 가족·취향·결정권자 정보를 머리로만 외움 → 놓침 |
| 3 | **다음 액션·재구매 리마인더** | 후속 누락 → 매출 손실 + 신뢰 깎임 |
| 4 | **한 손 / 큰 글씨 / 오프라인** | 외근·노안·지하·트럭 환경에서 사용 불가 |

### 3.3 제품이 하지 않을 일 (정체성 보호)

| 제외 영역 | 이유 |
|----------|------|
| 회사 시스템(ERP·VEEVA·Salesforce·본사 가입) 대체 | 13개 중 12개 페르소나에서 회사 강제 + 대체 불가 |
| 견적·계약·등기·법적 문서 자동 생성 | 책임 소재 + 회사 양식 다양성 (D-301·D-304) |
| 주문·개통·결제 처리 | 회사 시스템 영역 |
| 자동 고객 메시지 발송 | 단골 관계 훼손 + 스팸 위험 |
| 민감 정보(주민번호·신분증·환자·처방 데이터) 무제한 저장 | 컴플라이언스 위험 |
| AI 자동 가격 예측·표출 (정확도 95% 미달) | 오차 1억/수백만원 시 송사 (부동산·자동차) |
| 방산용 일반 클라우드 SaaS | 보안상 부적합 → 별도 보안형 제품 트랙 |

## 4. 어떻게 사용하나 — User Journey

### 4.1 산업재 영업 박상호 차장의 하루

**아침 8:00 — 차에서 출발 전**
- 앱 열기 → 오늘 방문할 거래처 4곳 자동 정렬 (저장 필터)
- 첫 방문지 "A 부품사" 카드 미리보기 → 마지막 메모: "구매팀 최 부장 둘째 딸 대학 합격, 카네이션 보내기로 함"
- "다음 액션: 견적 후속 — 어제 보낸 견적 답변 받기" 확인

**오전 10:00 — 운전 중**
- 다음 거래처 "B 정비공장" 도착 5분 전, 차량 블루투스로 음성 명령 (P1 기능): "B 정비공장 카드 열어줘"
- 화면을 안 봐도 핵심 메모 음성으로 들음: "사장님 K씨, 휴대폰 안 받음, 카톡으로 약속 잡음"

**점심 12:30 — 식사 직후**
- "방문 후 1분 정리" 모드 (FR-12 D-407 후보)
- 3칸만 입력: 오늘 이야기 / 다음 액션 / 다시 연락할 날짜
- 30초 안에 저장 완료

**오후 3:00 — 명함 받음**
- 명함 사진 1장 촬영 → OCR → 거래처 자동 매칭 (FR-09)
- OCR 실패 시 이미지 + 수동 입력 전환 (P0 보강, D-002 신뢰 원칙)

**저녁 7:00 — 사무실 도착 안 하고 집으로**
- PC 안 켜도 모바일에서 오늘 메모 다시 보기
- 내일 방문 거래처 카드 미리 훑어보기 (5분)

### 4.2 도매 영업 최정훈 대리의 하루

**새벽 6:00 — 출고장**
- 적재 중 한 손으로 앱 열기 (NFR-03 큰 버튼)
- 오늘 루트 12곳 자동 표시 (이전 방문 메모 + 외상 현황)

**오전 9:00 — 매장 도착, 박스 들고 매장 들어감**
- 매장 안에서 사장님과 인사 + 박스 정리
- 매장 나오면서 음성 메모: "오늘 B 식당 외상 5만 늘었음, 다음 주 수금" (P1 음성 기능)
- 자동 거래처 매칭

**오전 11:00 — 신규 매장 발견**
- 매장 사진 1장 촬영 + 위치 자동 태깅
- 한 줄 메모: "신규 후보 — 카페, 사장 30대 여, 명함 받음"

**점심 1:00 — 트럭 안에서 빠른 정산**
- 오프라인에서도 오늘 거래처 12곳 카드 다시 조회 (P0 MVP-10)
- 미수금 있는 거래처 필터 (저장 필터)
- 와이파이 잡히면 자동 동기화

**저녁 6:00 — 카톡 단톡방 정리**
- "어제 D 식당 사장님이 카톡으로 주문한 거 검색" (FR-11 D-403)
- 카드 매칭 → 다음 액션 등록

### 7.1 데이터 모델 핵심

```text
users · cards · notes · actions · attachments
       │       │      │       │
       └───────┴──────┴───────┘
       relationships (from_card_id, to_card_id, relation_type, roles[], memo)
```

확장 모델(업종별, P1+): `vehicles · product_usages · store_visits · property_listings · contracts · items`

**첫 화면** — 대시보드보다 검색·오늘 할 일 우선

- 오늘 할 일 (FR-05)
- 이번 주 연락 대상 (FR-07)
- 최근 열람 카드
- 저장 필터 (FR-06)

**카드 상세 상단** — 3~10초 조회의 핵심
- 이름·회사
- 핵심 태그
- **마지막 메모**
- **다음 액션**
- 마지막 접촉일

**빠른 입력** (FR-12 1분 정리 모드 포함)
- 빠른 메모 / 태그 원터치 / 다음 액션 생성 / 사진 첨부 / 음성 메모

## 9. PMF 검증 계획

### 9.1 검증 가설 (PMF-H1~H5, D-406)

### 9.2 v3 인터뷰 계획 (5명 × 30분)

**구성**:
- 산업재 영업 3명 + 도매 영업 2명
- 30분 줌 또는 대면
- 음성 녹음만 (동의 후)

**순서** (12번 6장 가이드):
1. 0~5분 워밍업 + 현재 행동 (제품 설명 X)
2. 5~12분 대체 도구 + 페인 정량화
3. 12~20분 시나리오 점검 (운전 중·오프라인·알림)
4. 20~25분 제품 컨셉 반응 (마지막에만)
5. 25~30분 가격 + 마무리 (지인 추천 부탁)

**기록**:
- 가설별 검증/반증/모호 + 정량 답변
- 신규 페인 + 인용할 한마디
- 결과는 [09_PMF_검증.md](../09_PMF_검증.md)에 H1~H5 결과로 편입
