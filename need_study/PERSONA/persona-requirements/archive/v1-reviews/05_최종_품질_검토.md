## 1. 최종 판정

원본 문서는 사용자 요청의 기본 요구를 충족한다. 다만 그대로 의사결정 문서로 쓰기에는 세 가지 약점이 있다.

1. 요구사항 후보의 분류 용어가 일부 섞여 있다.
2. PR-01~PR-14가 어떤 대표 페인포인트에서 나온 것인지 추적성이 약하다.
3. 검증 질문은 있지만 PMF 가설과 베타 지표가 분리되어 있지 않다.

따라서 원본은 “초안 완료” 상태이고, 실제 제품 의사결정에 쓰려면 아래 최종 반영안을 적용하는 것이 좋다.

## 2. 최종 반영안

### 2.1 요구사항 분류 체계 정리

원본의 분류 값을 아래로 통일한다.

| 분류 | 의미 |
|------|------|
| Committed Enhancement | 이미 확정된 기능의 중요도/표현/UX를 보강 |
| Candidate | 채택 후보. 결정 기록 후 07번 문서 편입 가능 |
| Deferred | Phase 4 이후 또는 베타 후 판단 |
| Rejected | 기존 결정과 충돌하거나 만들지 않음 |
| Separate Product | 현재 제품이 아니라 별도 B2B/보안형 제품 후보 |

### 2.2 원본 4장 요구사항 후보 최종 형태

| ID | 요구사항 | 대표 페르소나 | 대표 페인포인트 | 권장 분류 |
|----|----------|----------------|------------------|-----------|
| PR-01 | 빠른 거래처/고객 카드 조회 | P-05, P-12 | 방문 직전 담당자/최근 상황을 3~10초 안에 봐야 함 | Committed Enhancement |
| PR-02 | 회사 시스템과 분리된 개인 비공식 메모 | P-05, P-06 | 회사 ERP/CRM에 못 적는 관계 정보가 있음 | Committed Enhancement |
| PR-03A | 명함 원본 이미지와 OCR 결과를 거래처에 연결 | P-05, P-03 | 명함이 쌓이고 나중에 누군지 기억 안 남 | Committed Enhancement |
| PR-03B | 점포/매물/매대 사진 자동 매칭 | P-02, P-09 | 사진이 갤러리에 쌓이고 대상 매칭이 안 됨 | Deferred |
| PR-04 | 오프라인 조회와 메모 작성 | P-05, P-12 | 이동 중 네트워크가 끊겨도 조회/기록해야 함 | Committed Enhancement |
| PR-05 | 이름/회사명/메모/태그 통합 검색 | P-01, P-05, P-06 | 카톡/엑셀/수첩에서 과거 정보를 못 찾음 | Candidate |
| PR-06 | 소개망/관계망 기록 | P-01, P-03, P-06 | 누가 누구를 소개했고 누가 결정권자인지 머릿속에만 있음 | Candidate |
| PR-07 | 자동 발송 없는 직접 연락 리마인더 | P-01, P-06, P-07 | 연락 시점은 중요하지만 자동 메시지는 신뢰를 깎음 | Deferred |
| PR-08 | OCR 실패 시 수동 입력 전환 | P-02, P-03, P-05 | OCR 실패가 곧 기능 포기로 이어질 수 있음 | Committed Enhancement |
| PR-16 | 방문/미팅 후 1분 정리 모드 | P-05, P-10, P-12 | 미팅 직후 정리하지 않으면 다음 액션을 잊음 | Candidate |
| PR-S01 | 민감 정보 저장 금지/주의 안내 | P-04, P-06, P-08, P-11 | 개인 앱에 넣으면 안 되는 정보 경계가 불명확함 | Committed Enhancement |

### 2.3 MVP 보강 최종 우선순위

| 우선순위 | 항목 | 반영 위치 |
|----------|------|-----------|
| 1 | 거래처 상세 첫 화면에 최근 메모 / 다음 액션 / 관계 키워드 표시 | F-06, F-14, F-15 |
| 2 | OCR 실패 시 이미지 보존 + 수동 입력 전환 | F-25, F-26 |
| 3 | 오프라인 상태 표시와 동기화 대기 카운트 | F-28, F-29 |
| 4 | 방문/미팅 후 1분 정리 모드 | F-14, F-29 |
| 5 | 민감 정보 저장 금지/주의 안내 | F-04, F-14 |

### 2.4 Phase 4 이후로 명확히 보내야 할 항목

| 항목 | 이유 |
|------|------|
| 점포/매물/매대 사진 자동 매칭 | 거래처 모델을 점포/매물 객체로 확장해야 함 |
| 보고서 초안 자동 생성 | 회사별 양식 대응 시 기존 제외 원칙과 충돌 |
| 정책 변경 교육 카드 | 휴대폰 매장 특화 요구로 범용 MVP와 거리 있음 |
| 도메인별 리마인더 | 재구매/만기/약정/잔금/공정거래 한도 규칙이 산업별로 다름 |
| 보안형 온프레미스 제품 | 방산/반도체/제약 일부는 별도 B2B 제품에 가까움 |

## 3. 최종 검증 계획

### 3.1 인터뷰에서 반드시 확인할 질문

1. 마지막 방문/미팅 직전에 어떤 정보를 다시 확인했나요?
2. 그 정보를 찾는 데 몇 초 또는 몇 분이 걸렸나요?
3. 미팅 직후 메모를 남기지 않아 잊은 적이 있나요?
4. “오늘 이야기 / 다음 액션 / 다시 연락할 날짜” 세 칸이면 충분한가요?
5. 명함 OCR이 틀렸을 때 수동 입력으로 이어지면 계속 쓰겠습니까?
6. 회사 시스템에 절대 적지 않는 정보와 개인 앱에도 적으면 안 되는 정보는 무엇인가요?

### 3.2 베타에서 봐야 할 지표

| 지표 | 임계값 초안 |
|------|-------------|
| 첫 주 사용자당 거래처 상세 조회 수 | 중앙값 10회 이상 |
| 검색 후 상세 진입률 | 40% 이상 |
| 방문 당일 메모 작성 비율 | 50% 이상 |
| 메모 작성 완료 시간 | 중앙값 60초 이하 |
| OCR 실패 후 수동 저장 완료율 | 60% 이상 |
| 오프라인 저장 후 동기화 성공률 | 95% 이상 |
