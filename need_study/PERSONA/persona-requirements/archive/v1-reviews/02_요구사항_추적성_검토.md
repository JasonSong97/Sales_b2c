## 2. 누락 또는 약한 연결

### 2.1 P-02 화장품 리테일의 핵심인 “점포 사진 자동 정리”가 PR-03에 묶여 너무 넓다

PR-03은 명함/현장 사진/점포 사진을 모두 포괄한다. 하지만 P-02의 핵심은 명함이 아니라 “매대 사진 200~300장 → 점포 자동 매칭 → 보고서 작성 시간 단축”이다.

**권장**
- PR-03을 유지하되 하위 요구로 분해한다.
  - PR-03A 명함/고객 사진 매칭
  - PR-03B 점포/매물/매대 사진 매칭
  - PR-03C 사진 기반 보고서 초안

### 2.2 P-04 방산은 제외로만 소비되고 있다

방산은 일반 B2C 앱의 타겟이 아니지만, 요구사항 정의에서 배울 수 있는 제약이 있다.

**추출 가능한 공통 요구**
- 민감 정보 입력 제한
- 데이터 저장 위치 명시
- 감사 로그
- 오프라인/폐쇄망 사용 가능성

**권장**
- PR-X05와 별도로 `PR-S01 민감 정보 저장 정책 명시`를 추가 후보로 둔다.

### 2.3 P-07 휴대폰 매장의 “정책 변경 요약”이 요구사항 후보에 약하게 반영됐다

현재 P-07은 PR-07 알림과 PR-09 도메인 리마인더에만 걸린다. 하지만 실제 핵심은 “주 단위 정책 변경을 직원 교육용 1장 카드로 요약”이다.

**권장**
- `PR-15 외부/본사 정책 변경 메모와 교육 카드`를 Phase 4 후보로 추가한다.
- 단, 통신사 정책 자동 수집은 제외하고 수동 입력/붙여넣기 기반으로 제한한다.

### 2.4 P-10 SaaS의 “미팅 후 정리 → 시스템 입력 부담”은 현재 MVP와 거리가 있지만 중요한 패턴이다

SaaS AE의 Salesforce 연동은 MVP 범위를 벗어나지만, 미팅 직후 정리 부담은 여러 산업에 공통이다.

**권장**
- `PR-16 방문/미팅 후 1분 정리 모드`를 Core 후보로 검토한다.
- 자동 연동 없이 “오늘 만난 사람, 한 약속, 다음 액션”만 빠르게 남기는 범위면 기존 정체성과 맞는다.

## 4. 2차 수정 권장안

1. PR-03을 사진 유형별 하위 요구로 나눈다.
2. 보안 페르소나에서 공통 적용 가능한 `민감 정보 저장 정책`을 별도 요구로 추가한다.
3. P-07의 정책 변경 요약을 Phase 4 후보로 추가한다.
4. P-10의 미팅 후 정리 패턴을 MVP 보강 후보로 추가한다.
5. 요구사항별 대표 페인포인트를 원본 4장에 추가한다.

---

## 5. 3차 검토로 넘길 쟁점

3차 검토에서는 추적성이 아니라 **MVP 범위와 기존 확정 기능 충돌**을 본다.

- PR-15, PR-16을 추가하면 MVP가 과도하게 커지는가
- PR-03 하위 분해가 실제 구현 범위를 늘리는가
- 민감 정보 저장 정책은 기능인가 운영 원칙인가
- Phase 0~3에 넣을 후보와 Phase 4 이후 후보를 다시 나눌 필요가 있는가
