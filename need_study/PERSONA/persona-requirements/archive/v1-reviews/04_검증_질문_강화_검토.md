## 1. 검증해야 할 핵심 가설

### PMF-H01 빠른 거래처 카드 조회

**가설**
B2B 외근 영업사원은 방문 직전 10초 안에 최근 메모와 관계 키워드를 확인할 수 있으면 기존 카톡/수첩보다 이 앱을 더 자주 열 것이다.

**주요 대상**
P-05 산업재, P-12 도매, P-03 살롱, P-01 자동차

**성공 신호**
- 인터뷰 5명 중 3명 이상이 “방문 직전 확인”을 주 3회 이상 한다고 답한다.
- 베타 첫 주 사용자당 거래처 상세 조회 중앙값이 10회 이상이다.

**반증 신호**
- “카톡 검색이면 충분하다”
- “미팅 전에 다시 보는 습관이 없다”
- “앱을 여는 10초가 귀찮다”

### PMF-H02 방문/미팅 후 1분 정리

**가설**
영업사원은 미팅 직후 1분 안에 `오늘 이야기 / 다음 액션 / 다시 연락할 날짜`만 남길 수 있으면 장문의 CRM 입력보다 지속적으로 기록할 것이다.

**주요 대상**
P-05 산업재, P-12 도매, P-10 SaaS, P-08 제약

**성공 신호**
- 인터뷰 5명 중 3명 이상이 “방문 직후 바로 적지 않으면 잊는다”고 답한다.
- 베타 첫 주 메모 중 50% 이상이 방문 당일 작성된다.

**반증 신호**
- “차라리 음성녹음/카톡 나에게 보내기가 더 빠르다”
- “다음 액션을 구조화해서 적기 귀찮다”

### PMF-H03 명함 OCR 실패 UX

**가설**
OCR이 완벽하지 않아도, 원본 이미지 보존과 수동 입력 전환이 자연스러우면 사용자는 명함 스캔을 포기하지 않는다.

**주요 대상**
P-05 산업재, P-03 살롱, P-09 부동산, P-02 리테일

**성공 신호**
- OCR 실패 후 수동 저장 완료율이 60% 이상이다.
- 인터뷰에서 “실패해도 이미지가 남으면 다시 정리할 수 있다”는 답이 반복된다.

**반증 신호**
- “한 번 실패하면 리멤버나 그냥 사진첩으로 간다”
- “수동 입력할 거면 스캔할 이유가 없다”

### PMF-H04 민감 정보 안내

**가설**
앱이 저장 금지 정보와 삭제/내보내기 정책을 명확히 안내하면, 사용자의 개인정보/회사 기밀 입력 불안을 줄일 수 있다.

**주요 대상**
P-06 보험, P-08 제약, P-11 반도체, P-04 방산, P-05 산업재

**성공 신호**
- 인터뷰에서 “무엇을 넣으면 안 되는지 명확하면 쓸 수 있다”는 답이 나온다.
- 온보딩에서 민감 정보 안내 이탈률이 과도하게 높지 않다.

**반증 신호**
- “안내가 있어도 외부 앱에는 못 적는다”
- “회사 정책상 개인 앱 자체가 안 된다”

## 2. 인터뷰 질문 보강안

기존 원본 문서의 6.1 질문은 방향이 맞지만 아직 일반적이다. 아래처럼 더 행동 중심으로 바꾸는 것이 좋다.

### 2.1 빠른 조회 검증 질문

1. 마지막으로 거래처/고객을 만나기 직전에 무엇을 확인했나요?
2. 그 정보를 찾는 데 몇 초 또는 몇 분이 걸렸나요?
3. 못 찾아서 대화가 어색해진 적이 있나요?
4. 방문 직전 화면에 딱 3개만 보여준다면 무엇이 있어야 하나요?
5. 카톡/수첩/엑셀보다 이 앱을 열려면 얼마나 빨라야 하나요?

### 2.2 방문 후 1분 정리 검증 질문

1. 미팅 직후 메모를 남기는 가장 최근 사례를 말해주세요.
2. 그때 어디에 적었나요?
3. 안 적고 지나가서 잊은 적이 있나요?
4. “오늘 이야기 / 다음 액션 / 다시 연락할 날짜” 세 칸이면 충분한가요?
5. 세 칸 중 하나만 남길 수 있다면 무엇을 남기겠습니까?

### 2.3 OCR 실패 UX 검증 질문

1. 명함을 찍었는데 인식이 틀리면 보통 어떻게 하나요?
2. 원본 사진만 남고 이름/전화는 수동 입력해야 한다면 계속 쓰겠습니까?
3. OCR 실패 후 수동 입력은 몇 초 안에 끝나야 참을 수 있나요?
4. 명함 사진을 저장하는 것 자체가 개인정보 측면에서 부담스럽나요?

### 2.4 민감 정보 안내 검증 질문

1. 회사 시스템에 절대 적지 않는 정보는 무엇인가요?
2. 개인 앱에도 절대 적으면 안 되는 정보는 무엇인가요?
3. 앱이 “이 정보는 적지 마세요”라고 안내하면 신뢰가 올라가나요, 오히려 불안해지나요?
4. 계정 삭제와 데이터 내보내기가 있으면 불안이 줄어드나요?

## 5. 4차 수정 권장안

1. 기존 6.1 공통 질문을 행동 기반 질문으로 교체한다.
2. 반증 신호는 유지하되 각 PMF 가설 아래로 나눠 붙인다.
3. 베타 지표를 추가해 문서가 인터뷰 후에도 계속 쓰이게 한다.
4. PR-16은 “방문/미팅 후 1분 정리”라는 이름으로 가설과 연결한다.
5. 민감 정보 안내는 기능 검증이 아니라 신뢰 조건 검증으로 다룬다.

---

## 6. 5차 검토로 넘길 쟁점

5차 검토에서는 최종 산출물 관점에서 본다.

- 1~4차 검토 결과가 서로 충돌하지 않는가
- 최종적으로 원본 문서에 반영할 변경안이 명확한가
- 사용자가 다음에 바로 할 수 있는 작업 단위가 있는가
- 회차별 검토 문서가 실제로 순차적으로 연결됐는가
