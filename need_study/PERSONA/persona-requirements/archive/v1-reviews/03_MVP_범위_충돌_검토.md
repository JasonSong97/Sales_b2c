## 1. 기존 확정 기능과의 관계

현재 확정된 Phase 0~3 기능은 인증, 거래처/회사/영업건, 메모, 필터, 엑셀/PDF, 명함 OCR, 오프라인, Admin 골격이다.

따라서 페르소나 요구사항은 다음 세 가지 중 하나로 처리해야 한다.

| 처리 | 의미 | 예 |
|------|------|----|
| MVP 보강 | 기존 F-01~F-33의 표현/UX를 좁히는 수준 | 거래처 상세에 최근 메모 표시 |
| Phase 4 후보 | 새 객체/규칙/도메인 로직이 필요한 기능 | 재구매 알림, 약정 만료, 매물 카드 |
| 별도 제품 | 보안/회사 결재/법적 책임 때문에 범위가 다른 기능 | 방산 온프레미스, 제약 컴플라이언스 |

## 3. MVP에 넣어도 되는 보강

### 3.1 PR-16 방문/미팅 후 1분 정리 모드

**판정: MVP 보강 후보**

이 기능은 새 도메인을 만들지 않아도 된다. 기존 메모(F-14/F-15)와 오프라인 변경(F-29)에 얹을 수 있다.

**MVP 범위**
- 거래처 상세 또는 홈에서 “방문 후 정리” 버튼 제공
- 입력 필드 3개만 둔다.
  - 오늘 나눈 이야기
  - 약속한 다음 액션
  - 다시 연락할 날짜
- 오프라인에서도 저장

**제외 범위**
- 자동 음성 인식
- Salesforce/VEEVA/ERP 연동
- 자동 카톡/이메일 발송
- AI 요약

### 3.2 PR-S01 민감 정보 저장 정책 명시

**판정: 제품 원칙으로 즉시 반영**

기능이라기보다 신뢰 조건이다. 보험/제약/반도체/방산뿐 아니라 산업재와 도매에서도 “회사 시스템에 못 적는 메모”를 다루므로 필수다.

**MVP 범위**
- 도움말 또는 온보딩에서 “회사 기밀, 법적 문서, 신분증, 처방 데이터, 가격표는 입력하지 말라”는 안내
- 삭제/내보내기 정책과 연결
- 민감정보 예시를 사용자에게 보여줌

**제외 범위**
- 법률 자문
- 자동 민감정보 탐지
- 보안 인증 주장

### 3.3 PR-03A 명함/고객 사진 매칭

**판정: MVP 보강**

기존 명함 OCR F-23~F-26과 직접 연결된다. 단, 점포/매물/매대 사진까지 MVP에 넣으면 범위가 커진다.

**MVP 범위**
- 명함 원본 이미지를 거래처에 연결
- OCR 실패 시 이미지와 수동 입력을 함께 보존

**제외 범위**
- 매대/매물 사진 자동 태깅
- 사진 기반 보고서
- 이미지 AI 분석

## 4. Phase 4 이후로 보내야 하는 보강

### 4.1 PR-03B 점포/매물/매대 사진 매칭

P-02/P-09에게 가치는 높지만 기존 MVP의 “거래처/회사/영업건” 모델을 점포/매물까지 확장해야 한다. Phase 4 이후 산업 특화 모듈로 판단한다.

### 4.2 PR-03C 사진 기반 보고서 초안

기존 문서에서 견적서/회사 양식 자동 생성은 제외되어 있다. 보고서 초안도 회사별 양식 대응을 요구하면 같은 문제가 생긴다. Phase 4에서 “본인 정리용 표준 보고서”로만 제한해야 한다.

### 4.3 PR-15 정책 변경 메모와 교육 카드

P-07에게는 강하지만 휴대폰 매장 특화 요구다. 범용 MVP에 넣으면 제품 정체성이 “정책 교육 도구”로 흔들린다.

## 5. MVP 보강 후보 최종안

원본 문서 5.2의 보강 후보를 다음처럼 다듬는 것을 권장한다.

| 우선순위 | 보강 후보 | 연결 기능 | 범위 |
|----------|-----------|-----------|------|
| 1 | 거래처 상세 첫 화면에 최근 메모 / 다음 액션 / 관계 키워드 표시 | F-06, F-14, F-15 | MVP 보강 |
| 2 | 방문/미팅 후 1분 정리 모드 | F-14, F-29 | MVP 보강 후보 |
| 3 | 명함 OCR 실패 시 이미지 보존 + 수동 입력 전환 | F-25, F-26 | MVP 보강 |
| 4 | 오프라인 상태 표시와 동기화 대기 카운트 | F-28, F-29 | MVP 보강 |
| 5 | 민감 정보 입력 금지/주의 안내 | F-04, F-14 | 제품 원칙 |

## 6. 3차 수정 권장안

1. PR-03을 MVP와 Phase 4로 분리한다.
2. PR-16을 “방문/미팅 후 1분 정리 모드”로 추가한다.
3. PR-S01을 요구사항이 아니라 제품 원칙/운영 원칙으로 둔다.
4. P-07 정책 변경 요약은 Phase 4 산업 특화 후보로 둔다.
5. 원본의 5.2 MVP 보강 후보를 위 표 기준으로 교체한다.

---

## 7. 4차 검토로 넘길 쟁점

4차 검토에서는 MVP 범위가 아니라 **검증 질문과 가설 품질**을 본다.

- PR-16을 실제 인터뷰에서 어떻게 검증할 것인가
- 민감 정보 안내가 사용자의 불안을 줄이는지 어떻게 확인할 것인가
- 사진 매칭은 명함과 현장 사진을 분리해서 질문해야 하는가
- “카톡 검색이면 충분” 반증 신호를 더 날카롭게 물어야 하는가
