## 3. F·G·H·I 분류 결정

11번 v2 신규 추가 문제 4개를 10번 v5 프레임워크로 분류.

### 3.1 추가 문제 F — 회사·본사 정책의 잦은 변경

| 항목 | 내용 |
|------|------|
| 강한 페르소나 | 휴대폰 (P-08), 제약 (P-09), 반도체 (P-12) |
| 1차 타깃과의 연관도 | 낮음 (산업재·도매에선 약함) |
| 10번 v5 매핑 | 없음 |
| **결정** | **P2 후보 신설** — 휴대폰·제약 확장 시 검토. MVP 직후가 아니라 2차 타깃 확정 후 |
| 신설 형태 | FR 또는 별도 모듈 — "본사 정책 변경 주간 요약 + 직원 교육용 1장 카드" |
| 검증 필요 | 페르소나별 정책 구조가 너무 달라서 일반화 가능성 의문 — 휴대폰 단독 모듈로 시작 권장 |

### 3.2 추가 문제 G — 정량 KPI / 보고서 부담

| 항목 | 내용 |
|------|------|
| 강한 페르소나 | H&B 리테일 (P-03), 휴대폰 (P-08), SaaS (P-11), 반도체 (P-12) |
| 1차 타깃과의 연관도 | 낮음 |
| 10번 v5 매핑 | 없음 (PDF Export·Excel Import은 있지만 보고서 자동 초안은 별도) |
| **결정** | **P2 또는 EX 경계** — 회사별 보고 양식이 너무 달라 80% 완성도 어려움. 일단 P2 후보로 두고 베타 후 재판단 |

### 4.2 베타 1차 인터뷰 = 5명 구성

- 산업재 영업 3명 (자동차 부품 1 + 기계 부품 1 + 건자재 1)
- 도매 영업 2명 (식자재 1 + 음료/주류 1)
- **선택**: 방판 1명·보험 1명 추가 인터뷰 (시간 여유 시)

### 4.3 베타 2~3차로 미루는 타깃

- 자동차·살롱: 베타 2차
- 부동산·휴대폰·화장품 리테일: 베타 3차 (2차 타깃, 10번 3.2)
- 제약·반도체·SaaS: 제한 타깃 — 별도 B2B 컴플라이언스 확장 검토 (10번 3.3)
- 방산: 제외 (10번 3.4 + 11번 7장 부록)

## 6. v3 사용자 인터뷰 가이드 (D-406)

### 6.1 인터뷰 구조

- **대상**: 5명 (산업재 영업 3 + 도매 영업 2)
- **시간**: 30분 / 1명
- **방식**: 비대면 줌 또는 대면 카페
- **녹화**: 동의 후 음성만 녹음
- **인터뷰어**: 1명 + 메모어 1명 권장

### 6.2 가설 5개 (PMF-H1~H5)

| ID | 가설 | 검증 방법 | 성공 신호 |
|----|------|----------|----------|
| **PMF-H1** | 산업재·도매 영업은 거래처 정보를 회사 ERP/주문 앱과 분리된 개인 메모에 적고 싶다 | "지금 거래처 메모는 어디에 적나요?"·답: 엑셀·카톡·종이·머릿속 중 어느 것이라도 회사 시스템 외부 | 5명 중 4명 이상 "회사 시스템 외부에 따로 적는다" |
| **PMF-H2** | 거래처/고객 카드 조회에 현재 30초~수분 걸리며, 10초로 줄면 가치 있다 | "지금 거래처 정보 다시 찾는 데 평균 몇 초~몇 분 걸려요?" | 5명 중 4명 이상 30초~3분 답변 + "10초면 좋겠다" 호응 |
| **PMF-H3** | 도매 영업은 운전 중 한 손/음성 메모를 사용 가능성의 임계점으로 본다 | "운전 중 메모는 어떻게 하나요? 양손 입력 가능한가요?" | 도매 2명 중 2명 "한 손/음성 필수" |
| **PMF-H4** | 다음 액션 + 리마인더가 후속 연락 누락을 줄인다 | "지난달 후속 연락 놓친 거래처가 몇 곳이었나요?" + "어떤 식으로 챙기면 좋을까요?" | 평균 3건 이상 + 리마인더 형식에 호응 |
| **PMF-H5** | 자비 결제 월 9,900~19,900원 가능 | "이런 도구가 있다면 월 얼마면 자비 결제 가능?" | 5명 중 3명 이상 9,900~19,900 범위 |

### 6.3 인터뷰 순서 (30분)

**0~5분 — 워밍업 + 현재 행동 (제품 설명 X)**
- 최근 거래처/고객 정보를 다시 찾느라 가장 짜증났던 순간은 언제였나요?
- 지금 거래처별 비공식 메모는 어디에 적고 있나요?
- 방문/거래처 앞 5분 동안 보통 무엇을 확인하나요?

**5~12분 — 대체 도구 + 페인 정량화**
- 명함·사진·카톡 메시지·메모 중 가장 자주 잃어버리는 정보는 무엇인가요?
- 회사 시스템에 절대 적지 않는 정보는 무엇인가요?
- 거래처 정보 다시 찾는 데 보통 몇 초~몇 분 걸려요?
- 지난달 후속 연락 놓친 거래처가 몇 곳이었나요?

**12~20분 — 시나리오 점검**
- 운전 중 메모는 어떻게 하나요? 양손 입력 가능한가요? (도매·산업재 핵심)
- 오프라인이나 이동 중에 정보를 못 봐서 문제가 된 적이 있나요?
- 자동 알림이 온다면 어떤 알림은 받고, 어떤 알림은 끄겠어요?

**20~25분 — 제품 컨셉 반응 (마지막에만)**
- (1줄 컨셉 제시) "회사 시스템에 못 적는 영업 현장 메모를, 거래처 앞에서 10초 안에 찾고 다시 남기는 개인용 영업 카드 앱"
- 한 줄로 뭐 하는 서비스 같아요?
- 본인 페인을 가장 정확히 짚는 부분은?
- 안 쓰게 될 이유 1~2개는?

**25~30분 — 가격 + 마무리**
- 이런 도구가 있다면 월 얼마면 자비 결제 가능?
- 회사 결재 받으려면 어떤 명분이 필요할까요?
- 비슷한 일을 하는 동료를 1명 소개해주실 수 있나요? (눈덩이 표집)

### 6.4 반증 신호 (이 답이 반복되면 가설 재검토)

- "그건 카톡 검색이면 충분해요." → PMF-H1 약화
- "회사 시스템에 이미 있어요." → 정체성 위협
- "개인정보를 외부 앱에 못 넣어요." → 보안 정책 강화 필요
- "앱 하나 더 여는 것 자체가 귀찮아요." → MVP-07 모바일 우선 + NFR-02 빠른 입력 강화
- "명함 OCR은 리멤버로 충분해요." → 명함 OCR P2 유지 정당화
- "메모는 하지만 다시 찾아보지는 않아요." → PMF-H2 약화
- "자동 알림은 귀찮아서 다 꺼요." → FR-07 리마인더 정책 재설계 (P1)
- "글씨가 너무 작아서 안 보여요." → NFR-03 P0 격상 검증
- "본사 정책이 자주 바뀌어서 한 달 후엔 안 맞을 거예요." → 정책 변경 모듈 회의적
