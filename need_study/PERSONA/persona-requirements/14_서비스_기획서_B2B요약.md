## 0. B2B 한정 한 줄 정의

> **회사 시스템(ERP/CRM)에 못 적는 B2B 영업 현장 메모를, 거래처·매장·어카운트 앞에서 10초 안에 찾고 다시 남기는 개인용 영업 카드 앱.**

- **타깃**: 외근·라운딩형 **B2B 영업 사원 개인** (B2C 페르소나 5명 제외)
- **1차 베타**: 산업재 + 도매 (5명)
- **수익화**: 현재 제외

## 2. 왜 만드나 — B2B 공통 문제 7축

8개 B2B 페르소나에서 반복되는 페인.

| 축 | 문제 | 강한 페르소나 |
|----|------|-------------|
| 정보 분산 | 거래처·어카운트 정보가 ERP·엑셀·카톡·종이에 흩어짐 | P-06 산업재·P-13 도매·P-03 H&B |
| 기억 복원 | 방문 직전 거래처 맥락 떠올리기 어려움 | P-06·P-11 SaaS·P-09 제약·P-12 반도체 |
| 후속 누락 | 견적·샘플·POC·정비 후속 놓침 | P-06·P-11·P-09·P-13 |
| 관계 구조 | 결정권자·다층 트랙(원장↔실장↔디자이너 / Champion↔EB↔DM) 머릿속에만 | P-04 살롱·P-11·P-12·P-09 |
| 현장 입력 | 모바일 한 손·음성·오프라인 필수 | P-13 도매·P-06·P-03 |
| 정책 변경 추적 | 본사 정책·임상 가이드라인 주~분기 단위 갱신 | P-09·P-12 |
| 보고/KPI 부담 | 매주 1~2시간 본사 보고 + 정량 KPI 압박 | P-03·P-11·P-12 |

### 2.1 기존 솔루션의 한계

- **회사 ERP/CRM** (VEEVA·Salesforce·DMS): 영업자 시각 아님. 비공식 메모 못 적음. 강제 사용 + 대체 불가.
- **엑셀·종이·카톡**: 외근 중 못 봄. 검색 안 됨. 분실 위험.
- **일반 CRM**: 영업맨 개인 관계 메모용이 아님. 무거움.

## 3. 무엇을 만드나

### 3.1 핵심 가치 4가지

| # | 가치 | 없으면 |
|---|------|------|
| 1 | 3~10초 거래처 카드 조회 | 갓길에 차 세우고 카톡 뒤짐 (P-06) |
| 2 | 회사 시스템 분리 비공식 메모 | 결정권자·취향·관계 머리로만 → 놓침 |
| 3 | 다음 액션·후속 리마인더 | 견적·POC·정비 후속 누락 → 매출 손실 |
| 4 | 한 손/오프라인/큰 글씨 | 트럭·공장·지방 도로에서 사용 불가 |

### 3.2 안 만들 것 (정체성)

- 회사 ERP/CRM 대체 (P-06·P-11·P-09·P-12 거부)
- 견적·계약·자동 주문·자동 결제 (책임 + 회사 정책)
- 거래처에 앱 설치 요구 (전원 거부)
- 자동 메시지 발송 (관계 훼손)
- 민감 데이터 무제한 저장 (P-09·P-12 컴플라이언스)
- 방산용 일반 클라우드 SaaS → 별도 보안형 제품 트랙

## 4. MVP 범위 (P0 10개)

| ID | 기능 |
|----|------|
| MVP-01 | 카드 관리 (거래처·회사·매장·어카운트) |
| MVP-02 | 빠른 검색 (2초 이내) |
| MVP-03 | 현장 메모 (회사 시스템과 분리) |
| MVP-04 | 태그 (견적후속·미방문·교체·시연·민감) |
| MVP-05 | 다음 액션 |
| MVP-06 | 저장 필터 |
| MVP-07 | 모바일 우선 UI |
| MVP-08 | 사용자별 데이터 분리 |
| MVP-09 | 민감정보 경고 |
| **MVP-10** | **쉬운 모드 (한 손·큰 버튼·최소 메뉴) — v2 P0 격상** |

### 4.1 P1 (MVP 직후)

- 리마인더 (자동 발송 X) · 오프라인 조회 · 업종별 카드 템플릿
- 관계망 + 다층 트랙 (`relationships.roles[]`) — P-04·P-11·P-09 핵심
- 음성 메모 / 한 손 입력 — P-06·P-13 P0 임계점
- 방문 후 1분 정리 모드 (FR-12) — P-06·P-11·P-13

### 4.2 P2 / EX

- P2: 매대·매물 사진 (P-03 P0급) · 명함 OCR · 보고서 자동 초안 · POC/시연 산업 객체
- EX: 본사 정책 변경 모듈(P-09·P-12 별도 트랙) · 자동 견적 · 방산 일반 SaaS

> **B2B 한정에서 빠진 것**: FR-11 카톡 채팅방 검색 — 13번 v1에선 B2C(보험·방판) 근거가 강했고 B2B에선 약함. **P1 → P2 강등** 또는 제외. P-06/P-13에선 카톡 단톡방(거래처 사장님) 정도라 가치 낮음.

## 5. 1차 베타 타깃 (B2B 한정 조정)

### 5.1 메인 인터뷰 5명 — 유지

| 페르소나 | 인원 | 선정 근거 |
|---------|------|---------|
| **P-06 산업재** | 3명 | 11번 v2 매트릭스 P0 1위. 거래처·비공식 메모·모바일 모두 ★★★★★ |
| **P-13 도매** | 2명 | 한 손·오프라인 P0 임계점. 정보 분산·모바일 ★★★★★ |

### 5.2 보조 인터뷰 — **B2B로 재배정**

13번 v1의 보조(P-02 방판·P-07 보험)는 B2C라 제외. 대체:

| 페르소나 | 검증 목적 |
|---------|---------|
| **P-04 살롱** | 다층 트랙(`roles[]`) + 시연·트레이닝 이력 + 인물 관계도 |
| **P-03 H&B** | 매대 사진 자동 점포 매칭 + 정량 KPI 압박 |

> **시니어 UI 검증 갭**: 13번 v1에선 P-02 방판(47세 노안)이 NFR-03 P0 격상 핵심 근거였다. B2B 한정에서는 시니어 단일 페르소나가 없어 한 손 조작(P-06·P-13) 근거만 남는다. **NFR-03 P0 정당화는 유지하되 "시니어 큰 글씨" 부분은 P1로 약화** 검토.

### 5.3 베타 단계

| 단계 | 사용자 | 목표 |
|------|--------|------|
| v3 인터뷰 | 5명 (P-06·P-13) | PMF-H1~H5 검증 |
| 베타 1차 | 10명 (P-06·P-13 중심) | 정량 지표 측정 |
| 베타 2차 | 20명 (+ P-04·P-03) | 다층 트랙·사진 검증 |
| 정식 출시 | 무제한 | 수익화 별도 기획 |

## 6. PMF 가설 5개 (B2B 한정 재해석)

| ID | 가설 | 1차 신호 |
|----|------|---------|
| **H1** | B2B 영업은 거래처 정보를 회사 ERP/주문 앱과 분리된 개인 메모에 저장하고 싶다 | 5명 중 4명 "회사 시스템 외부에 따로 적는다" |
| **H2** | 거래처 카드 조회 현재 30초~수분, 10초로 줄면 가치 있다 | 5명 중 4명 30초~3분 답 + "10초면 좋겠다" |
| **H3** | 도매·산업재는 운전 중 한 손/음성 메모를 P0 임계점으로 본다 | 도매 2명 + 산업재 1명 "한 손/음성 필수" |
| **H4** | 다음 액션 + 리마인더가 견적·샘플·후속 누락을 줄인다 | 5명 평균 누락 3건+ + 리마인더 형식 호응 |
| **H5** | 자비 결제 월 9,900~14,900원 가능 | 5명 중 3명 9,900~14,900 답 (B2C 19,900보다 낮춤) |

> **B2C 한정 가설 제거**: 13번 v1에서 보험·방판 기반 가설(카톡 검색·라이프이벤트·노안 UI)은 본 요약에서 제외.

## 7. 성공 지표 핵심 6개

13번 8.1 정량 10개 중 B2B에 가장 의미 있는 6개:

| # | 지표 | 임계값 |
|---|------|------|
| 1 | 카드 검색 시간 | 모바일 10초 이내 |
| 2 | 메모 저장 시간 | 상담 직후 30초 이내 |
| 3 | 다음 액션 등록률 | 활성 사용자 60% 이상 |
| 4 | 7일 재방문율 | 베타 사용자 50% 이상 |
| 5 | 방문 당일 메모 작성 비율 | 50% 이상 |
| 6 | 오프라인 저장 후 동기화 성공률 | 95% 이상 |

### 7.1 정성 신호 (사용자 발언)

- "방문 전에 이 카드만 보면 거래처 맥락이 떠오른다."
- "ERP에는 못 적는 걸 여기에 적는다."
- "견적 후속·샘플 결과를 안 놓친다."
- "엑셀보다 빠르고 카톡보다 찾기 쉽다."

### 7.2 반증 신호

- "그건 ERP 검색이면 충분해요." → H1 약화
- "회사 시스템에 이미 있어요." → 정체성 위협
- "운전 중 메모는 어차피 안 해요." → H3 약화

## 8. 로드맵 요약

| Phase | 기간 | 핵심 |
|-------|------|------|
| 0 | Week 0 | 인프라 셋업 + 5명 인터뷰 모집 |
| 1 | Week 1 | 인증 · MVP-01·02·03 (카드·검색·메모) + 큰 버튼 |
| 2 | Week 2 | MVP-04·05·06 (태그·다음액션·필터) + 오프라인 + 1분 정리 |
| 3 | Week 3 | Import + 민감정보 + 사용자 분리 |
| 4 | Week 4 | 명함 OCR + 베타 모집 (산업재·도매 10명) |
| 5+ | 출시 후 | 음성 메모 · 다층 트랙 · 업종 템플릿 · 사진 |

### 별도 트랙 (B2B 확장)
- **방산 보안형 제품**: 사내망 + 보안 인증 (별도 시장 조사)
- **제약·반도체 컴플라이언스 모듈**: VEEVA·Salesforce 보조 도구
- **H&B·살롱 산업 특화**: 매대 사진 · 시연 이력 · 다층 트랙
