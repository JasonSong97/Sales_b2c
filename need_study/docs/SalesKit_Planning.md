## 2. 서비스 개요

### 2.1 서비스 정의

SalesKit은 계측기, 산업 장비, 설비 부품처럼 거래 단가가 높고 상담 주기가 긴 B2B 영업을 관리하기 위한 경량 영업 파이프라인 도구다.

영업 담당자는 여러 고객사의 영업 건을 단계별로 관리하고, 영업 팀장은 예상 매출, 수주 가능성, 마감 임박 건, 집중관리 대상을 빠르게 확인할 수 있다.

### 2.2 현재 데모의 성격

현재 `test.html`은 서버 없이 브라우저에서 실행되는 정적 데모다.

- 데이터는 JavaScript 배열에 하드코딩되어 있다.
- 새로 추가한 영업 건, 활동 기록, 토글 상태는 새로고침하면 사라진다.
- 로그인, 권한, 데이터베이스, API 연동은 없다.
- 회사 등록, 제품 등록, 단가 수정, 명함 스캔은 실제 기능이 아니라 안내 모달만 표시한다.
- CSV 내보내기는 브라우저 Blob 다운로드 방식으로 동작한다.

### 2.3 서비스가 해결하려는 문제

B2B 영업 현장에서는 다음 문제가 자주 발생한다.

- 고객사, 담당자, 제품, 견적 금액, 마감일 정보가 여러 문서에 흩어져 있다.
- 영업 건의 진행 단계와 수주 가능성을 한눈에 보기 어렵다.
- 팀장이 이번 달 마감 예정 건이나 확도 높은 건을 즉시 파악하기 어렵다.
- 영업 활동 기록이 개인 메모에 남아 팀 공유가 어렵다.
- 보고용 데이터를 다시 엑셀로 정리하는 반복 작업이 발생한다.

SalesKit은 이 문제를 가볍고 빠른 파이프라인 관리 화면으로 해결하는 것을 목표로 한다.

## 6. 화면별 상세 기획

### 6.1 대시보드

#### 6.1.1 목적

대시보드는 전체 영업 현황을 빠르게 요약해 팀장이 파이프라인 상태를 점검할 수 있게 한다.

#### 6.1.2 표시 지표

| 지표 | 계산 기준 | 현재 구현 |
| --- | --- | --- |
| 진행 중 파이프라인 | `stage !== "lost"`인 영업 건 수 | 구현됨 |
| 예상 매출 | 실패 제외 영업 건의 `amount` 합계 | 구현됨 |
| 70% 이상 건 | `prob >= 70`이고 `stage !== "won"`인 건 수 | 구현됨 |
| 수주 완료 | `stage === "won"`인 건 수 | 구현됨 |

#### 6.1.3 마감 임박 목록

- 전체 영업 건을 마감일 오름차순으로 정렬한다.
- 상위 5개 영업 건을 표시한다.
- 표시 항목은 건명, 거래처, 단계, 금액, 마감일이다.
- 현재 구현은 실패 건도 포함될 수 있으므로 향후에는 실패 건 제외 여부를 정책으로 정해야 한다.

#### 6.1.4 기대 동작

- 대시보드 행을 클릭하면 해당 영업 건 상세 패널을 열 수 있다.
- 데이터가 변경되면 대시보드 지표가 다시 계산되어야 한다.
- 향후 저장 기능이 추가되면 서버 데이터 기준으로 재계산해야 한다.

### 6.2 건 관리

#### 6.2.1 목적

건 관리는 SalesKit의 핵심 화면으로, 영업 건 목록 조회, 필터링, 상세 확인, 등록, 복사, 삭제, 활동 기록 추가를 담당한다.

#### 6.2.2 목록 테이블

| 컬럼 | 데이터 | 표시 방식 |
| --- | --- | --- |
| 건명 | `deal.title` | 굵은 텍스트 |
| 거래처 | `deal.contact`, `deal.person` | 회사/고객사명과 담당자명 2줄 표시 |
| 지역 | `deal.region` | 보조 텍스트 |
| 단계 | `deal.stage` | 색상 배지 |
| 금액 | `deal.amount` | 만원 단위 축약 표시 |
| 확도 | `deal.prob` | 진행 막대와 퍼센트 |
| 마감일 | `deal.date` | `YYYY.MM.DD` 문자열 |

#### 6.2.3 단계 정의

| 값 | 라벨 | 색상 의미 | 설명 |
| --- | --- | --- | --- |
| `prospect` | 초기접촉 | 파란색 | 신규 상담 또는 초기 검토 단계 |
| `negotiation` | 협의중 | 주황색 | 견적, 납기, 단가, 조건 협의 단계 |
| `won` | 수주 | 초록색 | 계약 또는 발주가 완료된 상태 |
| `lost` | 실패 | 빨간색 | 수주 실패, 보류, 경쟁사 선정 등 |

#### 6.2.4 탭 필터

| 탭 | 조건 |
| --- | --- |
| 전체 | 모든 영업 건 |
| 초기접촉 | `stage === "prospect"` |
| 협의중 | `stage === "negotiation"` |
| 수주 | `stage === "won"` |
| 집중관리 | `prob >= 70 && stage !== "won"` |
| 이번 달 | 마감일의 연/월이 현재 연/월과 같은 건 |

현재 구현상 이번 달 카운트는 `2026.05`로 고정 계산되고, 실제 필터링은 브라우저 현재 날짜를 사용한다. 향후에는 동일한 기준 함수로 통합해야 한다.

#### 6.2.5 검색

검색 대상 필드는 다음과 같다.

- 건명
- 거래처 표시명
- 담당자명
- 회사명
- 메모
- 제품명

검색 정책은 다음과 같다.

- 대소문자를 구분하지 않는다.
- 공백을 제거한 뒤 빈 검색어면 전체를 표시한다.
- 현재는 단순 문자열 포함 검색이다.
- 향후에는 초성 검색, 회사명/제품명 필드별 검색을 검토할 수 있다.

#### 6.2.6 필터

| 필터 | 값 | 설명 |
| --- | --- | --- |
| 지역 | 전체, 수원, 대전, 포항, 이천, 당진, 여수 | `deal.region` 기준 |
| 확도 | 전체, 40% 이상, 70% 이상, 90% 이상 | `deal.prob >= 선택값` 기준 |

#### 6.2.7 정렬

| 정렬 옵션 | 기준 |
| --- | --- |
| 마감일 빠른순 | `date` 오름차순 |
| 마감일 늦은순 | `date` 내림차순 |
| 금액 높은순 | `amount` 내림차순 |
| 금액 낮은순 | `amount` 오름차순 |
| 확도 높은순 | `prob` 내림차순 |

현재 날짜는 문자열 비교로 정렬된다. `YYYY.MM.DD` 형식을 유지하면 정상 동작하지만, 향후에는 Date 객체 또는 ISO 날짜 문자열로 변환해 비교해야 한다.

#### 6.2.8 빈 상태

필터 조건에 맞는 영업 건이 없으면 테이블 영역에 `조건에 맞는 영업 건이 없습니다.` 메시지를 표시한다.

#### 6.2.9 행 선택

- 사용자가 테이블 행을 클릭하면 해당 영업 건이 선택된다.
- 선택된 행은 강조 배경으로 표시된다.
- 우측 상세 패널이 열린다.
- 다른 행을 클릭하면 선택 대상이 교체된다.
- 상세 패널 닫기 또는 선택 해제 시 `selectedDealId`는 `null`로 초기화된다.

### 6.3 영업 건 상세 패널

#### 6.3.1 목적

상세 패널은 목록을 벗어나지 않고 선택한 영업 건의 핵심 정보와 활동 기록을 확인하는 영역이다.

#### 6.3.2 헤더

| 요소 | 내용 |
| --- | --- |
| 제목 | 영업 건명 |
| 보조 정보 | 거래처 또는 회사명 / 담당자명 |
| 닫기 버튼 | 상세 패널 닫기 |

#### 6.3.3 탭 구성

| 탭 | 표시 내용 |
| --- | --- |
| 정보 | 제품, 예상 금액, 단계, 확도, 지역, 마감일 |
| 활동 | 활동 타임라인, 새 활동 입력 |
| 메모 | 일반 상담 메모 |
| 오프레코드 | 내부 참고 메모 |

#### 6.3.4 활동 기록

활동 기록은 타임라인 형태로 표시된다.

- 자동 기록은 회색 점으로 표시한다.
- 수동 기록은 강조색 점으로 표시한다.
- 새 활동 입력 후 `추가` 버튼을 누르면 현재 날짜 기준으로 활동이 맨 위에 추가된다.
- 빈 입력값은 추가하지 않는다.

#### 6.3.5 상세 액션

| 액션 | 현재 동작 | 향후 개선 |
| --- | --- | --- |
| 복사 | 기존 영업 건을 복사해 새 건 생성 | 복사 전 확인 또는 복사 옵션 제공 |
| 삭제 | 선택된 영업 건 즉시 삭제 | 삭제 확인 모달 추가 |
| 선택 해제 | 상세 패널 닫기 | 유지 |

### 6.4 신규 영업 건 등록

#### 6.4.1 진입

사용자는 건 관리 화면 상단의 `건 등록` 버튼을 클릭해 등록 모달을 연다.

#### 6.4.2 입력 필드

| 필드 | 필수 여부 | 현재 처리 |
| --- | --- | --- |
| 건명 | 필수로 다루는 것이 적절 | `title`로 저장 |
| 회사 | 필수로 다루는 것이 적절 | `company`, `contact`로 저장 |
| 담당자 | 선택 | `person`으로 저장 |
| 지역 | 선택 | `region`으로 저장 |
| 금액 | 선택 | 숫자 이외 문자 제거 후 number 저장 |
| 확도 | 선택 | 0-100 범위로 보정 |
| 마감일 | 선택 | 문자열 그대로 저장 |
| 제품 | 선택 | 빈 값이면 `제품 미정` |
| 메모 | 선택 | `memo`로 저장 |

#### 6.4.3 등록 후 동작

- 기존 ID 중 최댓값에 1을 더해 새 ID를 만든다.
- 신규 건은 목록 맨 앞에 추가된다.
- 단계는 `prospect`로 저장된다.
- 오프레코드 메모는 빈 문자열로 저장된다.
- 활동 기록에 `영업 건 등록` 자동 기록이 추가된다.
- 모달을 닫고 입력값을 초기화한다.
- 탭은 전체로 변경된다.
- 신규 건이 자동 선택되어 상세 패널에 표시된다.

#### 6.4.4 향후 검증 정책

- 건명, 회사, 마감일은 필수로 검토한다.
- 금액은 0 이상 정수만 허용한다.
- 확도는 0-100 범위 숫자만 허용한다.
- 마감일은 `input type="date"` 또는 ISO 형식으로 관리한다.
- 회사, 담당자, 제품은 기존 마스터 데이터와 연결할 수 있어야 한다.

### 6.5 거래처

#### 6.5.1 목적

거래처 화면은 고객사 담당자 정보를 빠르게 조회하는 화면이다.

#### 6.5.2 표시 항목

| 항목 | 설명 |
| --- | --- |
| 아바타 | 담당자 이름 첫 글자, 지정 색상 |
| 이름 | 담당자명 |
| 직책 | 차장, 과장, 대리, 팀장 등 |
| 회사 | 소속 회사 |
| 부서 | 소속 부서 |
| 지역 | 담당 지역 |
| 관련 영업 건 수 | 해당 담당자와 연결된 영업 건 수 |
| 전화번호 | 마스킹된 전화번호 |

#### 6.5.3 검색

검색 대상은 이름, 회사, 부서다.

#### 6.5.4 명함 스캔

현재 `명함 스캔` 버튼은 안내 모달만 표시한다.

향후 실제 기능으로 확장할 경우 다음 흐름이 필요하다.

- 이미지 업로드 또는 카메라 촬영
- OCR 결과 추출
- 이름, 회사, 부서, 직책, 전화번호 자동 매핑
- 사용자의 확인 후 거래처 저장

### 6.6 회사

#### 6.6.1 목적

회사 화면은 고객사 단위의 기본 정보를 조회하는 화면이다.

#### 6.6.2 표시 항목

| 항목 | 설명 |
| --- | --- |
| 회사명 | 고객사명 |
| 산업군 | 반도체/전자, 화학, 철강, 정유/화학 등 |
| 지역 | 주요 담당 지역 |
| 담당자 수 | 등록된 담당자 수 |
| 영업 건 수 | 연결된 영업 건 수 |

#### 6.6.3 현재 한계

- 회사 등록 버튼은 안내 모달만 표시한다.
- 회사 상세 화면은 없다.
- 회사와 영업 건의 실제 관계 편집 기능은 없다.

| 최근 변경 이력 | 가장 최근의 이전 단가와 변경일 |

#### 6.7.3 향후 확장

- 단가 변경 이력 자동 누적
- 제품별 연결된 영업 건 목록
- 견적서 단가 자동 반영

## 8. 사용자 흐름

### 8.1 영업 현황 확인

1. 사용자는 SalesKit을 열면 기본적으로 건 관리 화면을 본다.
2. 대시보드 메뉴로 이동한다.
3. 진행 중 파이프라인, 예상 매출, 집중관리 건, 수주 완료 건을 확인한다.
4. 마감 임박 목록에서 우선 대응할 건을 확인한다.
5. 특정 행을 클릭해 상세 패널에서 세부 정보를 확인한다.

### 8.2 영업 건 검색 및 필터링

1. 사용자는 건 관리 화면에 진입한다.
2. 탭에서 전체, 초기접촉, 협의중, 수주, 집중관리, 이번 달 중 하나를 선택한다.
3. 검색어를 입력한다.
4. 지역과 확도 필터를 선택한다.
5. 정렬 조건을 변경한다.
6. 조건에 맞는 목록을 확인한다.
7. 결과가 없으면 빈 상태 메시지를 확인한다.

### 8.3 영업 건 상세 확인

1. 사용자는 영업 건 행을 클릭한다.
2. 우측 상세 패널이 열린다.
3. 정보 탭에서 제품, 금액, 단계, 확도, 지역, 마감일을 확인한다.
4. 활동 탭에서 이전 상담 기록을 확인한다.
5. 메모 탭에서 일반 상담 메모를 확인한다.
6. 오프레코드 탭에서 내부 참고 내용을 확인한다.

### 8.4 활동 기록 추가

1. 사용자는 상세 패널의 활동 탭을 연다.
2. 새 활동 기록 입력란에 내용을 입력한다.
3. `추가` 버튼을 클릭한다.
4. 활동 기록이 현재 날짜와 함께 타임라인 상단에 추가된다.
5. 빈 값인 경우 아무 동작도 하지 않는다.

### 8.5 신규 영업 건 등록

1. 사용자는 상단 `건 등록` 버튼을 클릭한다.
2. 등록 모달이 열린다.
3. 건명, 회사, 담당자, 지역, 금액, 확도, 마감일, 제품, 메모를 입력한다.
4. 등록 버튼을 클릭한다.
5. 신규 건이 목록 맨 앞에 추가된다.
6. 신규 건 상세 패널이 자동으로 열린다.

### 8.6 영업 건 복사

1. 사용자는 영업 건을 선택한다.
2. 상세 패널 하단의 `복사` 버튼을 클릭한다.
3. 기존 영업 건의 주요 데이터가 복사된다.
4. 새 건명은 기존 건명 뒤에 `복사본`이 붙는다.
5. 단계는 `초기접촉`, 확도는 `0%`로 초기화된다.
6. 자동 활동 기록에 `기존 영업 건에서 복사`가 추가된다.

### 8.7 영업 건 삭제

1. 사용자는 영업 건을 선택한다.
2. 상세 패널 하단의 `삭제` 버튼을 클릭한다.
3. 현재 구현에서는 확인 절차 없이 즉시 삭제된다.
4. 상세 패널이 닫히고 목록이 갱신된다.

향후에는 삭제 확인 모달을 반드시 추가해야 한다.

### 8.8 CSV 내보내기

1. 사용자는 사이드바 또는 건 관리 상단의 `CSV 내보내기` 버튼을 클릭한다.
2. 브라우저가 `saleskit-deals.csv` 파일을 다운로드한다.
3. CSV에는 BOM이 포함되어 Excel에서 한글이 깨지지 않도록 한다.
4. 포함 컬럼은 건명, 회사, 담당자, 지역, 단계, 금액, 확도, 마감일이다.

## 9. 정책 및 규칙

### 9.1 금액 표시

- 상세 패널에서는 원 단위 전체 금액을 표시한다.
- 목록과 대시보드에서는 만원 단위로 축약 표시한다.
- 숫자는 한국어 로케일 기준 천 단위 콤마를 사용한다.

### 9.2 확도 표시

| 확도 | 표시 색상 |
| --- | --- |
| 70% 이상 | 파란색 |
| 40% 이상 70% 미만 | 주황색 |
| 40% 미만 | 빨간색 |

### 9.3 날짜 정책

현재 날짜는 `YYYY.MM.DD` 문자열로 저장한다.

향후 정책은 다음과 같이 개선한다.

- 내부 저장은 `YYYY-MM-DD` ISO 형식 사용
- 화면 표시만 `YYYY.MM.DD`로 변환
- 날짜 필드는 `input type="date"` 사용
- 잘못된 날짜 입력 차단
- 마감 임박 계산은 브라우저 로컬 타임존 기준으로 통일

### 9.4 CSV 정책

- 파일명은 `saleskit-deals.csv`다.
- 인코딩은 UTF-8 BOM 포함을 기본으로 한다.
- 모든 셀은 쌍따옴표로 감싼다.
- 값 내부의 쌍따옴표는 두 번 반복해 이스케이프한다.
- 현재는 전체 영업 건을 내보내며, 필터된 결과만 내보내는 옵션은 없다.

### 9.5 데이터 저장 정책

현재 데모는 브라우저 메모리만 사용한다. 향후 단계별 저장 정책은 다음과 같다.

| 단계 | 저장 방식 |
| --- | --- |
| 1차 고도화 | LocalStorage |
| 2차 고도화 | API 서버와 DB |
| 3차 고도화 | 팀/권한 기반 멀티테넌트 저장 |

## 10. UI/UX 상세 방향

### 10.1 시각 스타일

- 배경은 연한 회색/베이지 톤을 사용해 업무 화면의 피로도를 낮춘다.
- 주요 액션은 파란색 계열로 통일한다.
- 성공, 경고, 위험 상태는 초록, 주황, 빨강으로 구분한다.
- 카드와 테이블은 8-12px 수준의 절제된 라운드를 사용한다.

### 10.2 정보 설계

- 한 화면에서 목록과 상세를 함께 확인할 수 있게 한다.
- 반복 조회가 많은 필터는 목록 상단에 고정 배치한다.
- 금액, 확도, 마감일처럼 판단에 중요한 값은 시각적 강조를 둔다.
- 보조 정보는 작은 글자와 muted 색상으로 구분한다.

### 10.3 인터랙션

- 버튼은 hover, active 상태를 제공한다.
- 선택된 메뉴와 탭은 active 상태를 명확히 표시한다.
- 테이블 행 hover와 selected 상태를 구분한다.
- 모달 바깥 영역 클릭 시 닫을 수 있다.
- 상세 패널 닫기 버튼을 제공한다.

### 10.4 접근성 개선 필요 항목

현재 데모에서 향후 개선할 접근성 항목은 다음과 같다.

- 아이콘 버튼에 명확한 `aria-label` 추가
- 모달 열림 시 포커스 트랩 적용
- 모달 닫힘 시 이전 버튼으로 포커스 복귀
- 키보드로 탭, 행, 상세 패널 탐색 가능하게 개선
- 색상만으로 상태를 구분하지 않도록 텍스트 라벨 유지
- 테이블 헤더와 데이터 셀 관계 명확화

## 11. 반응형 정책

### 11.1 900px 이하

- 사이드바 폭을 68px로 축소한다.
- 브랜드 보조 문구, 메뉴 텍스트, 카운트, 프로필 텍스트를 숨긴다.
- 메뉴 아이콘은 중앙 정렬한다.
- 상세 패널은 화면 우측 고정 오버레이로 표시한다.
- 대시보드 통계 카드는 2열로 표시한다.

### 11.2 640px 이하

- 상단 바 좌우 패딩을 줄인다.
- 주요 액션 외 보조 버튼을 숨긴다.
- 콘텐츠 패딩을 줄인다.
- 검색 입력은 전체 너비로 표시한다.
- 대시보드 통계 카드는 1열로 표시한다.
- 등록 모달의 폼 그리드는 1열로 변경한다.
- 테이블의 일부 컬럼은 숨긴다.

### 11.3 향후 모바일 개선

- 테이블을 카드형 목록으로 전환하는 방안을 검토한다.
- 상세 패널은 하단 시트 또는 전체 화면 패널로 전환할 수 있다.
- 필터 영역은 접기/펼치기 패널로 변경할 수 있다.
- 긴 버튼 텍스트는 아이콘 중심으로 축약한다.

## 13. 현재 한계와 리스크

### 13.1 기능 한계

- 새로고침 시 등록, 삭제, 활동 추가 데이터가 사라진다.
- 영업 건 수정 기능이 없다.
- 영업 단계 변경 기능이 없다.
- 실패 처리 사유를 기록할 수 없다.
- 거래처/회사/제품과 영업 건이 실제 ID로 연결되어 있지 않다.
- 제품 단가 변경이 영업 건 금액에 반영되지 않는다.
- CSV 가져오기가 없다.

### 13.2 데이터 한계

- ID 생성이 현재 배열의 최댓값 + 1 방식이다.
- 날짜가 문자열 비교에 의존한다.
- 담당자 수, 영업 건 수는 실제 관계 계산이 아니라 정적 값이다.
- 전화번호는 마스킹 문자열만 제공한다.
- 오프레코드 메모 권한 분리가 없다.

### 13.3 UX 한계

- 삭제 확인 절차가 없다.
- 필수 입력값 누락에 대한 명확한 오류 메시지가 없다.
- 모바일 테이블 사용성이 제한적이다.
- 모달 접근성 처리가 부족하다.
- 상세 패널에서 직접 편집할 수 없다.

### 13.4 기술 리스크

- 모든 로직이 단일 HTML 파일에 있어 유지보수가 어렵다.
- CSS와 JavaScript가 같은 파일에 있어 구조 확장이 어렵다.
- 상태 관리가 전역 변수 중심이다.
- 테스트 코드가 없다.
- 데이터가 많아질 경우 렌더링 최적화가 필요하다.

## 15. 수용 기준

### 15.1 공통

- 사용자는 브라우저에서 `test.html`을 열어 앱을 사용할 수 있어야 한다.
- 주요 메뉴를 클릭하면 해당 화면으로 전환되어야 한다.
- 현재 페이지 제목이 상단 바에 표시되어야 한다.
- 900px, 640px 이하 화면에서도 주요 탐색과 조회가 가능해야 한다.

### 15.2 대시보드

- 진행 중 파이프라인 수가 표시되어야 한다.
- 예상 매출 합계가 표시되어야 한다.
- 확도 70% 이상 건 수가 표시되어야 한다.
- 수주 완료 건 수가 표시되어야 한다.
- 마감 임박 목록이 최대 5건 표시되어야 한다.

### 15.3 건 관리

- 탭 선택 시 목록이 조건에 맞게 변경되어야 한다.
- 검색어 입력 시 목록이 실시간으로 필터링되어야 한다.
- 지역 필터와 확도 필터가 정상 적용되어야 한다.
- 정렬 조건 변경 시 목록 순서가 변경되어야 한다.
- 영업 건 행을 클릭하면 상세 패널이 열려야 한다.
- 선택된 행은 시각적으로 구분되어야 한다.
- 조건에 맞는 건이 없으면 빈 상태 메시지가 표시되어야 한다.

### 15.4 상세 패널

- 정보, 활동, 메모, 오프레코드 탭 전환이 가능해야 한다.
- 활동 기록이 타임라인 형태로 표시되어야 한다.
- 새 활동 입력 후 추가하면 타임라인에 반영되어야 한다.
- 복사 버튼 클릭 시 복사본이 생성되어야 한다.
- 삭제 버튼 클릭 시 해당 건이 목록에서 제거되어야 한다.
- 닫기 버튼 클릭 시 상세 패널이 닫혀야 한다.

### 15.5 등록 모달

- `건 등록` 버튼 클릭 시 모달이 열려야 한다.
- 등록 완료 시 신규 건이 목록에 추가되어야 한다.
- 신규 건은 자동 선택되어 상세 패널에 표시되어야 한다.
- 모달 닫기 버튼 또는 바깥 영역 클릭 시 모달이 닫혀야 한다.

### 15.6 CSV 내보내기

- CSV 내보내기 버튼 클릭 시 `saleskit-deals.csv`가 다운로드되어야 한다.
- CSV에는 건명, 회사, 담당자, 지역, 단계, 금액, 확도, 마감일이 포함되어야 한다.
- Excel에서 한글이 깨지지 않도록 UTF-8 BOM이 포함되어야 한다.
