# 1. Project One Pager

## 프로젝트명

- **SalesKit** (영업사원의 머릿속을 정리해주는 도구)

## 한 줄 설명

> 이 프로젝트는 **거래처 수십 곳을 동시에 관리하는 B2B 기술영업 개인**을 위한 **흩어진 영업 정보를 한 계정에 모아 PC·모바일 어디서든 꺼내보게 하는 개인용 영업 관리 서비스**이다.

- 회사가 도입하는 무거운 CRM이 아니라, 영업사원 개인이 자기 돈 내고 쓰는 **개인 소유 도구**.

## 해결하려는 문제

- 영업의 진짜 자산(누가 뭘 원하는지 · 언제 결정 나는지 · 어떻게 접근해야 하는지)이 **엑셀 · 명함 서랍 · 카톡 나에게 보내기에 흩어져** 있고, **현장에서 꺼내볼 수 없다**.
- 회사 CRM은 ① 너무 무겁고(팀 관리/리포트/권한), ② 회사 종속적이라 **퇴사 시 내 데이터를 못 가져간다**.

## 1차 타겟 사용자

| 항목 | 내용 |
| --- | --- |
| 핵심 사용자 | 계측기 기술영업사원(1차) → 산업재 · 의료기기 · 화학약품 등 **B2B 기술영업 전 도메인** |
| 인구통계 | **30~50대 남성** (주 타겟) |
| 사용자 직군/상황 | 거래처 수십 곳을 깊고 오래 관리하며 각 거래처의 상황·맥락을 기억해야 하는 개인 영업직 |
| 사용 환경 | PC(사무실) + 모바일(외근/차량/고객사). 오프라인 상황 빈번 |
| 제외할 사용자 | 팀 단위 도입 고객, 영업관리자(팀장) 리포팅 목적 사용자, 단순 개인 연락처 관리 수요 |

## 핵심 가치

| 번호 | 핵심 가치 |
| --- | --- |
| 1 | 현장에서 바로 꺼내본다 — PC 입력이 모바일에 즉시. 오프라인에서도 마지막 데이터 유지 |
| 2 | 공식 기록에 못 남기는 비공식 인사이트(오프더레코드 + 담당자 메모 로그)를 안전하게 보관 |
| 3 | 지금 뭐가 급한지 한눈에 — 이번 달 마감 / 가능성 70%+ 가 버튼 하나로 |
| 4 | 명함이 데이터가 된다 — 카메라 촬영 → AI OCR → 검색 가능한 거래처 / AI 회의록으로 미팅 맥락 자동 정리 |

## MVP 목표

- **웹(User 웹)에서 영업 관리 핵심 루프 완성**: 거래처/회사/담당자 등록 → 딜 생성·단계 관리·활동 타임라인 → 명함 OCR → AI 회의록 → 필터/검색/내보내기.
- 개발자 본인이 **실제 영업 데이터로 매일 사용**할 수 있는 수준.

## 성공 기준

| 지표 | 목표 |
| --- | --- |
| 핵심 루프 완주율 (거래처 등록 → 딜 생성 → 활동 기록) | 신규 사용자의 70%+ |
| 주간 재방문 (영업사원이 매일 아침 여는 화면이 되는가) | WAU/가입자 40%+ |
| 명함 OCR 추출 후 수정 없이 저장 비율 | 60%+ (OCR 신뢰도 체감 지표) |
| AI 회의록 생성 후 저장 전환율 | 50%+ |

> 위 목표 수치는 **초기 가설값**. 베타 데이터 확보 후 재설정.

## 이번 버전에서 하지 않을 것

| 제외 범위 | 제외 이유 |
| --- | --- |
| 견적서 자동 생성 | 회사마다 양식 달라 완성도 보장 불가 |
| 팀 공유 / 협업 | 개인용 서비스 — 팀 기능은 정체성 정면 훼손 |
| 결제 / 구독 (수익화) | 서비스 검증 후 별도 기획 (MVP 제외) |
| 캘린더 연동 (구글 캘린더) | 향후 기능 — 현 버전 미구현 |
| 전화 녹음 | 전화 끊김 등 구조적으로 완성도 낮음 |

# 2. Problem Hypothesis

## 문제 가설

> 우리는 사용자가 **"내가 발로 뛰며 쌓은 영업 정보가 흩어져 있고, 정작 필요한 현장에서 꺼내볼 수 없다"** 는 문제를 겪고 있다고 가정한다.

- 특히 회사 CRM에 못 쓰는 **비공식 인사이트**(결정권자, 성향, 예산 시점)를 기록할 곳이 없다.

## 원인 가설

| 번호 | 원인 가설 |
| --- | --- |
| 1 | 도구가 흩어져 있다 — 엑셀(딜) + 명함 서랍(인맥) + 메신저(메모)가 각각 분리됨 |
| 2 | 기존 CRM이 개인에게 안 맞다 — 무겁고, 회사 종속이라 떠나면 못 가져감 |
| 3 | 입력 비용이 높다 — 명함 수기 입력, 회의록 작성, 단가 찾기에 시간 소모 |

## 현재 사용자 행동 가설

> 사용자는 현재 이 문제를 해결하기 위해 **엑셀 + 명함 서랍 + 카카오톡 "나에게 보내기"** 를 조합해 쓰고 있다.

| 현재 사용 도구/방식 | 장점 | 한계 |
| --- | --- | --- |
| 노트북 엑셀 | 자유로운 양식, 익숙함 | 모바일에서 못 봄, 검색 약함, 우선순위 안 보임 |
| 명함 서랍 / 명함 앱 | 물리적 보관 간편 | 검색 불가, 딜·이력과 연결 안 됨 |
| 카카오톡 나에게 보내기 | 즉시 메모 | 맥락 단절, 거래처/딜과 연결 안 됨, 나중에 못 찾음 |
| 회사 CRM | 팀 표준, 보고 연동 | 무겁고 회사 종속, 비공식 인사이트 기록 불가 |

## 해결 가설

> 우리가 **"한 계정에 거래처/딜/명함/회의록을 모으고 PC·모바일에서 동일하게 꺼내보게"** 제공하면 사용자는 **흩어진 도구를 SalesKit 하나로 대체하고, 현장에서 즉시 조회/입력하는 행동**을 보일 것이다.

- 행동 변화 지표: 명함을 서랍 대신 촬영, 미팅 후 회의록을 SalesKit에 작성, 아침에 "이번 달 마감" 탭 확인.

## 수익/사업성 가설

> 사용자는 이 문제가 해결된다면 **월 9,900~19,900원** 수준의 구독료를 지불할 가능성이 있다.

- 근거 가설: 개인 생산성 도구(노션/투두 류) 지불 경험 + 영업 성과 직결성. **결제는 MVP 제외 → 검증 후 별도 기획**이므로 본 가설은 베타 인터뷰로 우선 검증.

## 검증해야 할 질문

| 번호 | 검증 질문 |
| --- | --- |
| 1 | 영업사원은 정말 "현장에서 꺼내볼 수 없어서" 곤란을 겪는가, 아니면 그냥 익숙해서 엑셀을 쓰는가? |
| 2 | 비공식 인사이트(오프더레코드)를 앱에 적는 데 거부감은 없는가? (보안 신뢰) |
| 3 | 명함 OCR 정확도가 어느 수준이어야 "서랍 대신 촬영"으로 행동이 바뀌는가? |
| 4 | AI 회의록이 실제 미팅 맥락을 쓸 만하게 요약하는가? (화자 분리 아닌 맥락 저장으로 충분한가) |
| 5 | 개인 영업사원이 자기 돈으로 월 구독할 의향이 있는가? 가격 저항선은? |

| Strength | 개인 영업에 좁게 특화 / 오프라인 쓰기 / 명함 OCR + AI 회의록 / 데이터 소유권("회사 옮겨도 내 것") |
| Weakness | 신규 브랜드 인지도 0 / 결제·수익화 미검증 / OCR·STT 외부 AI 비용 의존 |
| Opportunity | 개인 영업사원용 도구 공백 / 한국 영업직 수백만 / AI로 입력 비용 급감 |
| Threat | 리멤버 등 인접 서비스의 영업 기능 확장 / 빅 CRM의 개인 요금제 / 데이터 보안 민감도(오프더레코드) |

## TOWS 전략

| 전략 | 내용 |
| --- | --- |
| SO 전략 | "개인 특화 + AI 자동화"로 빅 CRM이 못 가는 개인 시장 선점 |
| WO 전략 | 무료/베타로 인지도 확보 후, 검증된 가치 위에서 결제 도입 |
| ST 전략 | 데이터 소유권·오프더레코드 보안을 신뢰 자산으로 차별화 |
| WT 전략 | 기능 범위를 의도적으로 좁혀("고깃집 김치찌개") 완성도로 방어 |

## 우리가 선택할 포지셔닝

> 우리는 **"팀/기업용 CRM"** 포지션이 아니라, **"영업사원 개인이 자기 자산을 쌓고 어디서든 꺼내보는, 의도적으로 좁은 개인 도구"** 포지션으로 간다.

# 6. User Flow + Wireframe

## 정보 구조

- 메인 화면
    - 대시보드 (이번달 마감 / 가능성 70%+ / 최근 활동)
    - 상단 전역 검색
- 상세 화면
    - 딜 목록 / 딜 상세 (정보 + 활동 타임라인)
    - 거래처 목록 / 거래처 상세 (정보 + 오프더레코드 + ContactLog + 연결 딜)
    - 회사 목록 / 회사 상세 (정보 + 소속 담당자 + 연결 딜 + 메모)
    - 제품 목록 / 제품 상세 (정보 + 단가 이력)
    - AI 회의록 목록 / 생성 / 상세
    - 명함 스캔
- 설정 화면
    - 계정 / 알림 설정 / 휴지통

## 핵심 사용자 흐름

1. 소셜 로그인 → 대시보드 진입
2. 명함 스캔(또는 수동) → 거래처/회사 등록
3. 딜 생성 → 담당자·제품 연결 → 단계/가능성/마감일 입력
4. 미팅 후 AI 회의록 작성 → 딜 연동(활동기록 자동 로그)
5. 대시보드/필터로 이번 달 우선순위 확인 → 후속 활동 기록

## 필수 화면 목록

| 화면 | 목적 | 핵심 요소 | 우선순위 |
| --- | --- | --- | --- |
| 로그인 (/login) | 소셜 인증 진입 | 카카오/구글/애플/네이버 버튼 | Must |
| 대시보드 (/) | 우선순위 한눈에 | 이번달 마감·70%+·최근활동 | Must |
| 딜 목록 (/deals) | 딜 관리 진입 | 탭/컬럼 필터·정렬·검색 | Must |
| 딜 상세 (/deals/:id) | 딜 맥락 추적 | 정보·활동 타임라인·복사·삭제 | Must |
| 거래처 상세 (/contacts/:id) | 담당자 맥락 | 정보·오프더레코드·ContactLog·연결딜 | Must |
| 회사 상세 (/companies/:id) | 조직 단위 파악 | 소속담당자·연결딜·메모 | Must |
| 제품 상세 (/products/:id) | 단가 관리 | 정보·단가 이력 | Should |
| 명함 스캔 (/contacts/scan) | 빠른 등록 | 카메라/업로드·OCR 결과 확인 | Must |
| AI 회의록 (/meetings, /new, /:id) | 미팅 맥락 저장 | 음성/텍스트 입력·9항목 요약·딜연동 | Must |
| 설정 (/settings) | 계정/알림/휴지통 | 알림 on/off·휴지통 복구 | Should |

## 화면별 메모

### 화면 1: 딜 상세 (/deals/:id)

- 목적: 하나의 거래가 어디까지 왔는지 맥락을 한 화면에서 파악
- 주요 버튼: 수정 / 복사 / 삭제 / 단계 변경 / 활동 추가 / 회의록 연동
- 입력값: 딜명·담당자·제품·금액·단계·가능성·마감일·지역 / 활동(날짜+텍스트)
- 예외 상황: 담당자 미연결 저장 불가, 단계 변경 시 자동 활동기록 생성, 삭제 시 활동기록 동반 휴지통

### 화면 2: 거래처 상세 (/contacts/:id)

- 목적: 담당자 한 명의 정보 + 비공식 맥락(고정 메모 + 시간순 로그) + 연결 딜을 모아 봄
- 주요 버튼: 수정 / 삭제 / 합치기(병합) / 메모 로그 추가
- 입력값: 이름(필수)·회사·부서·직책·전화·이메일·지역 / 오프더레코드(단일) / ContactLog(날짜+텍스트)
- 예외 상황: 오프더레코드·ContactLog는 전역검색 제외, 병합은 비가역(확인 다이얼로그), 삭제 시 연결 딜은 유지·연결만 해제

### 화면 3: AI 회의록 생성 (/meetings/new)

- 목적: 음성/텍스트를 AI가 9항목 회의록으로 구조화 (맥락 저장이 목표, 화자분리 아님)
- 주요 버튼: 녹음 시작/정지 / 텍스트 입력 / AI 생성 / 수정 / 저장 / 딜 연동
- 입력값: 음성 또는 자유 텍스트 → (자동) 날짜·회사·담당자·부서·품목·진행단계·상세내용·향후계획·필요액션
- 예외 상황: STT 실패 시 텍스트 입력 fallback, 저장 후에도 딜 연동 가능, 연동 시 딜 활동기록에 링크 로그 자동 생성

# 7. PRD

## 배경

- 영업사원의 핵심 자산인 영업 정보가 엑셀·명함·메신저에 흩어져 현장에서 활용 불가. 회사 CRM은 무겁고 회사 종속적. SalesKit은 개인이 소유하는 가벼운 영업 관리 도구로 이 공백을 채운다.

## 목표

| 목표 | 설명 |
| --- | --- |
| 흩어진 정보 통합 | 거래처·딜·명함·회의록을 한 계정에 |
| 현장 즉시 활용 | PC↔모바일 동기화 + 오프라인 쓰기 |
| 입력 비용 절감 | 명함 OCR + AI 회의록 + 엑셀 AI 매핑 |

## 타겟 사용자

- 계측기 기술영업사원(1차) → B2B 기술영업 전반 개인 사용자.

## 주요 기능 요구사항

## 기능 1: Deal(딜) 관리

### 목적

- 진행 중인 모든 거래를 한 곳에서 관리하고, 단계·가능성·마감일로 우선순위를 가시화.

### 입력값

| 항목 | 필수 여부 | 설명 |
| --- | --- | --- |
| 딜명 | 필수 | 거래 식별명 |
| 담당자(Contact) | 필수 | 딜 1건은 담당자 1명에 연결 |
| 제품(Product) | 선택 | 연결 시 단가 자동입력(이후 수정 가능, 시점 고정) |
| 예상 금액 | 선택 | 직접 입력/수정 |
| 단계 | 필수 | 초기접촉 / 협의중 / 성사 / 실패 |
| 구매 가능성 | 선택 | 0~100% 직접 입력 |
| 마감 예정일 | 선택 | 알림 대상 기준 |
| 지역 | 선택 | 필터 기준 |

### 기능 동작

1. 딜 생성 시 담당자 필수 연결, 제품 연결 시 단가 자동입력
2. 단계 변경 시 활동 타임라인에 "단계 변경: A → B" 자동 기록
3. 딜 복사 시 딜명/담당자/제품/금액/지역만 복사, 단계=초기접촉·가능성=0·마감일·활동기록은 초기화

### 완료 조건

- [ ] 딜 CRUD + 소프트 딜리트(30일 휴지통, 활동기록 동반) 동작
- [ ] 탭 필터(이번달 마감/70%+ 등) + 컬럼 필터 + 정렬 동작
- [ ] 검색에 딜명/담당자/회사/메모 포함, 오프더레코드 제외

### 예외 조건

- 담당자 미연결 시 저장 불가
- 제품 단가가 나중에 바뀌어도 딜 저장 금액은 불변

## 기능 2: AI 회의록

### 목적

- 영업 미팅 내용을 음성/텍스트로 입력하면 AI가 구조화된 회의록을 자동 생성하여 미팅 **맥락을 저장**(화자 분리 아님).

### 입력값

| 항목 | 필수 여부 | 설명 |
| --- | --- | --- |
| 입력 소스 | 필수 | 음성 녹음 또는 텍스트 직접 입력 |
| 연동 딜 | 선택 | 저장 후 언제든 연동 가능 |

### 기능 동작

1. 음성 → STT 텍스트 추출 → AI 회의록 생성 / 텍스트 → 바로 AI 회의록 생성
2. AI가 9항목(날짜·회사·담당자·부서·품목·진행단계·상세내용·향후계획·필요액션)으로 구조화, 저장 전 수정 가능
3. 딜 연동 시 해당 딜 활동기록 탭에 "AI 회의록 연동 — [제목]" + 상세 링크 로그 자동 생성

### 완료 조건

- [ ] 음성/텍스트 두 경로 모두 회의록 생성
- [ ] 생성 결과 수정 후 저장 + 목록/상세 조회
- [ ] 딜 연동 → 딜 활동 타임라인에 링크 로그 자동 반영

### 예외 조건

- STT 실패 시 텍스트 입력으로 fallback
- 딜 미연동 단독 저장 허용
