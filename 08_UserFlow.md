# User Flow (사용자 흐름도)

> [07_현재_확정된_기능.md](07_현재_확정된_기능.md)의 F-01~33을 기반으로,
> 실제 사용자가 앱을 어떻게 경유하는지 흐름으로 정리.
>
> **목적**:
> - 화면/기능을 만들 때 "이 흐름 안에서 어디 위치하는가?" 답하기
> - 빠진 화면/누락된 분기 발견
> - UX 일관성 검증 (예: 모든 에러 경로에 복구 액션이 있는가?)
>
> **표기**:
> - `[화면]` = UI 화면
> - `{시스템}` = 백엔드/외부 시스템 처리
> - `(조건)` = 분기 조건
> - `→` = 다음 단계
> - `⤴` = 이전 화면으로 복귀
>
> 작성일: 2026-05-21

## 0. 사용자 페르소나 (이 흐름의 주인공)

### P-1. 김영업 (User 웹 + 모바일)
- 30대 후반, 반도체 부품 영업
- 노트북(사무실) + 아이폰(외근/운전)
- 거래처 ~150명, 영업건 ~30건 동시 진행
- 핵심 페인포인트: "노트북에 적은 걸 운전 중 핸드폰에서 못 봄"

### P-2. 박관리 (Admin 웹)
- 서비스 운영자
- PC에서 전체 사용자/거래처 모니터링
- 사용자 정지·통계 조회 등

```

                    └────────────┬────────────┘
                                 │
                  ┌──────────────┴──────────────┐
                  │                             │
            (로그인 안 됨)                 (로그인 됨)
                  │                             │
                  ▼                             ▼
           [회원가입/로그인]              [홈/대시보드]
              (Flow A)                       (Flow B)
                  │                             │
                  ▼                             │
                  └──────────────┬──────────────┘
                                 │
        ┌────────────┬───────────┼───────────┬─────────────┐
        ▼            ▼           ▼           ▼             ▼

```

## 2. Flow A — 회원가입 / 로그인

**관련 기능**: F-01, F-02, F-03, F-05

```
[앱 시작]
  │
  ├─ (저장된 토큰 있음 + 유효함) → ⤴ Flow B [홈]
  │
  ├─ (저장된 토큰 있음 + 만료됨)
  │     │
  │     ▼
  │  {refresh token으로 갱신 시도}
  │     │
  │     ├─ (성공) → ⤴ Flow B [홈]
  │     └─ (실패) → [로그인 화면]
  │
  └─ (토큰 없음) → [로그인/회원가입 선택 화면]
                          │
                  ┌───────┴────────┐
                  ▼                ▼
            [로그인]           [회원가입]
            email + pw         email + pw + 확인
                  │                │
                  ▼                ▼
          {Supabase Auth}    {Supabase Auth + 사용자 생성}
                  │                │
        ┌─────────┴───┐     ┌──────┴──────┐
        ▼             ▼     ▼             ▼
     (성공)        (실패) (성공)        (실패)
        │             │     │             │
        │             │     │             │
        │   [에러 토스트]   │   [에러 토스트]
        │   "이메일/비밀번호 │   "이미 가입된 이메일"
        │    확인" + 다시   │    또는
        │             │     │   "비밀번호 형식 오류"
        │             │     │
        ▼             ⤴     ▼             ⤴
   [홈 (Flow B)]         [홈 (Flow B)]
```

**누락 위험 체크**:
- [ ] "비밀번호 잊음" 흐름은? → [P-002 소셜 로그인](04_의사결정_기록.md) 이전엔 Supabase Auth 표준 흐름 사용
- [ ] 이메일 인증 요구? → Supabase 기본 설정에 따름 (출시 전 결정 필요)

## 3. Flow B — 홈 / 메인 진입

**관련 기능**: F-05 + 다른 흐름의 진입점

### 3.1 User 웹 (노트북)
```
[홈/대시보드]
  │
  │ 좌측 사이드바 메뉴:
  │ - 홈
  │ - 거래처 (Flow C)
  │ - 영업건 (Flow D)
  │ - 회사 (Flow C의 일부)
  │ - 설정 (Flow H)
  │
  │ 메인 영역:
  │ - 빠른 검색 박스 (F-08)
  │ - 최근 영업건 N개 (선택)
  │ - 빠른 진입: "거래처 추가", "영업건 추가"
```

### 3.2 모바일 (Expo Router 탭)
```
[탭바] (하단 고정)
  │
  ├─ 홈 (대시보드)
  ├─ 거래처 (Flow C)
  ├─ 명함 (Flow E) ← 카메라 버튼 강조
  └─ 설정 (Flow H)

[홈 화면]
  │
  │ 상단: 검색 박스 (F-08)
  │ 중단: 동기화 상태 인디케이터 🟢🟡🔴 (F-29)
  │ 하단: 최근 본 거래처 / 진행 중 영업건
```

## 4. Flow C — 거래처 관리

**관련 기능**: F-06, F-07, F-08, F-09, F-10, F-14, F-15, F-16, F-17, F-18

```
[홈]
  │ "거래처" 메뉴
  ▼
[거래처 리스트]
  │
  │ 상단 영역:
  │  - 탭: [전체] [이번 달 가능성 높음] [지역별] (F-17)
  │  - 검색 박스 (F-08)
  │  - 컬럼 필터 드롭다운 (F-16): 지역, 상태, 회사 등
  │  - URL에 필터 상태 동기화 (F-18)
  │
  │ 메인 영역:
  │  - 거래처 카드/행 리스트
  │  - 각 카드: 이름, 회사, 전화, 마지막 메모 1줄
  │
  │ 우상단/플로팅: [+ 거래처 추가] 버튼
  │
  ├──→ [거래처 추가] (수동)         ─┐
  │     - 이름(필수)                │
  │     - 전화/이메일/지역/직급(선택) │
  │     - 회사 선택 또는 새로 만들기 │
  │     - 저장 → ⤴ [거래처 리스트]   │
  │                                  │
  ├──→ [거래처 추가 (명함)]          ─┤ → Flow E
  │     모바일 전용                   │
  │                                  │
  └──→ [거래처 상세 (특정 거래처)] ←─┘
        │
        │ 탭 구조:
        │ ┌──────┬───────┬─────────┬──────────┐
        │ │ 정보 │ 영업건│ 메모    │ 명함     │
        │ │      │       │(F-14,15)│ (원본)   │
        │ └──────┴───────┴─────────┴──────────┘
        │
        ├─ [정보 탭]
        │   - 이름, 전화, 이메일, 직급, 지역 표시
        │   - [수정] 버튼 → 인라인 폼
        │   - [삭제] 버튼 → 확인 다이얼로그 → Soft Delete (06번 C-01)
        │   - 회사 정보 (F-09, F-10): 클릭 시 [회사 상세]로 이동
        │
        ├─ [영업건 탭] → 이 거래처의 영업건 리스트 (Flow D 연결)
        │
        ├─ [메모 탭] (off-the-record, F-14, F-15)
        │   - 메모 리스트 (최신순)
        │   - [+ 메모 추가] → 텍스트 입력 → 저장
        │   - 각 메모: [수정] [삭제]
        │
        └─ [명함 탭]
            - OCR로 등록된 경우: 원본 이미지 표시 (06번 E-04)
            - 수동 등록된 경우: "명함 없음" + [명함 추가] 버튼
```

**[회사 상세] 화면** (F-09):
```
[회사 상세]
  │
  ├─ 회사 정보 (이름, 산업, 주소 등)
  ├─ 이 회사 소속 거래처 리스트 (F-10 역방향)
  └─ 회사 메모 탭 (off-the-record)
```

**에러/예외 경로**:
- 거래처 삭제 후 영업건이 남아있으면? → 영업건의 거래처 참조를 어떻게 할지 (DB 스키마 결정 시점)
- 같은 전화번호 거래처 중복 등록? → [06번 C-03 입력 검증](06_추가_권장_작업.md)에서 결정

## 5. Flow D — 영업건 관리

**관련 기능**: F-11, F-12, F-13

```
[홈]
  │ "영업건" 메뉴
  ▼
[영업건 리스트]
  │
  │ 탭/필터:
  │  - 상태별: 전체 / lead / in_progress / won / lost (F-13)
  │  - 이번 달 가능성 높음 (F-17)
  │
  │ 각 영업건 표시:
  │  - 제목, 거래처, 상태, 예상 금액, 가능성, 예상 마감일
  │
  ├──→ [+ 영업건 추가]
  │     - 제목(필수)
  │     - 거래처 선택(필수) ← Flow C와 연결
  │     - 상태(default: lead)
  │     - 금액/가능성/예상 마감일 (선택)
  │     - 메모
  │     - 저장 → ⤴ [영업건 리스트]
  │
  └──→ [영업건 상세]
        │
        ├─ 기본 정보 표시 + [수정] [삭제]
        ├─ 상태 변경 (lead → in_progress → won/lost)
        ├─ 연결된 거래처 → 클릭 시 Flow C로
        └─ 메모/히스토리
```

**상태 전이 규칙** (확정된 의미만):
```
  lead ─┬→ in_progress ─┬→ won
        │               └→ lost
        └→ lost
```

> 역방향(won → lead) 허용 여부는 DB 스키마 단계에서 결정.

## 6. Flow E — 명함 스캔 (모바일 전용)

**관련 기능**: F-23, F-24, F-25, F-26

```
[홈] 또는 [거래처 리스트]
  │ 카메라 버튼 또는 "명함 추가"
  ▼
[카메라 권한 확인]
  │
  ├─ (권한 있음) → [카메라 화면]
  │
  └─ (권한 없음)
        │
        ▼
     [권한 요청 다이얼로그]
        │
        ├─ (허용) → [카메라 화면]
        │
        └─ (거부)
              │
              ▼
           [권한 거부 안내 화면]
              "명함 스캔 기능에는 카메라 권한이 필요합니다"
              [설정 앱으로 이동] | [건너뛰고 수동 입력]
                    │                      │
                    ▼                      ▼
                [iOS/Android 설정]    [거래처 수동 입력]
                                       (Flow C로)

[카메라 화면]
  │ - 명함 가이드 박스 (정사각형 또는 직사각형 프레임)
  │ - 촬영 버튼
  │ - 취소 버튼 → ⤴
  │
  ▼
[촬영 후 미리보기]
  │
  ├─ [다시 찍기] → ⤴ [카메라 화면]
  └─ [사용] →
        │
        ▼
    {1. signed URL 요청}
    {2. Supabase Storage 직접 업로드}
    {3. 백엔드에 OCR 처리 요청 (CLOVA)}
        │
        ├─ (성공) → [OCR 결과 확인 화면]
        │
        └─ (실패) → [OCR 실패 안내]
                      "명함을 인식하지 못했습니다"
                      [다시 시도] | [수동 입력] | [이미지만 저장하고 나중에]
                          ▲           │              │
                          │           ▼              ▼
                      [카메라]    [수동 입력 폼]   {OCR 대기 상태}
                                  (이미지 포함)

[OCR 결과 확인 화면] (F-25)
  │
  │ 자동 채워진 필드 (수정 가능):
  │  - 이름
  │  - 전화번호
  │  - 이메일
  │  - 회사명
  │  - 직급
  │  - 부서
  │  - 주소
  │
  │ 원본 이미지 미리보기 (06번 E-04)
  │
  ├─ 회사명이 기존 회사와 매칭되는 경우:
  │   "삼성반도체에 이미 등록된 회사가 있습니다. 연결하시겠습니까?"
  │   [기존 회사에 연결] | [새 회사로 등록]
  │
  ├─ [저장] → 거래처 자동 생성 → ⤴ [거래처 상세 (방금 생성된 것)]
  │
  └─ [취소] → 확인 다이얼로그 ("스캔한 정보가 사라집니다")
              → ⤴ [홈]
```

**예외 케이스 — 누락 가능**:
- 카메라가 없는 디바이스? → 모바일 한정 기능이므로 거의 무시 OK
- OCR이 같은 사람을 이미 등록된 거래처로 인식 → 중복 체크 정책 ([06번 C-03](06_추가_권장_작업.md))
- 신뢰도 낮은 OCR 결과 → [06번 C-07 OCR 실패/저품질 처리](06_추가_권장_작업.md)와 연결

## 7. Flow F — 검색 / 필터

**관련 기능**: F-08, F-16, F-17, F-18

```
[거래처 리스트] 또는 [영업건 리스트]
  │
  ▼
[검색 / 필터 영역]
  │
  ├─ 검색 박스 (F-08)
  │     │ 타이핑 → 디바운스 → 백엔드 검색 호출
  │     │ 검색 범위: 거래처명 / 회사명 / 메모 (06번 C-04)
  │     │
  │     ├─ (결과 있음) → 리스트 갱신
  │     └─ (결과 없음) → "검색 결과 없음" + [필터 초기화]
  │
  ├─ 컬럼 필터 (F-16)
  │     │ 드롭다운: 지역, 상태, 회사 등
  │     │ 다중 선택 가능
  │     │ 선택 → URL searchParams 갱신 (F-18) → 백엔드 쿼리
  │
  └─ 탭 필터 (F-17)
        │ [전체] [이번 달 가능성 높음] [지역별]
        │ 탭 변경 → URL 변경 → 리스트 갱신
        │ "지역별" 탭은 지역별 그룹핑 UI로 전환
```

**URL 동기화** (F-18):
```
/customers?region=서울&status=in_progress&tab=monthly-likely&q=송재근
```
이 URL을 누군가 공유하면 같은 필터 상태로 진입.

## 8. Flow G — 엑셀 / PDF 입출력

**관련 기능**: F-19, F-20, F-21, F-22

### 8.1 엑셀 Export (F-19)
```
[거래처 리스트] 또는 [영업건 리스트]
  │ 현재 필터된 상태에서
  ▼
[Export 메뉴]
  │
  ├─ [엑셀로 내보내기]
  │     │ 옵션 선택 (선택 사항):
  │     │  - 어떤 컬럼 포함할지
  │     │  - 메모 포함 여부
  │     │
  │     ▼
  │  {백엔드: exceljs로 파일 생성}
  │     │
  │     ▼
  │  [다운로드 자동 시작]
  │
  └─ [PDF로 내보내기] (F-20)
        │ "월말 보고용" 같은 템플릿 선택 (06번 E-07)
        ▼
     {백엔드: PDF 생성}
        │
        ▼
```

## 11. Flow J — Admin 웹 (관리자 전용)

**관련 기능**: F-31, F-32, F-33

```
[admin.yourdomain.com 진입]
  │
  ▼
[로그인 화면]
  │ email + password
  ▼
{Supabase Auth + JWT 발급}
  │
  │ JWT에 role 포함
  │
  ├─ (role !== 'ADMIN') → 즉시 로그아웃 + 에러
  │     "관리자 권한이 없습니다"
  │     ⤴ [로그인]
  │
  └─ (role === 'ADMIN') → [Admin 대시보드]

[Admin 대시보드]
  │
  │ 좌측 사이드바:
  │  - 대시보드 (지표/차트)
  │  - 사용자 (사용자 목록/상세)
  │  - 전체 거래처 (모든 사용자의)
  │  - 전체 영업건
  │  - 통계
  │  - 감사 로그 (자리만)
  │  - 설정
  │
  │ 메인 영역:
  │  - 지표 카드 (DAU, 가입자 수 등)
  │  - 차트 (Recharts)
  │
  ├──→ [사용자 목록]
  │     │ TanStack Table (정렬/필터/페이지)
  │     │ 행 클릭 → [사용자 상세]
  │     │
  │     ▼
  │   [사용자 상세]
  │     - 사용자 정보
  │     - 그 사용자의 거래처/영업건 수
  │     - [정지] [활성화] (AlertDialog + 사유 입력)
  │     │
  │     └─ (위험 액션) → [확인 다이얼로그 + 사유 텍스트]
  │           → {감사 로그 기록 + DB 변경 (같은 트랜잭션)}
  │
  ├──→ [전체 거래처] (모든 사용자의 거래처)
  ├──→ [전체 영업건]
  └──→ [감사 로그] (Phase 2~3 — 자리만 확보)
```

**Admin 화면 UX 원칙** ([AGENT/Admin/architecture.md](AGENT/Admin/architecture.md) 10):
- 모바일 반응형 X (1280px+ 데스크탑 전용)
- 위험한 액션은 항상 AlertDialog + 사유 입력
- 모든 변경은 감사 로그에 자동 기록
