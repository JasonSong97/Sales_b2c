# SalesKit 기능 명세 v2

---

## 1. 서비스 정의

**한 줄 정의**
영업직 개인이 고객 · 딜 · 거래처 · 제품 데이터를 한 곳에 쌓고, PC와 모바일에서 동일하게 조회 · 관리할 수 있는 개인용 영업 관리 도구.

**타겟 사용자**
계측기 기술영업사원을 시작으로, B2B 기술영업 전반 (산업재, 의료기기, 화학약품 등) 개인 사용자. 팀 단위 도입이 아닌 개인이 직접 결제해서 쓰는 B2C 구조.

**핵심 페인포인트**

- 노트북에 기록한 데이터를 이동 중 모바일에서 못 봄
- 거래처 정보가 명함 · 엑셀 · 메신저에 흩어져 있음
- 영업 기록할 때 제품 단가를 다른 파일에서 찾아야 함
- 미팅 후 회의록 작성에 시간이 걸림
- 기존 CRM은 회사용이라 개인이 쓰기 무거움

## 2. 도메인 구조

서비스의 핵심 데이터는 7개 엔티티로 구성된다.

```
User (사용자)
 ├── Company (회사)
 │    └── Contact (담당자)
 │         ├── ContactLog (담당자 메모 로그)
 │         └── Deal (딜)
 │              ├── DealActivity (딜 활동 기록)
 │              └── MeetingNote (회의록, 선택 연동)
 └── Product (제품)
```

**User** — 서비스 계정. 모든 데이터는 User 단위로 완전히 격리된다.

**Company** — 거래처 회사. 담당자(Contact) 여러 명이 하나의 Company에 속할 수 있다.

**Contact** — 거래처 담당자 개인. 명함의 정보 단위. Company에 소속되거나 독립적으로 존재할 수 있다.

**ContactLog** — 담당자에 누적되는 메모 로그. 날짜 + 자유 텍스트 타임라인. 딜과 무관한 담당자 개인에 대한 맥락 정보를 시간 순서로 쌓는다.

**Deal** — 영업 건(기회). 하나의 Contact에 연결되며, 하나의 제품을 참조할 수 있다.

**DealActivity** — 딜에 쌓이는 활동 기록. 날짜 + 내용의 타임라인.

**MeetingNote** — AI가 생성한 영업 회의록. 딜과 선택적으로 연동.

**Product** — 판매 제품 및 단가. 딜 생성 시 참조용.

## 3. 기능 상세

---

### 3.1 인증

**소셜 로그인**

- 카카오 로그인
- 구글 로그인
- 애플 로그인
- 네이버 로그인

**이메일 로그인 / 회원가입**

- 이메일 + 비밀번호
- 가입 후 이메일 인증 필수. 미인증 상태에서는 로그인 불가
- 인증 메일 유효시간 24시간. 만료 시 재발송 가능

**공통**

- 로그인 유지: 자동 토큰 갱신 (30일)
- 로그아웃 시 토큰 무효화
- 비밀번호 재설정: 이메일로 재설정 링크 발송, 유효시간 1시간

---

### 3.2 Company (회사 관리)

**등록**

- 회사명 (필수)
- 업종, 주소, 대표 전화, 웹사이트 (선택)
- 오프더레코드 메모 — 비공개 자유 메모. 예: "연말에 예산 풀림", "구매 결정권은 설비팀장"

**조회 / 수정**

- 회사 상세 화면에서 소속 담당자(Contact) 목록 바로 확인
- 회사 상세 화면에서 이 회사와 연결된 딜 목록 바로 확인

**삭제**

- 소프트 딜리트. 삭제 시 30일간 휴지통 보관 후 완전 삭제
- 회사 삭제 시 소속 담당자(Contact)는 삭제되지 않음. Company 연결만 해제됨

---

### 3.3 Contact (거래처 담당자 관리)

**등록 — 명함 OCR**

- 카메라 촬영 또는 이미지 업로드
- AI(Claude API Vision)가 이름 · 회사 · 부서 · 직책 · 전화번호 · 이메일 · 주소 자동 추출
- 추출 결과는 수정 가능한 폼으로 표시. 확인 후 저장
- 저장 시 기존 Company와 자동 매칭 시도 (회사명 일치 시 연결 제안). 없으면 새 Company 생성 또는 독립 Contact으로 저장 선택

**등록 — 수동 입력**

- 이름 (필수)
- 회사(Company) 연결 (선택), 부서, 직책, 전화번호, 이메일, 지역 (선택)

**담당자 오프더레코드 메모**

- 담당자 개인 단위 비공개 고정 메모 (단일 텍스트 필드)
- 자주 바뀌지 않는 기본 맥락 정보를 적어두는 용도
- 예: "직접 연락 싫어함. 항상 대리 통해서", "실제 결정권 없음. 김부장 통해야"

**담당자 메모 로그 (ContactLog)**

담당자에 대해 시간 순서로 메모를 누적하는 타임라인. 딜 단위가 아닌 담당자 개인에 대한 맥락 정보를 쌓는 용도.

- 날짜 + 자유 텍스트 입력으로 메모 추가
- 최신 항목이 위에 표시 (역순 정렬)
- 항목별 수정 / 삭제 가능
- 오프더레코드 메모(고정 단일 필드)와는 별도로 관리
- 등록 예시:
    - "2025-05-10 — 삼성전자 출신. 구매 키맨임. 예산 결정권 있음"
    - "2025-05-20 — 경쟁사 제품 써봤다고 함. 납기 민감"
    - "2025-05-27 — 직책 바뀜. 책임에서 수석으로 승진"
- 검색 대상: 오프더레코드 메모와 동일하게 전역 검색에서 제외

**거래처 합치기**

- 같은 사람이 중복 등록된 경우 두 Contact를 하나로 병합
- 병합 시 두 Contact에 연결된 딜은 모두 살아남은 Contact로 이전
- 병합은 되돌릴 수 없음. 실행 전 확인 다이얼로그 표시

**엑셀 일괄 가져오기**

- .xlsx 파일 업로드
- 업로드 후 컬럼 매핑 화면 표시 (엑셀 컬럼 → 앱 필드 연결)
- 중복 감지: 이메일 또는 전화번호 기준으로 기존 데이터와 대조
- 중복 발견 시 "덮어쓰기 / 건너뛰기" 선택
- 오류 행은 건너뛰고 성공한 행만 등록. 완료 후 "성공 N건 / 실패 N건" 결과 표시

**삭제**

- 소프트 딜리트. 30일 휴지통
- Contact 삭제 시 연결된 Deal은 삭제되지 않음. Contact 연결만 해제됨

---

### 3.4 Deal (딜 관리)

**등록**

- 딜명 (필수)
- 담당자(Contact) 연결 (필수)
- 제품(Product) 연결 (선택) — 연결 시 단가 자동 입력. 이후 금액은 사용자가 직접 수정 가능. 제품 단가가 나중에 바뀌어도 딜에 저장된 금액은 변경되지 않음
- 예상 금액 (선택)
- 단계: 초기접촉 / 협의중 / 성사 / 실패
- 구매 가능성: 0~100% 직접 입력
- 마감 예정일 (선택)
- 지역 (선택)

**딜 복사**

- 복사되는 항목: 딜명, 담당자, 제품, 금액, 지역
- 복사되지 않는 항목: 단계(초기접촉으로 초기화), 가능성(0으로 초기화), 마감 예정일, 활동 기록

**딜 활동 기록 (타임라인)**

- 날짜 + 텍스트로 누적 기록. 최신 항목이 위에 표시
- 항목별 수정 / 삭제 가능
- 단계 변경 시 자동으로 활동 기록 생성: "단계 변경: 초기접촉 → 협의중"
- 회의록 연동 시 해당 날짜로 활동 기록 자동 생성. 내용에 회의록 링크 포함

**조회 · 정렬 · 필터**

탭 필터 (고정)

- 전체 / 초기접촉 / 협의중 / 성사 / 실패
- 이번달 마감 (마감 예정일이 이번 달인 딜)
- 가능성 70%+ (구매 가능성 70 이상)

컬럼 필터 (조합 가능)

- 지역, 구매 가능성 범위, 담당자, 회사

정렬 기준

- 마감 예정일 가까운 순 / 먼 순
- 예상 금액 높은 순 / 낮은 순
- 구매 가능성 높은 순 / 낮은 순
- 최근 수정순 (기본값)

검색 대상

- 딜명, 담당자 이름, 회사 이름, 메모 내용
- 오프더레코드 메모는 검색 제외

**삭제**

- 소프트 딜리트. 30일 휴지통
- 딜 삭제 시 활동 기록도 함께 삭제

---

### 3.5 Product (제품 관리)

**등록**

- 제품명 (필수)
- 모델번호, 단가, 카테고리, 설명 (선택)

**단가 이력 관리**

- 단가 수정 시 이전 단가와 변경일을 이력으로 보관
- 딜에 이미 연결된 금액은 변경되지 않음 (입력 시점 고정)
- 제품 상세 화면에서 단가 변경 이력 조회 가능

**엑셀 일괄 가져오기**

- Contact 가져오기와 동일한 방식 (컬럼 매핑 → 부분 성공 허용)

**삭제**

- 소프트 딜리트. 30일 휴지통
- 제품 삭제 시 연결된 딜의 금액은 유지. 제품 연결만 해제됨

### 3.6 AI 회의록

### 개요

영업 미팅 내용을 음성 또는 텍스트로 입력하면 AI가 구조화된 회의록을 자동 생성한다. 생성된 회의록은 딜과 연동 가능하며, 딜의 활동 기록 타임라인에 자동 반영된다.

### 화면 위치

상단 탭 내비게이션의 **"AI 회의록"** 탭에 독립적으로 위치.

### 화면 구성

1. **회의록 목록** — 생성된 회의록 카드 목록. 날짜 / 회사명 / 딜 연동 여부 표시. 클릭 시 회의록 상세 페이지 이동
2. **회의 음성노트 버튼** — 음성 녹음 기반 회의록 생성 플로우 진입
3. **회의 노트 버튼** — 텍스트 직접 입력 기반 회의록 생성 플로우 진입

### 기능 상세

**음성노트 플로우**

1. 녹음 시작 / 정지
2. 녹음 완료 후 음성 → 텍스트 자동 추출 (STT)
3. 추출된 텍스트를 기반으로 AI 회의록 자동 생성

**텍스트 노트 플로우**

1. 회의 내용 자유 텍스트 입력
2. 입력 내용을 기반으로 AI 회의록 자동 생성

**AI 회의록 생성 결과**

생성된 회의록은 아래 구조로 자동 정리된다. 모든 항목은 저장 전 수정 가능.

| 항목 | 설명 |
| --- | --- |
| 날짜 | 미팅 날짜 (자동 추출 또는 입력) |
| 회사 | 거래처 회사명 |
| 담당자 | 미팅 상대방 이름 / 직책 |
| 부서 | 담당 부서 |
| 품목 | 논의된 제품 또는 서비스 |
| 진행단계 | 현재 영업 단계 요약 |
| 상세내용 | 미팅 주요 내용 |
| 향후계획 | 다음 액션 방향 |
| 필요액션 | 구체적인 후속 조치 항목 |

**딜 연동**

- 회의록 저장 후 "영업 딜과 연동" 버튼 표시
- 연동할 딜 선택 시 해당 딜의 활동 기록 탭에 자동으로 로그 생성
    - 로그 내용: 해당 날짜, "AI 회의록 연동 — [회의록 제목]" + 회의록 상세 페이지 링크
- 연동은 회의록 저장 이후에도 언제든 가능
- 연동된 딜 정보는 회의록 상세 화면에 표시

### 3.7 검색

- 전역 검색 (상단 검색창): 딜 · 담당자 · 회사 통합 검색
- 검색 대상: 딜명, 담당자 이름, 회사명, 딜 메모 내용
- 검색 제외: 오프더레코드 메모
- 결과는 카테고리별 (딜 / 담당자 / 회사) 구분 표시

---

### 3.8 데이터 입출력

### 엑셀 내보내기

- 대상: 딜 목록 / 거래처(Contact) 목록 / 회사(Company) 목록 각각 개별 내보내기
- 현재 적용된 필터 · 탭 조건 그대로 내보내기
- 오프더레코드 메모는 기본적으로 제외. 사용자가 포함 여부 선택 가능

### PDF 내보내기

- 대상: 딜 목록 / 거래처(Contact) 목록 / 회사(Company) 목록 각각 개별 내보내기
- 헤더에 날짜, 사용자 이름 자동 포함
- 오프더레코드 메모 제외

### 엑셀 가져오기 — AI 자동 변환 (딜 / 거래처 / 제품 공통)

기존 엑셀 파일의 양식이 서비스 양식과 다를 경우에도 AI가 자동으로 변환한다.

1. 사용자가 .xlsx 파일 업로드
2. AI가 컬럼 구조를 분석하여 서비스 필드와 자동 매핑 (JSON 변환)
3. 매핑 결과를 화면에 표시. 사용자가 확인 및 수정 후 가져오기 확정
4. 중복 감지: 이메일 또는 전화번호(Contact), 딜명 기준(Deal)으로 기존 데이터 대조
5. 중복 발견 시 "덮어쓰기 / 건너뛰기" 선택
6. 오류 행은 건너뛰고 성공한 행만 등록
7. 완료 후 "성공 N건 / 실패 N건 / 건너뜀 N건" 결과 표시

### 3.9 알림

**이메일 알림**

- 마감 예정일 3일 전 딜 알림 이메일 발송
- 마감 예정일 당일 알림 이메일 발송
- 설정에서 on/off 가능

**푸시 알림 (PWA)**

- 위와 동일한 조건으로 푸시 알림 발송
- 브라우저 알림 권한 요청 후 사용자 동의 시 활성화
- 설정에서 on/off 가능

**알림 조건**

- 마감 예정일이 입력된 딜만 알림 대상
- 단계가 성사 또는 실패인 딜은 알림 제외

### 3.10 멀티 디바이스 동기화

- PC에서 입력한 데이터는 모바일에서 즉시 조회 가능
- 오프라인 상태에서는 마지막 동기화 데이터 조회만 가능. 데이터 쓰기는 온라인 상태에서만 허용
- 온라인 복귀 시 자동 동기화. 별도 조작 불필요

### 3.11 휴지통

**동작 방식**

- 딜(Deal) · 거래처(Contact) · 회사(Company) · 제품(Product)은 삭제 시 즉시 완전 삭제되지 않고 휴지통으로 이동
- 삭제 시점에 삭제 날짜(deleted_at)가 기록되며, 해당 항목은 일반 화면에서 즉시 보이지 않음
- 휴지통 화면에서는 삭제된 항목과 삭제 날짜를 확인 가능
- 삭제일로부터 30일 이내에는 복구 가능
- 30일 경과 후 완전 삭제 (복구 불가)

**휴지통 화면**

- 설정 메뉴 → 휴지통에서 접근
- 항목 유형별 탭 구분: 딜 / 거래처 / 회사 / 제품
- 각 항목에 삭제 날짜 및 완전 삭제까지 남은 일수 표시
- 개별 복구 / 영구 삭제 선택 가능
- 전체 비우기 (30일 미경과 항목도 즉시 영구 삭제) 가능

**연관 데이터 처리**

| 삭제 대상 | 연관 데이터 처리 |
| --- | --- |
| Deal | 활동 기록(DealActivity)도 함께 휴지통 이동 |
| Contact | 메모 로그(ContactLog)도 함께 휴지통 이동. 연결된 Deal은 유지. Contact 연결만 해제 |
| Company | 소속 Contact는 유지. Company 연결만 해제 |
| Product | 연결된 딜의 금액은 유지. Product 연결만 해제 |

### 3.12 캘린더 연동 (향후 기능)

현재 버전에서는 미구현. 아래는 향후 개발 방향으로 기획에 포함.

- 구글 캘린더 OAuth 연동
- 딜에 일정을 직접 연결하는 방식으로 구현 (딜 상세 → 일정 추가)
- 구글 캘린더 일정과 딜을 양방향 연동: 일정 변경 시 딜 내 일정 정보도 업데이트
- 대시보드에서 이번 주 / 이번 달 연동 일정 조회 가능
